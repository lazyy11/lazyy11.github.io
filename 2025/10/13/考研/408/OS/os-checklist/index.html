操作系统自查表

第一章 操作系统概述 - 考点自查 Checklist

一、 概念要点

1. 操作系统的基本概念

  • 操作系统的定义与层次

    • [ ] 你能从“资源管理者”和“服务提供者”两个不同角度，完整地阐述操作系统的定义吗？

    • [ ] 操作系统的四大核心管理功能是什么？请为每一项管理功能，至少列举出2个具体的管理任务。（例如：处理机管理 -> 进程调度、进程同步）
    • [ ] 操作系统提供给用户的接口有哪两类？这两类接口中的“联机命令接口”和“脱机命令接口”分别适用于哪种类型的操作系统？
    • [ ] 应用程序员通过什么方式来请求操作系统服务？我们常用的图形用户界面（GUI）属于操作系统的核心部分吗？为什么？
    • [ ] 操作系统的两个核心目标是什么？它如何实现将“裸机”改造为“扩充机器”？

  • 操作系统的特征（高频考点）

    • [ ] 并发 vs 并行：请用一句话说出“并发”与“并行”最根本的区别。单核CPU上能否实现并行？能否实现并发？
    • [ ] 共享：资源共享分为哪两种方式？打印机属于哪一种共享方式？硬盘文件属于哪一种？为什么必须这样划分？
    • [ ] 并发与共享的关系：为什么说并发和共享是互为存在条件？
    • [ ] 虚拟：虚拟技术是通过哪两种核心技术实现的？请分别为“时分复用”和“空分复用”技术各举一个典型的例子。
    • [ ] 异步：进程的“异步性”是指什么？这种特性对程序设计提出了什么挑战？操作系统必须为此提供什么保证？
2. 操作系统的发展历程
  • [ ] 手工操作阶段：这个阶段最突出的问题是什么？（提示：从人与机器两个角度回答）
  • [ ] 批处理阶段：
    • [ ] “单道批处理系统”相比手工操作，解决了什么核心问题？但它自身最主要的缺点是什么？（提示：与CPU利用率有关）
    • [ ] “多道批处理系统”的核心思想是什么？它又是如何解决单道批处理系统缺点的？
    • [ ] 多道批处理系统的引入，给操作系统带来了哪四大需要解决的新问题？
    • [ ] 多道批处理系统的主要优点是什么？它最大的缺点又是什么？（提示：与用户交互有关）
  • [ ] 分时操作系统：
    • [ ] 分时系统设计的首要目标是什么？其核心技术是什么？
    • [ ] 请描述分时系统的四个主要特征，并解释“及时性”与实时系统的“及时性”有何不同。
  • [ ] 实时操作系统：
    • [ ] 实时系统最重要的特征是什么？它与分时系统的关键区别在哪里？
    • [ ] “硬实时”和“软实时”系统如何区分？请各举一个例子。
  • [ ] 网络OS vs 分布式OS：这两个概念极易混淆。请问它们最本质的区别是什么？（提示：从多台计算机的“关系”和“任务目标”上区分）

    • [ ] CPU为什么要有“用户态”和“内核态”之分？这两种状态在权限上的核心区别是什么？硬件上通常用什么来标识当前状态？

  • 中断和异常（必考点）

    • [ ] 中断和异常的根本区别是什么？（提示：从事件来源区分）
    • [ ] 中断分为哪两类？异常分为哪三类？“缺页”属于哪一种？“系统调用”的陷入指令属于哪一种？

    • [ ] 什么是中断向量表？它的作用是什么？
    • [ ] 中断处理和子程序调用有何区别？（至少说出三点）

  • 系统调用

    • [ ] 为什么说系统调用是应用程序访问操作系统内核功能的唯一途径？
    • [ ] 系统调用处理全流程：请描述用户程序发起一次系统调用的完整步骤，包括参数传递、如何从用户态陷入内核态、内核如何找到对应的服务程序、以及最后如何返回用户态。
    • [ ] 我们在C语言中使用的printf()或fopen()这类库函数，和系统调用是什么关系？调用库函数的开销和直接调用系统调用的开销，哪个更大？为什么？
4. 操作系统结构

  • [ ] 宏内核 vs 微内核（高频对比）

    • [ ] 微内核的设计哲学是什么？哪些最核心的功能被保留在内核中？文件系统、设备驱动等服务在哪里运行？

  • 其他结构

  • [ ] MBR（主引导记录）位于硬盘的什么位置？它包含了哪两部分关键信息？
