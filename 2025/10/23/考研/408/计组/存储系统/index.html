存储系统

存储器概述

  • 存储器的分类

    • 按在计算机中的作用（层次）分类
      • 主存储器（主存/内存，Main Memory）
        • 定义：存放计算机运行期间所需程序和数据，CPU可直接随机访问，可与Cache、辅存交换数据的存储器。
        • 性质/特点：容量较小；存取速度较快；单位成本较高。
      • 辅助存储器（辅存/外存，Auxiliary Memory）
        • 定义：存放当前暂时不用的程序、数据及需永久性保存信息，内容需调入主存后才能被CPU访问的存储器。
        • 性质/特点：容量大；存取速度较慢；单位成本低。
      • 高速缓冲存储器（Cache）
        • 定义：位于主存和CPU之间，存放CPU当前常用指令和数据，以匹配CPU速度的高速存储器。
        • 性质/特点：存取速度与CPU速度匹配；存储容量小；价格高；现代计算机通常集成在CPU中。
    • 按存储介质分类
      • 定义：根据存储信息的物理载体划分，包含四类。
      • 具体类型：
        • 磁表面存储器：如磁盘、磁带。
        • 磁芯存储器：早期存储器，现已较少使用。
        • 半导体存储器：如MOS型存储器、双极型存储器（SRAM、DRAM属于此类）。
        • 光存储器：如光盘。
    • 按存取方式分类
      • 随机存储器（RAM，Random Access Memory）
        • 定义：任何存储单元可随机存取，存取时间与物理位置无关的存储器。
        • 性质/特点：读/写方便、使用灵活；主要用作主存或Cache；分为静态RAM（SRAM）和动态RAM（DRAM）。
      • 只读存储器（ROM，Read-Only Memory）
        • 定义：内容仅可随机读出，断电后信息不丢失的存储器，广义上含可反复重写类型。
        • 性质/特点：断电内容保留；随机读取；写入速度远慢于读取速度；常用于存放固定程序、常数、汉字字库。
        • 衍生类型：掩模式ROM（MROM）、一次可编程ROM（PROM）、可擦除可编程ROM（EPROM）、Flash存储器。
      • 串行访问存储器
        • 定义：需按物理位置先后顺序寻址的存储器，分为顺序存取和直接存取两类。
        • 具体类型：
          • 顺序存取存储器（SAM）：如磁带，存取时间与信息物理位置相关，速度慢。
          • 直接存取存储器（DAM）：如磁盘、光盘，先找小区域（如磁道），再顺序查找，速度介于RAM与SAM之间。
    • 按信息的可保存性分类
      • 易失性存储器
        • 定义：断电后存储信息立即消失的存储器。
        • 典型示例：RAM（SRAM、DRAM）。
      • 非易失性存储器
        • 定义：断电后信息仍保持的存储器。
        • 典型示例：ROM、磁表面存储器、光存储器。
      • 破坏性/非破坏性读出
        • 破坏性读出：读出时原存储信息被破坏，需读后再生（如DRAM）。
        • 非破坏性读出：读出时原存储信息不破坏（如SRAM、ROM）。

  • 存储器的性能指标

    • 存储容量
      • 定义：存储器可存储二进制信息的总量，计算公式为“存储容量 = 存储字数 × 字长”。
      • 单位换算：1B（Byte，字节）= 8b（bit，位）；常见单位：MB（2²⁰B）、GB（2³⁰B）。
      • 说明：存储字数表示地址空间大小，字长表示一次存取的数据量。
    • 单位成本
      • 定义：衡量存储器性价比的指标，计算公式为“位价 = 总成本 / 总容量”。
    • 存储速度
      • 定义：衡量存储器数据读写快慢的指标，包含存取时间、存取周期、主存带宽三个核心参数。
      • 具体参数：
        • 存取时间（TaT_aTa​）：从启动存储器操作到完成操作的时间，分读出时间和写入时间。
        • 存取周期（TmT_mTm​）：连续两次独立访存操作（读/写）的最小时间间隔，通常 Tm>TaT_m > T_aTm​>Ta​（含复原时间）；破坏性读出存储器可能 Tm=2TaT_m = 2T_aTm​=2Ta​（需再生）。
        • 主存带宽（BmB_mBm​）：每秒从主存进出信息的最大数量，单位为字/秒、B/s或b/s，计算公式为“数据传输速率 = 数据宽度 / 存取周期”。

  • 多级层次的存储系统

    • 定义：为解决存储系统“大容量、高速度、低成本”三者矛盾，采用由上至下速度递减、容量递增、位价递减的多级结构。
    • 结构/组成：
      • 层级划分：从CPU到外存依次为“寄存器 → Cache → 主存 → 磁盘 → 磁带/光盘”。
      • 核心层次：
        • Cache-主存层：解决CPU与主存速度不匹配问题，数据调动由硬件自动完成，对程序员透明。
        • 主存-辅存层：解决存储容量问题，数据调动由硬件+操作系统完成，对应用程序员透明（形成虚拟存储系统）。
    • 核心思想：上一层存储器作为下一层的高速缓存，CPU访存时优先访问上层，缺失则向下层查找并调入上层；从CPU视角，上层速度+下层容量/位价，平衡三者矛盾。
    • 关键特性：上一层内容是下一层的副本（如Cache是主存的部分副本，主存是辅存的部分副本）。

主存储器

  • SRAM芯片和DRAM芯片

    • 静态随机存储器（SRAM）
      • 定义：采用双稳态触发器（六晶体管MOS）存储信息的随机存储器，属于易失性存储器。
      • 工作原理：通过触发器的两种稳定状态记忆0和1，读出后信息保持不变（非破坏性读出），无需再生。
      • 性质/特点：
        • 存取速度快；集成度低；功耗大；价格高。
        • 无需定时刷新；地址线同时传送行、列地址，无复用。
      • 主要用途：高速缓冲存储器（Cache）。
    • 动态随机存储器（DRAM）
      • 定义：利用栅极电容电荷存储信息的随机存储器，属于易失性存储器，主存主要组成部分。
      • 工作原理：电容带电为1、不带电为0；电容电荷仅维持1~2ms，需定时刷新；读操作会破坏电荷（破坏性读出），需读后再生。
      • 核心技术：
        • 刷新方式：
          1. 集中刷新：刷新周期内用固定时间逐行再生，期间无法访存（有死区）。
          2. 分散刷新：工作周期分读/写和刷新两部分，无死区但存取周期变长。
          3. 异步刷新：刷新周期内每行仅刷新一次，死区分散（相邻行刷新间隔 = 刷新周期 / 行数）。
        • 地址引脚复用：行、列地址通过相同引脚分两次输入，减少引脚数（地址引脚数减半）。
        • 行、列数优化：满足“|行数 - 列数|最小”且“行数 ≤ 列数”，减少引脚数和刷新开销。
        • 行缓冲器：缓存指定行数据（列数×存储元位数，用SRAM实现），支持突发传输。
      • 性质/特点：集成度高；位价低；功耗小；存取速度比SRAM慢；需定时刷新和读后再生。
      • 主要用途：大容量主存系统；常见类型为SDRAM（同步DRAM，与系统时钟同步，支持突发传输）。
    • SRAM与DRAM的对比
      • 存储信息：SRAM用触发器，DRAM用电容。
      • 破坏性读出：SRAM非，DRAM是。
      • 刷新需求：SRAM不需要，DRAM需要。
      • 地址传送：SRAM同时送，DRAM分两次送（复用）。
      • 运行速度：SRAM快，DRAM慢。
      • 集成度：SRAM低，DRAM高。
      • 存储成本：SRAM高，DRAM低。
      • 主要用途：SRAM用于Cache，DRAM用于主机内存。
    • 存储器芯片的内部结构
      • 结构/组成：包含存储体、I/O电路、地址译码器、控制电路。
      • 各组件功能：
        • 存储体（存储矩阵）：存储单元集合，由行（X）、列（Y）选择线选通单元，多位（位平面数）同时读写。
        • 地址译码器：将地址转换为译码输出线高电平，分单译码法（一维，输出线多）和双译码法（二维，X+Y译码，DRAM常用）。
        • I/O电路：控制选中单元的读/写，放大信号。
        • 控制电路：含片选控制线（CS‾\overline{CS}CS，选中特定芯片）和读/写控制线（WE‾\overline{WE}WE，控制读/写操作）。

  • 只读存储器（ROM）

    • ROM的特点
      • 定义：支持随机访问，信息一旦写入（或出厂固化），断电不丢失的存储器。
      • 性质/特点：
        • 结构简单，位密度高于RAM。
        • 非易失性，可靠性高。
        • 与RAM区别：RAM易失、可读可写；ROM非易失，写入方式/速度与RAM不同。
    • ROM的类型
      • 掩模式只读存储器（MROM）
        • 定义：由半导体厂商在生产过程中按用户需求直接写入内容，不可修改的ROM。
        • 性质/特点：可靠性高、集成度高、价格便宜；灵活性差，仅适用于批量生产。
      • 一次可编程只读存储器（PROM）
        • 定义：用户可通过编程器一次性写入程序，写入后不可修改的ROM。
        • 性质/特点：灵活性优于MROM；仅支持一次编程，错误写入后芯片报废。
      • 可擦除可编程只读存储器（EPROM）
        • 定义：用户可多次擦除（通常用紫外线）并重新编程的ROM。
        • 性质/特点：可多次改写；但编程次数有限、写入时间长，不能取代RAM。
      • Flash存储器（闪存）
        • 定义：基于EPROM发展，可在线快速擦除与重写，兼具ROM非易失性和RAM读写灵活性的存储器。
        • 性质/特点：
          • 保留Flash长期存信息、快速擦写特性。
          • 集成度高、价格便宜（同EPROM），电可擦写（同EEPROM），擦写速度快。
      • 固态硬盘（SSD，Solid State Drive）
        • 定义：由控制单元和Flash存储单元组成，基于闪存技术的“硬盘”。
        • 性质/特点：保留Flash的非易失性、快速擦写特性；读/写速度快、低功耗；价格高于传统磁盘。

  • 主存储器的基本组成

    • 结构/组成：核心为存储矩阵（存储体），配合地址寄存器（MAR）、数据寄存器（MDR）、地址译码器、读/写控制电路。
    • 关键组件功能：
      • 存储矩阵：由记忆单元（存储0/1的物理器件）构成，是存储器核心；编址单位可为字节（现代常用）或字。
      • 地址寄存器（MAR）：存放CPU访问的主存单元地址，位数与地址线位数相同，决定最大寻址范围（如36位MAR对应寻址范围0∼236−10\sim2^{36}-10∼236−1）。
      • 数据寄存器（MDR）：存放主存读出/写入的数据，位数与数据线位数相同（通常等于存储字长）。
      • 地址译码器：将MAR中的地址转换为存储矩阵的行/列选择信号，选中目标单元。
      • 读/写控制电路：接收CPU的读/写控制信号，控制数据在MDR与存储单元间传输。
    • 访存过程（以读操作为例）：
      1. CPU将访存地址送入MAR，通过地址线传至主存地址寄存器。
      2. CPU将读控制信号通过控制线传至主存读/写控制电路。
      3. 地址译码器译码，选中主存目标单元。
      4. 主存将目标单元内容通过数据线送至MDR。
      5. MDR将数据送至CPU。

  • 多模块存储器

    • 定义：采用空间并行技术，通过多个结构相同的存储模块并行工作，提高存储器吞吐率的存储系统。
    • 分类及特点：
      • 单体多字存储器
        • 定义：每个存储单元存储m个字，总线宽度为m个字，一次并行读出m个字的存储器。
        • 工作原理：一个存取周期内取出m条指令，逐条送至CPU（每隔1/m1/m1/m存取周期送一条）。
        • 优点：提高单体存储器速度，充分利用CPU资源。
        • 缺点：仅当指令/数据连续存放时有效；遇转移指令或非连续数据时，提速效果差。
      • 多体并行存储器
        • 定义：由多个独立模块（相同容量、速度，独立读/写控制电路、地址/数据寄存器）组成，支持并行或交叉工作的存储器。
        • 编址方式：
          1. 高位交叉编址（顺序方式）：
            • 定义：高位地址表示模块号（体号），低位地址表示模块内地址。
            • 工作原理：连续访存时先访问一个模块，完成后转下一个，模块串行工作。
            • 缺点：无法提高吞吐率，仍为顺序存储器。
          2. 低位交叉编址（交叉方式）：
            • 定义：低位地址表示模块号（体号=单元地址%模块数m），高位地址表示模块内地址。
            • 工作原理：程序连续存于相邻模块，支持轮流启动或同时启动。
            • 启动方式：
              • 轮流启动：每隔1/m1/m1/m存取周期启动一个模块，连续存取m个字耗时T+(m−1)τT + (m-1)\tauT+(m−1)τ（T为模块存取周期，τ\tauτ为总线周期），速度提高m倍；需满足“模块数≥m”，避免访存冲突。
              • 同时启动：所有模块并行读写，总位数等于数据总线位数（如4个16位模块并行提供64位数据）。
            • 优点：大幅提高存储器带宽，减少访存冲突（理想情况下每隔1/m1/m1/m周期读写一个数据）。
            • 访存冲突：相邻m次访问地址在同一模块时发生，需延迟冲突请求。

主存储器与CPU的连接

  • 连接原理

    • 定义：主存通过数据总线、地址总线、控制总线与CPU连接，实现地址传输、数据交换和控制信号交互。
    • 总线功能：
      • 数据总线：传输CPU与主存间的数据，位数×工作频率正比于数据传输速率。
      • 地址总线：传输CPU访问主存的地址，位数决定最大可寻址内存空间（如32位地址总线对应2322^{32}232B=4GB空间）。
      • 控制总线：传输读/写控制信号（如RD‾\overline{RD}RD、WE‾\overline{WE}WE），指示总线周期类型和I/O操作完成时刻。
    • 核心逻辑：单个芯片容量有限，需通过芯片扩展组成主存空间，再通过总线与CPU相连。

  • 主存容量的扩展

    • 位扩展法
      • 定义：当CPU数据线数多于存储芯片数据线数时，通过扩展存储字长，使芯片数据位数与CPU数据线数匹配的扩展方式。
      • 连接方式：
        • 各芯片地址线、片选控制线（CS‾\overline{CS}CS）、读/写控制线（WE‾\overline{WE}WE）与系统总线对应并联。
        • 各芯片数据线单独引出，分别连接系统数据线的不同位。
      • 示例：用8片8K×1位RAM芯片组成8K×8位存储器，8片芯片的地址线、CS‾\overline{CS}CS、WE‾\overline{WE}WE共接，数据线分别接CPU数据线D0~D7。
    • 字扩展法
      • 定义：当系统地址线数多于存储芯片地址线数时，通过扩展存储字数量，使存储字数满足系统需求的扩展方式（字长不变）。
      • 连接方式：
        • 各芯片地址线（低位）与系统地址线低位对应相连。
        • 各芯片数据线、读/写控制线与系统总线对应并联。
        • 系统地址线高位经译码器生成各芯片的片选信号。
      • 示例：用4片16K×8位RAM芯片组成64K×8位存储器，地址线A15~A14经译码器生成4个片选信号，分别控制4片芯片（A15A14=00选第一片，01选第二片，依此类推）。
    • 字位同时扩展法
      • 定义：同时扩展存储字数量和字长，解决芯片在字数和字长两方面均不满足系统需求的情况。
      • 连接方式：
        • 先将n片芯片按“位扩展”组成一组（字长满足系统需求）。
        • 将m组芯片按“字扩展”组成总存储器（字数满足系统需求），系统地址线高位译码生成各组的片选信号。
      • 示例：用8片16K×4位RAM芯片组成64K×8位存储器，每2片组成16K×8位的组（位扩展），4组按字扩展组成64K×8位存储器（A15~A14译码控制组选）。

  • 存储芯片的地址分配和片选

    • 核心逻辑：CPU访存需两步——片选（选中目标芯片/组）和字选（选中芯片内目标单元）；字选由芯片内地址线完成，片选由系统地址线高位生成。
    • 片选信号生成方法：
      • 线选法
        • 定义：用除片内寻址外的高位地址线直接连接芯片片选端，某地址线为“0”时选中对应芯片（每次仅1位有效）。
        • 优点：无需译码器，线路简单。
        • 缺点：地址空间不连续；地址线需分时为低电平；地址资源浪费。
        • 示例：4片2K×8位芯片组成8K×8位存储器，低位A10~A0字选，高位A14~A11分别接4片芯片CS‾\overline{CS}CS（A14=0选0#芯片，A13=0选1#芯片，依此类推）。
      • 译码片选法
        • 定义：用除片内寻址外的高位地址线经地址译码器生成片选信号，支持多芯片/组的灵活选中。
        • 优点：地址空间连续；充分利用地址资源；支持大容量扩展。
        • 示例：8片8K×8位芯片组成64K×8位存储器（16位地址线），片内A12~A0字选，高位A15~A13经74LS138译码器生成8个片选信号（A15A14A13=000选第一片，001选第二片，依此类推）。

  • 存储器与CPU的连接步骤

    1. 选择存储芯片：根据需求选ROM（存系统程序、常数）或RAM（用户编程），确定芯片类型和数量（尽量简化连线）。
    2. 地址线连接：CPU地址线低位与芯片地址线相连（字选），高位用于片选（译码或线选）。
    3. 数据线连接：CPU与芯片数据线位数匹配时直接相连；不匹配时进行位扩展。
    4. 读/写控制线连接：CPU读/写控制线（如RD‾\overline{RD}RD、WE‾\overline{WE}WE）直接接芯片对应控制端（高电平读、低电平写，或低电平有效）。
    5. 片选控制线连接：结合线选法或译码片选法，将片选信号（需结合CPU访存控制信号MREQ‾\overline{MREQ}MREQ​，仅访存时有效）接芯片CS‾\overline{CS}CS端。

外部存储器

  • 磁盘存储器

    • 定义：以磁盘为存储介质，通过磁头读/写磁道信息的外部存储器，具有大容量、低位价、非易失性特点。
    • 结构/组成：
      • 硬件组成：磁盘驱动器（驱动磁盘转动、磁头读/写）、磁盘控制器（主机与驱动器接口，解释命令、控制信号）、盘片（存储介质）。
      • 存储区域：
        • 记录面：每个盘片有2个记录面（上下），一个记录面对应一个磁头（磁头数=记录面数）。
        • 磁道：记录面上的同心圆，不同记录面相同编号的磁道构成柱面（柱面数=每面磁道数）。
        • 扇区：磁道上的扇形区域，是读/写最小单位（按块存取），相邻磁道/扇区间有间隙（避免精度错误）。
      • 磁盘高速缓存：内存中开辟的缓冲区域，减少小块数据频繁写盘，提高数据复用率。
    • 磁记录原理：磁头与磁盘相对运动时，通过电磁转换实现读/写；编码方法将二进制数据转换为磁化翻转序列，常用调频制（FM）、改进型调频制（MFM）。
    • 性能指标：
      • 记录密度：
        • 道密度：沿半径方向单位长度磁道数。
        • 位密度：磁道单位长度记录的二进制位数（从外道到内道增加）。
        • 面密度：位密度×道密度。
      • 存储容量：
        • 非格式化容量：磁记录表面可利用的磁化单元总数 = 记录面数×柱面数×每磁道磁化单元数。
        • 格式化容量：按特定格式存储的信息总量 = 记录面数×柱面数×每道扇区数×每扇区容量（小于非格式化容量）。
      • 存取时间：T存取=T寻道+T旋转延迟+T传输T_{存取} = T_{寻道} + T_{旋转延迟} + T_{传输}T存取​=T寻道​+T旋转延迟​+T传输​。
        • 寻道时间：磁头移动到目标磁道的时间，通常取平均寻道时间（最外道到最内道时间的一半）。
        • 旋转延迟时间：磁头定位到目标扇区的时间，通常取平均旋转延迟（旋转半周的时间，如7200转/分磁盘的平均旋转延迟≈4.17ms）。
        • 传输时间：传输数据的时间 = 数据量 / 数据传输速率。
      • 数据传输速率：单位时间向主机传送的字节数，公式为D=r×ND = r \times ND=r×N（r为磁盘转数/秒，N为每磁道容量）。
    • 磁盘地址：由“柱面号（磁道号）、磁头号（盘面号）、扇区号”组成，标识目标扇区。
    • 工作过程：核心操作是寻址、读盘、写盘；读/写串行，同一时刻仅能进行一项读/写操作。

  • 磁盘阵列（RAID）

    • 定义：将多个独立物理磁盘组成逻辑盘，通过数据交叉存储、并行访问，提升存储性能、可靠性和安全性的技术（RAID：Redundant Array of Independent Disks）。
    • 核心特性：
      • 并行访问：多磁盘同时读/写，提高传输速率。
      • 冗余/校验：通过镜像、奇偶校验等实现容错，磁盘损坏时可恢复数据。
    • RAID分级：
      • RAID 0（无冗余）：数据条带化存储（连续数据块交替存于不同磁盘），无容错；容量=磁盘数×单盘容量，速度快但无可靠性。
      • RAID 1（镜像）：两磁盘同时读/写，互为备份；容量=单盘容量，可靠性高但容量利用率低（50%）。
      • RAID 2（海明码纠错）：用海明码实现纠错，需多磁盘存储校验信息，成本高，少用。
      • RAID 3（位交叉奇偶校验）：数据位交叉存储，单磁盘存奇偶校验信息；读/写快，校验盘故障不影响数据，数据盘故障可恢复。
      • RAID 4（块交叉奇偶校验）：数据块交叉存储，单磁盘存奇偶校验信息；独立访问各磁盘，校验盘易成为瓶颈。
      • RAID 5（无独立校验）：奇偶校验信息分散存于各磁盘，无独立校验盘；可靠性高，性能均衡，应用广泛。

  • 固态硬盘（SSD）

    • 定义：基于闪存技术，由控制单元（闪存翻译层）和存储单元（Flash芯片）组成的外部存储器，无机械部件。
    • 结构/组成：
      • 闪存翻译层：将CPU的逻辑块读/写请求转换为底层物理设备控制信号，替代磁盘控制器功能。
      • Flash芯片：由多个块组成，每块含多个页（页：读/写单位，512B~4KB；块：擦除单位，16KB~512KB）；需先擦除块才能重写页。
    • 性质/特点：
      • 优点：随机访问速度快（无机械延迟）；无噪声、低功耗、抗震性好、安全性高。
      • 缺点：随机写速度慢（需擦除块，擦除速度比读慢一个数量级）；闪存擦写寿命有限（几百~几千次）。
    • 核心技术：磨损均衡（Wear Leveling）
      • 定义：通过算法使各闪存块擦写次数均匀，延长SSD寿命的技术。
      • 分类：
        • 动态磨损均衡：优先选择擦除次数少的新块写入，减少老块损耗。
        • 静态磨损均衡：自动调整数据分配，让老块承担读为主的任务，新块承担写为主的任务，平衡各块寿命。
    • 应用场景：替代传统机械硬盘，适用于对速度、抗震性要求高的场景（如笔记本电脑、服务器）。

高速缓冲存储器（Cache）

  • 程序访问的局部性原理

    • 定义：程序执行时，对存储器的访问具有局部集中的特性，是Cache技术的核心依据。
    • 分类：
      • 时间局部性：最近未来要用到的信息，大概率是当前正在使用的信息（如循环指令、重复访问的变量）。
      • 空间局部性：最近未来要用到的信息，大概率与当前使用信息在存储空间上邻近（如顺序执行的指令、数组的连续元素）。
    • 应用示例：
      • 数组按行优先存储时，行遍历（程序A）的空间局部性优于列遍历（程序B），因行遍历访问顺序与存储顺序一致，列遍历需跳过大量元素。
      • 循环体指令具有良好的时间局部性（反复执行）和空间局部性（指令连续存放）。

  • Cache的基本工作原理

    • 定义：在CPU与主存间设置的高速小容量存储器，存放主存中最活跃的块副本，通过局部性原理减少CPU访存时间。
    • 结构/组成：包含Cache存储体、地址映射机构、替换机构、读/写控制电路。
    • 核心概念：
      • 块（Block）：Cache和主存的基本交换单位，Cache块也称Cache行（Line），块长（行长）为固定字节数（如64B）。
      • 命中（Hit）：CPU访存地址在Cache中，直接访问Cache。
      • 缺失（Miss）：CPU访存地址不在Cache中，需从主存调入对应块到Cache（若Cache满则替换）。
    • 访存过程（读操作）：
      1. CPU发出读请求，地址映射机构判断是否命中。
      2. 命中：将主存地址转换为Cache地址，读Cache并送CPU。
      3. 缺失：访问主存，将该地址所在块调入Cache（需替换时执行替换算法），同时送CPU。
    • 命中率（H）：CPU访存命中次数占总访存次数的比例，公式为H=Nc/(Nc+Nm)H = N_c / (N_c + N_m)H=Nc​/(Nc​+Nm​)（NcN_cNc​为命中次数，NmN_mNm​为缺失次数），越接近1越好。
    • 平均访问时间（TaT_aTa​）：Ta=H×tc+(1−H)×tmT_a = H \times t_c + (1-H) \times t_mTa​=H×tc​+(1−H)×tm​（tct_ctc​为Cache访问时间，tmt_mtm​为缺失时访存时间）。

  • Cache和主存的映射方式

    • 核心逻辑：主存块需按规则映射到Cache行，通过标记位（Tag）标识主存块来源，有效位（Valid）标识Cache行数据是否有效。
    • 直接映射（Direct Mapping）
      • 定义：主存每一块仅能映射到Cache的唯一行，映射关系为“Cache行号 = 主存块号 % Cache总行数”。
      • 地址结构：主存地址分为“标记（Tag）+ Cache行号 + 块内地址”，标记位数 = 主存地址位数 - 行号位数 - 块内地址位数。
      • 访存过程：
        1. 由主存地址中间的行号找到对应Cache行。
        2. 比较Cache行标记与主存地址标记，若相等且有效位为1，则命中，按块内地址存取。
        3. 不命中：调入主存块到对应Cache行，设置标记和有效位。
      • 优点：实现简单，判断速度快，硬件成本低。
      • 缺点：块冲突概率高（不同主存块映射到同一行），空间利用率低。
    • 全相联映射（Fully Associative Mapping）
      • 定义：主存每一块可映射到Cache的任意行，无固定映射关系。
      • 地址结构：主存地址分为“标记（Tag）+ 块内地址”，标记位数 = 主存地址位数 - 块内地址位数。
      • 访存过程：
        1. 将主存地址标记与所有Cache行标记并行比较。
        2. 若某行标记相等且有效位为1，则命中，按块内地址存取。
        3. 不命中：调入主存块到任意空闲行，设置标记和有效位。
      • 优点：块冲突概率低，空间利用率高，命中率高。
      • 缺点：标记比较速度慢（需多比较器），硬件成本高，不适用于大容量Cache。
    • 组相联映射（Set-Associative Mapping）
      • 定义：将Cache分为Q个组，主存块映射到固定组的任意行（组间直接映射，组内全相联映射），r路组相联表示每组有r行。
      • 映射关系：“Cache组号 = 主存块号 % Cache组数（Q）”。
      • 地址结构：主存地址分为“标记（Tag）+ 组号 + 块内地址”，标记位数 = 主存地址位数 - 组号位数 - 块内地址位数。
      • 访存过程：
        1. 由主存地址中间的组号找到对应Cache组。
        2. 比较该组内所有行的标记与主存地址标记，若相等且有效位为1，则命中。
        3. 不命中：调入主存块到该组任意空闲行，设置标记和有效位。
      • 优点：平衡直接映射（速度）和全相联映射（命中率），成本适中，应用广泛。
      • 关键参数：r（路数）越大，块冲突概率越低，但比较器复杂度越高。

  • Cache中主存块的替换算法

    • 核心逻辑：全相联或组相联映射下，Cache/组满时需替换旧块，直接映射无需替换算法（固定替换）。
    • 分类及特点：
      • 随机算法（RAND）
        • 定义：随机选择Cache行替换。
        • 优点：实现简单。
        • 缺点：未利用局部性原理，命中率低。
      • 先进先出算法（FIFO）
        • 定义：选择最早调入的Cache行替换。
        • 优点：实现简单（记录调入时间戳）。
        • 缺点：未利用局部性原理，最早调入的块可能仍常用（如循环块）。
      • 近期最少使用算法（LRU）
        • 定义：选择近期内最久未访问的Cache行替换，符合局部性原理。
        • 实现方式：为每个Cache行设置计数器（LRU位），命中时计数器清零，其他计数器加1；替换时选择计数器最大的行。
        • 优点：平均命中率高于FIFO，是堆栈类算法（Cache容量越大命中率越高）。
        • 缺点：实现复杂度随路数增加而提高（如4路组相联需2位LRU位）。
      • 最不经常使用算法（LFU）
        • 定义：选择一段时间内访问次数最少的Cache行替换。
        • 实现方式：为每个Cache行设置访问计数器，访问时加1，替换时选择计数器最小的行。
        • 优点：考虑访问频率。
        • 缺点：未考虑近期访问情况，可能替换早期高频但近期未用的块。

  • Cache的写策略（一致性维护）

    • 核心问题：Cache命中时，需保证Cache与主存数据一致；Cache不命中时，需决定是否调入主存块。
    • 写命中策略：
      • 全写法（直写法，Write Through）
        • 定义：CPU写Cache时，同时写主存，确保Cache与主存始终一致。
        • 优点：实现简单，数据一致性高。
        • 缺点：增加访存次数，降低效率；优化措施：加写缓冲（FIFO队列），CPU写Cache和写缓冲，写缓冲再写主存，缓解速度不匹配。
        • 替换处理：替换时无需写主存（主存已同步）。
      • 回写法（Write Back）
        • 定义：CPU写Cache时，仅写Cache，不立即写主存；仅当该块被替换时，若修改位（脏位，Dirty）为1才写主存。
        • 优点：减少访存次数，效率高。
        • 缺点：存在数据不一致隐患；需设置修改位（1表示修改过，0表示未修改）。
        • 替换处理：修改位为1则写主存，为0则直接覆盖。
    • 写不命中策略：
      • 写分配法（Write Allocate）
        • 定义：写不命中时，先将主存块调入Cache，再写Cache。
        • 优点：利用空间局部性，后续写操作可命中。
        • 缺点：每次写不命中需调入块，增加开销；通常与回写法合用。
      • 非写分配法（No-Write-Allocate）
        • 定义：写不命中时，仅写主存，不调入Cache。
        • 优点：避免无用块调入，减少开销；通常与全写法合用。
    • 分离Cache（指令Cache与数据Cache）
      • 定义：将指令和数据的Cache分开设计，避免指令预取与数据存取的冲突。
      • 优点：充分利用指令（顺序访问）和数据（随机访问）的不同局部性，提升流水线效率。
      • 应用：现代CPU的L1 Cache通常为分离Cache，L2/L3 Cache为统一Cache。

虚拟存储器

  • 虚拟存储器的基本概念

    • 定义：由主存和辅存组成的逻辑存储器，通过硬件+操作系统实现地址映射，为程序员提供远大于主存的虚拟地址空间，解决主存容量不足问题。
    • 核心概念：
      • 虚拟地址（逻辑地址）：程序员编程使用的地址，空间远大于主存物理地址空间。
      • 物理地址（实地址）：主存实际单元地址，空间等于主存容量。
      • 地址映射：将虚拟地址转换为物理地址的过程，由MMU（内存管理单元）和操作系统完成。
      • 缺页（缺段）：虚拟地址对应的块/段不在主存中，需从辅存调入（操作系统处理）。
    • 工作原理：
      1. CPU发出虚拟地址，MMU判断该地址对应块是否在主存。
      2. 若在主存（命中）：地址映射为物理地址，访问主存。
      3. 若不在主存（缺页）：操作系统将该块从辅存调入主存（需替换时执行替换算法），更新地址映射表，重新执行访存。
    • 关键特性：
      • 透明性：对应用程序员透明（无需关注实际主存容量），对系统程序员不透明（需实现地址映射）。
      • 映射策略：采用全相联映射（主存块可放任意页框），提高命中率（缺页代价大）。
      • 写策略：采用回写法（避免频繁写辅存，辅存速度远慢于主存）。

  • 页式虚拟存储器

    • 定义：以“页（Page）”为基本交换单位的虚拟存储器，主存页称页框（Page Frame），虚拟页称虚页。
    • 核心组件：页表（Page Table）
      • 定义：存放虚页与页框映射关系的表格，常驻主存，每个进程有独立页表。
      • 页表项（PTE）结构：包含有效位（装入位，1表示虚页在主存）、物理页框号（虚页在主存的位置）、脏位（1表示虚页被修改）、引用位（记录访问情况，用于替换算法）、磁盘地址（虚页在辅存的位置，缺页时使用）。
    • 地址转换过程：
      1. 虚拟地址分为“虚页号 + 页内地址”（页内地址与物理页内地址相同，因页大小一致）。
      2. 由页表基址寄存器（存放页表首地址）和虚页号，找到对应的页表项。
      3. 若有效位为1：物理地址 = 物理页框号 + 页内地址，访问主存。
      4. 若有效位为0：触发缺页异常，操作系统调入该页到主存，更新页表项，重新执行。
    • 快表（TLB，Translation Lookaside Buffer）
      • 定义：存放页表常用项的高速缓冲存储器（SRAM实现），解决页表访问主存的开销。
      • 工作原理：地址转换时先查TLB，命中则直接获取物理页框号；未命中则查主存页表，将该页表项调入TLB（TLB满则替换）。
      • 映射方式：通常采用全相联或组相联映射（命中率高），TLB标记为虚页号（或高位）。
    • 优点：页大小固定，页表简单，调入方便。
    • 缺点：存在页内碎片（最后一页未用完的空间）；页不是逻辑独立单元，不利于共享和保护。

  • 段式虚拟存储器

    • 定义：以“段（Segment）”为基本交换单位的虚拟存储器，段按程序逻辑结构划分（如函数、数组），长度可变。
    • 核心组件：段表（Segment Table）
      • 定义：存放段与主存映射关系的表格，每个段对应一个段表项。
      • 段表项结构：包含装入位（1表示段在主存）、段基址（段在主存的起始地址）、段长（段的长度，用于地址越界检查）、磁盘地址（段在辅存的位置）。
    • 地址转换过程：
      1. 虚拟地址分为“段号 + 段内地址”。
      2. 由段表基址寄存器和段号，找到对应的段表项。
      3. 检查段内地址是否超过段长（越界检查），若超过则触发越界异常。
      4. 若装入位为1：物理地址 = 段基址 + 段内地址，访问主存。
      5. 若装入位为0：触发缺段异常，操作系统调入该段到主存，更新段表项。
    • 优点：段是逻辑独立单元，利于编译、修改、保护和共享（如共享代码段）；无页内碎片。
    • 缺点：段长度可变，分配主存困难，易产生段间碎片；段表复杂，地址转换速度慢。

  • 段页式虚拟存储器

    • 定义：结合页式和段式的优点，先按逻辑分段，每段再划分为固定大小的页，主存按页划分，以页为交换单位。
    • 结构/组成：每个程序对应一个段表，每段对应一个页表（段表项存放页表首地址）。
    • 地址转换过程：
      1. 虚拟地址分为“段号 + 段内页号 + 页内地址”。
      2. 由段表基址和段号找到段表项，获取该段的页表首地址。
      3. 由段内页号和页表首地址找到页表项，获取物理页框号。
      4. 物理地址 = 物理页框号 + 页内地址，访问主存（缺段/缺页时操作系统处理）。
    • 优点：兼具段式的逻辑独立性（共享、保护）和页式的分配灵活性（无段间碎片）。
    • 缺点：地址转换需两次查表（段表、页表），系统开销大；硬件复杂度高。

  • 虚拟存储器与Cache的比较

    • 相同之处：
      1. 目标：提高系统性能，均有“容量、速度、价格”梯度。
      2. 数据单位：均划分小信息块（Cache块、虚页/段）作为交换单位。
      3. 核心技术：均涉及地址映射、替换算法、一致性维护。
      4. 依据：均利用局部性原理，将活跃数据放在高速部件。
    • 不同之处：
      1. 解决问题：Cache解决CPU与主存速度不匹配；虚拟存储器解决主存容量不足。
      2. 实现方式：Cache全由硬件实现，对所有程序员透明；虚拟存储器由硬件+操作系统实现，对系统程序员不透明。
      3. 不命中代价：Cache不命中代价小（访问主存，约10~100ns）；虚拟存储器不命中代价大（访问辅存，约1~10ms）。
      4. 访问通路：CPU可直接访问Cache和主存；CPU不能直接访问辅存，需经主存中转。

本章小结与常见问题

  • 本章小结

    • 存储器层次化的原因与管理
      • 原因：Cache-主存层解决速度不匹配（速度≈Cache，容量/位价≈主存）；主存-辅存层解决容量不足（容量/位价≈辅存，速度≈主存），平衡“大容量、高速度、低成本”矛盾。
      • 管理：Cache-主存数据调动由硬件自动完成；主存-辅存由硬件+操作系统完成（虚拟存储），程序员用虚拟地址编程。
    • 影响Cache性能的因素
      1. 映射方式：全相联命中率最高，直接映射最低。
      2. Cache容量：容量越大，命中率越高（边际效益递减）。
      3. 块大小：块大小适中（几十~上百字节），过小则空间局部性利用不足，过大则Cache行数减少、冲突概率增加。
      4. 其他：Cache级数（多级Cache提升命中率）、指令/数据Cache是否分离、总线架构等。
    • 虚拟存储页面大小的选择
      • 页面不宜过小：页内碎片小，但页表大，不能充分利用空间局部性。
      • 页面不宜过大：页表小，但页内碎片大，页面调入/调出时间长。
      • 结论：页面大小适中（常见4KB、8KB、16KB），平衡碎片、页表大小和调页效率。

  • 常见问题与易混淆知识点

    • Cache行大小与命中率的关系
      • 行长大：利于利用空间局部性，减少缺失次数；但失效损失大（未命中时读主存块时间长），Cache行数减少导致冲突概率增加。
      • 行长短：失效损失小，但空间局部性利用不足，命中率低。
      • 结论：行大小需权衡，通常为32B~128B。
    • 取指令Cache缺失的处理过程
      1. 程序计数器（PC）恢复当前指令地址（因缺失时PC已自增）。
      2. 向主存发送读请求，读取该指令所在的Cache块。
      3. 将读入的块写入Cache，更新有效位和标记位。
      4. 重新执行当前指令（PC指向当前指令地址）。
    • Cache总容量与映射方式的关系
      • Cache总容量 = （每个Cache行标记项容量 + Cache行长）× Cache总行数。
      • 标记项组成：
        • 必选：有效位（1位）、标记位（取决于映射方式）。
        • 可选：脏位（回写法需1位）、LRU位（LRU算法需log⁡2r\log_2 rlog2​r位，r为组内块数）。
      • 示例：直接映射Cache的标记位 = 主存地址位数 - 行号位数 - 块内地址位数；组相联标记位 = 主存地址位数 - 组号位数 - 块内地址位数。
