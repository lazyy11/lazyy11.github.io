第2章 数据的表示和运算

一、数制与编码

  • 1. 进位计数制及其相互转换

    • 核心基础：二进制编码的原因（2018年命题追踪）
      • 物理实现简单：仅需两个稳定状态的物理器件（如高低电平、电荷正负极），制造成本低。
      • 逻辑适配性强：二进制位“1”“0”与逻辑值“真”“假”直接对应，便于实现逻辑运算和程序逻辑判断。
      • 运算规则简洁：编码和运算规则简单，可通过逻辑门电路便捷实现算术运算。
    • 常用进位计数制
      • 定义：按“逢r进一”规则计数，r为基数，每个数位的权值为rir^iri（i为数位序号），数值表示为∑i=n−mKiri\sum_{i=n}^{-m} K_i r^i∑i=n−m​Ki​ri（KiK_iKi​为0~r-1的数码）。
      • 分类及特点：
        • 十进制（D）：基数10，日常使用，如123.45 = 1×10² + 2×10¹ + 3×10⁰ + 4×10⁻¹ + 5×10⁻²。
        • 二进制（B）：基数2，计算机内部通用，仅含0/1，权值为2i2^i2i，如(1101.1)₂ = 1×2³ + 1×2² + 0×2¹ + 1×2⁰ + 1×2⁻¹ = 13.5。
        • 八进制（O）：基数8，含0~7，1位八进制对应3位二进制，便于简化二进制表示。
        • 十六进制（H）：基数16，含0~9、A~F（A=10~F=15），1位十六进制对应4位二进制，如(3C2.68)₁₆对应二进制0011 1100 0010.0110 1000。
    • 不同进制的相互转换
      • 二进制 ↔ 八进制/十六进制：
        • 整数部分：从小数点左起，3位（八进制）/4位（十六进制）分组，高位补0；
        • 小数部分：从小数点右起，3位/4位分组，低位补0；
        • 每组对应1位目标进制数（例：二进制11110001011.101转换为八进制为2303.5，十六进制为3C3.8）。
      • 任意进制 → 十进制：按权展开相加法，如(1101.1)₂ = 1×2³ + 1×2² + 0×2¹ + 1×2⁰ + 1×2⁻¹ = 13.5。
      • 十进制 → 任意进制（2021、2022年命题追踪）：
        • 整数部分：除基取余法，“先余为低，后余为高”，商为0时结束；
        • 小数部分：乘基取整法，“先整为高，后整为低”，乘积为1.0或满足精度时结束；
        • 例：十进制123.6875转换为二进制为1111011.1011。
      • 注意：并非所有十进制小数都能精确表示为二进制小数（如0.3），但任意二进制小数可精确表示为十进制小数。

  • 2. 定点数的编码表示

    • 真值与机器数
      • 真值：带“+”“-”的实际数值，如+15、-8。
      • 机器数：符号数字化的数（0表正、1表负），含符号位和数值位，常用原码、补码、反码、移码表示。
    • 定点数的表示格式
      • 定义：约定小数点固定位置的数，分定点小数和定点整数。
      • 分类及结构：
        • 定点小数：纯小数，小数点在符号位后、数值最高位前，格式为“符号位.数值位”（如x0.x1x2...xnx_0.x_1x_2...x_nx0​.x1​x2​...xn​，x0x_0x0​为符号位）。
        • 定点整数：纯整数，小数点在数值最低位后，格式为“符号位数值位.”（如x0x1x2...xnx_0x_1x_2...x_nx0​x1​x2​...xn​，xnx_nxn​为最低有效位）。
    • 四种编码表示法
      • 原码
        • 定义：最高位为符号位，其余位为数值绝对值，公式为：
          • 整数：[x]原={x0≤x<2n2n−x−2n<x≤0[x]_{原} = \begin{cases} x & 0 \leq x < 2^n \\ 2^n - x & -2^n < x \leq 0 \end{cases}[x]原​={x2n−x​0≤x<2n−2n<x≤0​（字长n+1位）；
          • 例：字长8位，x₁=+1110 → [x₁]原=00001110，x₂=-1110 → [x₂]原=10001110。
        • 性质/特点：
          • 0的表示不唯一（+0：000…0，-0：100…0）；
          • 与真值对应直观，乘除运算简便，但加减运算复杂。
        • 表示范围：n+1位整数原码范围为−(2n−1)≤x≤2n−1-(2^n - 1) \leq x \leq 2^n - 1−(2n−1)≤x≤2n−1（关于原点对称）。
      • 补码（2010、2013、2014、2020、2022、2023年命题追踪）
        • 定义：正数补码=原码，负数补码=模-绝对值，统一公式为[x]补=2n+1+x(mod 2n+1)[x]_{补} = 2^{n+1} + x \quad (\text{mod } 2^{n+1})[x]补​=2n+1+x(mod 2n+1)（字长n+1位，−2n≤x<2n-2^n \leq x < 2^n−2n≤x<2n）；
        • 例：字长8位，x₁=+1010 → [x₁]补=00001010，x₂=-1010 → [x₂]补=11110110。
        • 特殊值表示：
          • [+0]补=[-0]补=000…0（唯一）；
          • [-1]补=111…1（n+1个1）；
          • [2n−1]补=011...1[2^n - 1]_{补}=011...1[2n−1]补​=011...1（n个1，最大正数）；
          • [−2n]补=100...0[-2^n]_{补}=100...0[−2n]补​=100...0（n个0，最小负数）。
        • 性质/特点：
          • 加减运算统一为加法（[A−B]补=[A]补+[−B]补[A-B]_{补}=[A]_{补}+[-B]_{补}[A−B]补​=[A]补​+[−B]补​）；
          • 符号位与数值位共同参与运算，结果符号位自动生成；
          • 表示范围比原码大（多表示−2n-2^n−2n），n+1位整数补码范围为−2n≤x≤2n−1-2^n \leq x \leq 2^n - 1−2n≤x≤2n−1。
        • 真值与补码转换：
          • 真值→补码：正数同原码；负数符号位为1，数值位“按位取反，末位加1”。
          • 补码→真值：符号位为0同原码；符号位为1，数值位“按位取反，末位加1”，结果加负号。
        • 变形补码（模4补码）：双符号位（左符为真符号，右符判溢出），00表正、11表负，溢出判断为双符号位异或（1表溢出）。
      • 反码（了解即可）
        • 定义：正数反码=原码；负数反码=符号位为1，数值位“按位取反”（无末位加1）；
        • 性质/特点：0的表示不唯一（+0：000…0，-0：111…1）；表示范围比补码小（少表示−2n-2^n−2n），仅为补码转换中间形式。
      • 移码（常用于浮点数阶码）
        • 定义：真值加偏置值（通常为2n2^n2n，字长n+1位），公式为[x]移=2n+x(−2n<x<2n)[x]_{移}=2^n + x \quad (-2^n < x < 2^n)[x]移​=2n+x(−2n<x<2n)；
        • 例：字长8位（偏置值27=1282^7=12827=128），x₁=+10101 → [x₁]移=10010101，x₂=-10101 → [x₂]移=01101011。
        • 性质/特点：
          • 0的表示唯一（[+0]移=[−0]移=100...0[+0]_{移}=[-0]_{移}=100...0[+0]移​=[−0]移​=100...0）；
          • 与补码仅符号位相反（补码符号位取反得移码）；
          • 保持数值大小顺序（移码大→真值大）；
          • 全0对应最小真值−2n-2^n−2n，全1对应最大真值2n−12^n - 12n−1。
    • 四种编码对比：
      • 正数的原码、补码、反码相同；
      • 原码、反码表示对称（含±0），补码、移码表示不对称（0唯一，多表示1个数）；
      • 负数补码/反码需通过规则判断大小（数值位越小→绝对值越大）。

  • 3. 整数的表示

    • 无符号整数（2021年命题追踪）
      • 定义：无符号位，全部二进制位为数值位，默认正数，用于地址、指针等场景。
      • 表示范围：n位无符号整数范围为0≤x≤2n−10 \leq x \leq 2^n - 10≤x≤2n−1（例：8位→0~255）；
      • 对比有符号整数：字长相同时，无符号数表示范围更大（8位有符号补码范围-128~127）。
    • 有符号整数
      • 表示方式：计算机中统一用补码表示，因补码0唯一、运算简便、表示范围大。
      • 表示范围：n位有符号整数（含1位符号位）范围为−2n−1≤x≤2n−1−1-2^{n-1} \leq x \leq 2^{n-1} - 1−2n−1≤x≤2n−1−1（例：32位int→−231 231−1-2^{31}~2^{31}-1−231 231−1）。

  • 4. C语言中的整数类型及类型转换（2011、2016、2017、2019、2021、2024年命题追踪）

    • C语言整型类型
      • 分类及字长：
        • 有符号型：short（16位）、int（32位）、long（32/64位，依机器）；
        • 无符号型：unsigned short、unsigned int、unsigned long（在对应有符号类型前加unsigned）；
        • 字符型char（8位，默认无符号）；
        • 所有类型均按补码存储（无符号数视为正整数补码）。
    • 类型转换规则
      • 有符号数 ↔ 无符号数（同字长）：
        • 二进制位值不变，仅改变解释方式（例：short x=-4321（二进制1110111100011111）→ unsigned short y=61215，位值相同）；
        • 混合运算时，C语言默认按无符号数运算。
      • 不同字长整数转换：
        • 大字长→小字长：截断高位，保留低位（例：int x=165537（0x000286A1）→ short y=-31071（0x86A1））；
        • 小字长→大字长：
          • 无符号数：零扩展（高位补0，例：16位unsigned short→32位unsigned int，高16位补0）；
          • 有符号数：符号扩展（高位补原符号位，例：16位short x=-4321（0xEF1F）→32位int y=-4321（0xFFFFEF1F））。

二、运算方法和运算电路

  • 1. 基本运算部件

    • 一位全加器（FA）
      • 定义：实现1位二进制数加法的基本单元，含3个输入（加数AiA_iAi​、加数BiB_iBi​、低位进位Ci−1C_{i-1}Ci−1​）和2个输出（本位和SiS_iSi​、高位进位CiC_iCi​）。
      • 逻辑表达式：
        • 和：Si=Ai⊕Bi⊕Ci−1S_i = A_i \oplus B_i \oplus C_{i-1}Si​=Ai​⊕Bi​⊕Ci−1​（奇数个1时Si=1S_i=1Si​=1）；
        • 进位：Ci=AiBi+(Ai⊕Bi)Ci−1C_i = A_i B_i + (A_i \oplus B_i) C_{i-1}Ci​=Ai​Bi​+(Ai​⊕Bi​)Ci−1​。
    • 串行进位加法器
      • 结构：n个全加器串联，低位进位Ci−1C_{i-1}Ci−1​直接传入高位，进位信号逐级传递。
      • 特点：结构简单，但运算速度慢（延迟由进位传递时间决定，位数越多延迟越长）；
      • 功能：实现n位二进制数加法，结果为S=Sn...S1S=S_n...S_1S=Sn​...S1​，进位CnC_nCn​（模2n2^n2n运算，高位自动丢失）。
    • 并行进位加法器（先行进位CLA）
      • 结构：n个全加器+先行进位部件，并行产生所有进位信号（C1 CnC_1~C_nC1​ Cn​同时生成）。
      • 特点：运算速度快（消除进位延迟），但电路结构复杂（位数越多越复杂）；
      • 示例：4位全先行进位加法器，通过CLA部件同时生成C1 C4C_1~C_4C1​ C4​，提升加法效率。
    • 带标志加法器
      • 定义：在n位加法器基础上增加逻辑电路，生成运算标志位，用于判断运算结果状态。
      • 标志位及生成：
        • 溢出标志OF：OF=Cn⊕Cn−1OF = C_n \oplus C_{n-1}OF=Cn​⊕Cn−1​（1表有符号数溢出，0表无溢出）；
        • 符号标志SF：SF=Fn−1SF = F_{n-1}SF=Fn−1​（1表结果负，0表结果正，Fn−1F_{n-1}Fn−1​为结果最高位）；
        • 零标志ZF：ZF=1ZF=1ZF=1当且仅当结果F=0F=0F=0（1表结果为0，0表非0）；
        • 进位/借位标志CF：CF=Cout⊕CinCF = C_{out} \oplus C_{in}CF=Cout​⊕Cin​（1表无符号数溢出/借位，0表无溢出）。
    • 算术逻辑单元（ALU）（Arithmetic Logic Unit, ALU）
      • 定义：实现算术运算（加、减、乘、除）和逻辑运算（与、或、非、异或）的组合逻辑电路。
      • 结构：核心为带标志加法器，辅以移位器、多路选择器（MUX）；
        • 输入：操作数A/B（n位）、进位CinC_{in}Cin​、控制信号ALUop（决定运算类型）；
        • 输出：运算结果F、标志位（OF、SF、ZF、CF）。
      • 功能：通过ALUop控制实现不同运算（例：ALUop=Add时执行加法，ALUop=And时执行与运算）；支持移位操作。

  • 2. 定点数的移位运算（2012、2017、2018年命题追踪）

    • 核心作用：通过移位+加法实现乘除运算（左移1位≈×2，右移1位≈÷2，需结合类型判断）。
    • 逻辑移位（无符号数）
      • 规则：视为无符号数，左移→高位移出，低位补0（高位1移出则溢出）；右移→低位移出，高位补0；
      • 例：8位无符号数10101100左移1位→01011000（溢出，高位1丢失），右移1位→01010110。
    • 算术移位（有符号数，补码表示）
      • 规则：
        • 左移：高位移出，低位补0；若移出高位≠移位后符号位（左移前后符号位不同），则溢出；
        • 右移：低位移出，高位补符号位；若低位1移出，则损失精度；
      • 例：8位补码10010101（-107）左移1位→00101010（42，溢出，符号位从1变0），右移1位→11001010（-54）。

  • 3. 定点数的加减运算（2009、2011、2014、2017、2018、2021、2022、2023、2024年命题追踪）

    • 补码加减运算
      • 运算公式（字长n+1位）：
        • [A+B]补=[A]补+[B]补(mod 2n+1)[A+B]_{补} = [A]_{补} + [B]_{补} \quad (\text{mod } 2^{n+1})[A+B]补​=[A]补​+[B]补​(mod 2n+1)；
        • [A−B]补=[A]补+[−B]补(mod 2n+1)[A-B]_{补} = [A]_{补} + [-B]_{补} \quad (\text{mod } 2^{n+1})[A−B]补​=[A]补​+[−B]补​(mod 2n+1)（[−B]补[-B]_{补}[−B]补​为[B]补[B]_{补}[B]补​“按位取反，末位加1”）。
      • 特点：
        • 按二进制规则运算，逢二进一；
        • 符号位与数值位共同参与运算，结果符号位自动生成；
        • 高位丢弃，保留n+1位，结果为补码。
      • 示例：字长8位，A=15（00001111），B=24（00011000）；
        • [A+B]补=00001111+00011000=00100111[A+B]_{补}=00001111+00011000=00100111[A+B]补​=00001111+00011000=00100111（+39）；
        • [−B]补=11101000[-B]_{补}=11101000[−B]补​=11101000，[A−B]补=00001111+11101000=11110111[A-B]_{补}=00001111+11101000=11110111[A−B]补​=00001111+11101000=11110111（-9）。
    • 溢出判别方法（仅当同号相加/异号相减时可能溢出）
      • 方法1：单符号位判断：
        • 逻辑表达式：V=AsBsSs‾+AsBs‾SsV = A_s B_s \overline{S_s} + \overline{A_s B_s} S_sV=As​Bs​Ss​​+As​Bs​​Ss​（AsA_sAs​、BsB_sBs​为操作数符号位，SsS_sSs​为结果符号位）；
        • V=1V=1V=1表溢出，V=0V=0V=0表无溢出（同号操作数结果符号位不同则溢出）。
      • 方法2：双符号位判断（模4补码）：
        • 双符号位Ss1Ss2S_{s1}S_{s2}Ss1​Ss2​，00表正、11表负；
        • 逻辑表达式：V=Ss1⊕Ss2V = S_{s1} \oplus S_{s2}V=Ss1​⊕Ss2​（V=1V=1V=1表溢出，01正溢出、10负溢出）。
      • 方法3：进位判断：
        • 逻辑表达式：V=Cn⊕Cn−1V = C_n \oplus C_{n-1}V=Cn​⊕Cn−1​（CnC_nCn​为符号位进位，Cn−1C_{n-1}Cn−1​为最高数值位进位）；
        • V=1V=1V=1表溢出，V=0V=0V=0表无溢出（两进位不同则溢出）。
    • 加减运算电路
      • 结构：加法器+反向器+2选1多路选择器，控制端Sub（0表加法，1表减法）；
      • 原理：
        • 加法（Sub=0）：输入AAA、BBB，进位Cin=0C_{in}=0Cin​=0，实现A+BA+BA+B；
        • 减法（Sub=1）：输入AAA、B‾\overline{B}B（反向器取反），进位Cin=1C_{in}=1Cin​=1（末位加1），实现A+B‾+1=[A]补+[−B]补A+\overline{B}+1=[A]_{补}+[-B]_{补}A+B+1=[A]补​+[−B]补​；
      • 适配性：可同时实现无符号数和有符号数加减（无符号数视为正整数补码）。
    • 标志位作用（2024年命题追踪）
      • ZF：对无符号数和有符号数均有效，判断结果是否为0；
      • OF：仅对有符号数有效，判断是否溢出；
      • SF：仅对有符号数有效，判断结果正负；
      • CF：仅对无符号数有效，判断进位/借位（加法CF=1表溢出，减法CF=1表借位）。
    • 原码加减运算（了解）
      • 规则：符号位与数值位分开处理，“同号求和，异号求差”；
      • 加法：符号位相同→数值位相加，符号位不变（进位则溢出）；符号位不同→数值位相减（大减小），符号位随大数；
      • 减法：减数符号取反，按加法规则运算；
      • 缺点：规则复杂，计算机极少使用。

  • 4. 定点数的乘除运算（2019、2020、2021、2024年命题追踪）

    • 定点乘法运算
      • 基本原理（原码乘法）：
        • 符号位：两乘数符号位“异或”（同号为正，异号为负）；
        • 数值位：两乘数绝对值相乘（视为无符号数），通过“加法+右移”迭代实现；
        • 递推公式：Pi+1=2−1(Pi+X×yn−i)P_{i+1} = 2^{-1}(P_i + X \times y_{n-i})Pi+1​=2−1(Pi​+X×yn−i​)（P0=0P_0=0P0​=0，XXX为被乘数，yyy为乘数，共n次加法+右移）。
      • 乘法运算电路
        • 结构：被乘数寄存器X、乘数寄存器Y、乘积寄存器P（初值0）、进位位C、计数器、ALU、控制逻辑；
        • 流程：
          1. 计数器初值n，P初值0；
          2. 控制逻辑根据Y最低位（yiy_iyi​）决定是否将X加到P（yi=1y_i=1yi​=1则加X，yi=0y_i=0yi​=0则加0）；
          3. C、P、Y同步逻辑右移1位（Y最低位移出，C移入P最高位）；
          4. 计数器减1，重复步骤2~4直至计数器为0；
          5. 符号位由X、Y符号位异或生成，最终乘积为“符号位+P”。
      • 溢出判断：
        • 无符号数（n位）：乘积高n位全0则无溢出，否则溢出；
        • 有符号数（n位）：乘积高n位与低n位符号一致（全0或全1）则无溢出，否则溢出；
      • 阵列乘法器：快速乘法器，所有部分积同时生成并组成阵列，多操作数相加，1个时钟周期完成运算。
    • 定点除法运算
      • 基本原理（原码除法）：
        • 符号位：两数符号位“异或”；
        • 数值位：被除数绝对值÷除数绝对值（2n2n2n位被除数÷n位除数，得n位商+余数），通过“减法+左移”迭代实现；
        • 规则：被除数（中间余数）减除数，够减则上商1，不够减则上商0；每次余数左移1位，重复n次。
      • 除法运算电路
        • 结构：除数寄存器Y、余数寄存器R（初存被除数高n位）、商寄存器Q（初存被除数低n位）、ALU、计数器、控制逻辑；
        • 流程：
          1. 计数器初值n，R初存被除数高n位，Q初存被除数低n位；
          2. R、Q同步左移1位（Q最高位移入R最低位）；
          3. ALU计算R−YR-YR−Y，若结果非负（够减），则Q最低位置1，R存R−YR-YR−Y；否则Q最低位置0，R存原R；
          4. 计数器减1，重复步骤2~4直至计数器为0；
          5. 符号位由R、Y符号位异或生成，最终商为“符号位+Q”，余数为R。
      • 溢出判断：2n2n2n位÷n位时，第一次试商为1则溢出（商超n位）；n位÷n位时，第一次试商为0，无溢出。

三、浮点数的表示和运算

  • 1. 浮点数的表示（2014、2017、2018、2020、2022、2023、2024年命题追踪）

    • 基本格式：N=(−1)S×M×REN = (-1)^S \times M \times R^EN=(−1)S×M×RE，其中：
      • SSS：符号位（0正1负）；
      • MMM：尾数（二进制定点小数，通常原码表示，需规格化）；
      • EEE：阶码（二进制定点整数，通常移码表示）；
      • RRR：基数（隐含为2、4、16等，默认2）。
    • 表示范围：
      • 上溢：运算结果>最大正数或<最小负数，需中断处理；
      • 下溢：运算结果在0~最小正数或0~最大负数之间，视为机器零；
    • 规格化（提升精度，使尾数最高位为有效位）
      • 基数2的原码规格化：
        • 正数：0.1x...x0.1x...x0.1x...x（1/2≤M<11/2 \leq M < 11/2≤M<1）；
        • 负数：1.1x...x1.1x...x1.1x...x（−1<M≤−1/2-1 < M \leq -1/2−1<M≤−1/2）；
      • 操作：
        • 左规：尾数最高位非有效位（如0.0x...x0.0x...x0.0x...x），尾数左移1位，阶码减1（需多次）；
        • 右规：尾数有效位超小数点前（如1.x...x1.x...x1.x...x），尾数右移1位，阶码加1（仅1次）；
    • IEEE 754标准（计算机通用浮点数标准）
      • 格式分类：
        类型 符号位S（1位） 阶码E（位） 尾数f（位） 总位数 偏置值（十进制） 基数
        单精度（float） 1 8 23 32 127 2
        双精度（double） 1 11 52 64 1023 2
      • 核心特点：
        • 隐藏位：规格化数尾数最高位为1，存储时隐藏（23位尾数实际表示24位有效数字）；
        • 阶码表示：移码=阶码真值+偏置值（单精度：E=e+127，双精度：E=e+1023）；
        • 真值公式：
          • 单精度：N=(−1)S×(1.f)×2E−127N = (-1)^S \times (1.f) \times 2^{E-127}N=(−1)S×(1.f)×2E−127（E≠0且E≠255）；
          • 双精度：N=(−1)S×(1.f)×2E−1023N = (-1)^S \times (1.f) \times 2^{E-1023}N=(−1)S×(1.f)×2E−1023（E≠0且E≠2047）；
      • 特殊值解释：

        正/负无穷大 S=0/1, E=255, f=0 S=0/1, E=2047, f=0 +∞/-∞
        非数（NaN） S=0/1, E=255, f≠0 S=0/1, E=2047, f≠0 无定义数

      • 表示范围：
        • 单精度：最小规格化正数2−1262^{-126}2−126，最大规格化正数2127×(2−2−23)2^{127}×(2-2^{-23})2127×(2−2−23)；
        • 双精度：最小规格化正数2−10222^{-1022}2−1022，最大规格化正数21023×(2−2−52)2^{1023}×(2-2^{-52})21023×(2−2−52)；
    • 定点数与浮点数的区别：
      对比维度 定点数 浮点数
      表示范围 小（字长固定，范围固定） 大（阶码扩展范围）
      精度 高（全字长表数值） 低（部分字长表阶码，尾数精度降）
      运算复杂度 简单（仅数值运算） 复杂（阶码+尾数运算，需规格化）
      溢出判断 数值超范围则溢出 仅阶码超范围则溢出（尾数溢出可右规）

  • 2. 浮点数的加减运算（2009、2015、2017年命题追踪）

    • 核心步骤（阶码运算与尾数运算分开）
      • 步骤1：对阶（使两数小数点对齐）
        • 求阶差：ΔE=∣EA−EB∣\Delta E = |E_A - E_B|ΔE=∣EA​−EB​∣；
        • 小阶向大阶看齐：阶码小的数，尾数右移ΔE\Delta EΔE位，阶码加ΔE\Delta EΔE（尾数右移时保留低位，避免精度损失）；
        • 原因：大阶向小阶看齐会导致尾数左移，高位有效位丢失，精度严重下降。
      • 步骤2：尾数加减
        • 对阶后，尾数按定点原码小数加减规则运算（需还原隐藏位1）；
      • 步骤3：尾数规格化
        • 右规：尾数为1.x...x1.x...x1.x...x或0.x...x0.x...x0.x...x（超范围），尾数右移1位，阶码加1（仅1次）；
        • 左规：尾数为0.0x...x0.0x...x0.0x...x（非有效位），尾数左移k位，阶码减k（需多次，直至尾数最高位为1）；
      • 步骤4：舍入（对阶/右规时尾数右移，需保留精度）
        • 就近舍入：四舍五入，中间值（如1.245→1.24或1.25，选偶数）；
        • 正向舍入：朝+∞舍入（取右边最近数）；
        • 负向舍入：朝-∞舍入（取左边最近数）；
        • 截断法：直接截取，丢弃低位（趋向原点）；
      • 步骤5：溢出判断（仅阶码上溢为溢出）
        • 上溢：阶码>最大允许阶码（单精度E>254，双精度E>2046），中断处理；
        • 下溢：阶码<最小允许阶码（单精度E<1，双精度E<1），视为机器零；
    • 示例：单精度浮点数X=27×29/322^7×29/3227×29/32（00111 1100010），Y=25×5/82^5×5/825×5/8（00101 0010100）；
      • 对阶：E_X=7>E_Y=5，Y尾数右移2位→0000101，E_Y=7；
      • 尾数相加：1100010 + 0000101 = 1100111；
      • 规格化：尾数为1.100111，无需规格化；
      • 阶码：7，结果为27×1.1001112^7×1.10011127×1.100111（00111 1100111）。

  • 3. C语言中的浮点数类型（2010、2017年命题追踪）

    • 类型对应：
      • float→IEEE 754单精度；
      • double→IEEE 754双精度；
      • long double→扩展双精度（字长/格式依编译器/处理器）；
    • 类型转换规则（隐式转换，低精度→高精度）：
      • char→int→long→double；
      • float→double（float与double运算时，float先转double）；
    • 转换精度影响：
      • int→float：int（32位）转float（24位有效位），若int二进制超24位，需舍入，精度损失；
      • int/double→double：无精度损失（double有效位53位）；
      • double→float：可能溢出（范围小）或舍入（精度低）；
      • float/double→int：截断小数部分，可能溢出（int范围小）。

  • 4. 数据的大小端和对齐存储（2012、2016、2018、2019、2020、2023、2024年命题追踪）

    • 大小端存储（字节编址，多字节数据的字节排列方式）
      • 大端方式（big endian）：高位字节存低地址，低位字节存高地址（与数据原序列一致）；
        • 例：int x=0x01234567（地址0x8000），存储为0x8000:01H，0x8001:23H，0x8002:45H，0x8003:67H；
      • 小端方式（little endian）：低位字节存低地址，高位字节存高地址（与数据原序列相反）；
        • 例：同上x，存储为0x8000:67H，0x8001:45H，0x8002:23H，0x8003:01H；
      • 判断方法：通过代码读取多字节数据的低地址字节，若为数据低位则为小端，反之大端。
    • 边界对齐存储（空间换时间，提升访存效率）
      • 定义：数据存储地址需为自身字节长度的整数倍（char（1B）→地址任意，short（2B）→地址偶，int（4B）→地址4的倍数）；
      • 结构体对齐规则：
        1. 每个成员按自身类型对齐（如int成员地址为4的倍数）；
        2. 结构体总长度为成员最大对齐值的整数倍（不足补空字节）；
      • 示例：32位环境，struct A{char b; short c; int a;}；
        • b（1B）存0x0000，c（2B）存0x0002（补1B空），a（4B）存0x0004，总长度8B（最大对齐值4，8是4的倍数）；
      • 优势：单字长数据1次访存完成（无需拆分），适配RISC指令流水。

四、本章小结与常见问题

  • 1. 开篇问题参考答案

    • Q1：计算机为何用二进制表示数据？
      • 物理实现简单（两稳定状态器件）；
      • 适配逻辑运算（1/0对应真/假）；
      • 运算规则简洁（易通过逻辑门实现）。
    • Q2：字长足够时计算机能否精确表示所有数？
      • 整数：可精确表示（二进制可表示任意整数）；
      • 小数：不能，如0.3（二进制为无限循环小数0.0100110011…），无法用有限位表示。
    • Q3：字长相同时，定点数与浮点数的范围和精度区别？
      • 范围：浮点数大（阶码扩展范围）；
      • 精度：定点数高（全字长表数值，浮点数部分字长表阶码）。
    • Q4：移码表示浮点数阶码的好处？
      • 便于阶码比较（移码大→真值大）；
      • 特殊值判断简单（全0→最小阶码，全1→最大阶码）。

  • 2. 常见易混淆知识点

    • Q1：数值数据是否都是二进制？
      • 否，计算机中还有二进制编码的十进制（BCD码），用于特定场景（如十进制加法指令）。
    • Q2：无符号整数的“溢出”是什么？
      • 运算结果超出无符号数表示范围，高位被舍弃，低n位无法正确表示结果（如8位无符号数255+1=0，溢出）。
    • Q3：如何判断浮点数是否为规格化数？
      • 原码尾数：最高位为1（基数2）；
      • 补码尾数：符号位与最高数值位相反（如1.0x…x、0.1x…x）；
      • IEEE 754数：E≠0且E≠全1（规格化数）。
    • Q4：同字长定点数与浮点数的可表示个数是否相同？
      • 相同，n位编码仅能表示2n2^n2n个不同数（部分浮点数可能有重复编码，如±0，但总数接近2n2^n2n）。
    • Q5：现代计算机是否需要考虑原码加减？
      • 需要，IEEE 754浮点数尾数用原码表示，加减时需处理原码尾数运算（可转补码运算后再转原码）。
