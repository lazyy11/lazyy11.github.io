计算机组成原理：指令系统

1. 指令系统基础概念

  • 指令 (Machine Instruction)

    • 定义：指示计算机执行某种操作的二进制命令，是机器语言的基本语句。
    • 核心作用：明确操作类型（如算术加、转移）和操作数地址，是硬件可直接识别的最小执行单元。

  • 指令系统 (Instruction Set)

    • 定义：一台计算机所有机器指令的集合，也称指令集，是指令集体系结构（ISA）的核心部分。
    • 作用：
      • 作为软硬件交互的核心接口，屏蔽底层硬件细节。
      • 决定计算机的功能范围和编程灵活性，是表征计算机性能的关键因素。

  • 指令集体系结构 (Instruction Set Architecture, ISA)

    • 定义：完整定义软件与硬件之间的接口，是机器语言/汇编语言程序员需掌握的底层规范。
    • 结构/组成（规定的核心内容）：
      • 指令相关：指令格式、寻址方式、操作类型及操作数规则。
      • 数据相关：操作数类型、大/小端存放方式。
      • 存储与寄存器：可访问寄存器的编号/个数/位数、存储空间大小与编址方式。
      • 执行控制：指令执行过程的控制方式（如程序计数器PC、条件码定义）。
    • 作用：
      • 为编译器生成机器码提供依据，为硬件设计提供规范。
      • 程序员若了解ISA，可优化程序以适配硬件特性，提升性能。

2. 指令格式

  • 指令基本结构

    • 结构/组成：一条指令由两部分构成：
      • 操作码字段 (Opcode)：指出指令的操作类型（如ADD、JMP），是识别指令功能的关键。
      • 地址码字段 (Address Code)：给出操作数（或其地址）、结果地址、转移目标地址等，需结合寻址方式确定有效地址。

  • 指令字长

    • 定义：一条指令包含的二进制代码位数，取决于操作码长度、地址码长度及地址码个数。
    • 与机器字长的关系：
      • 单字长指令：指令长度 = 机器字长（访存1次即可取出）。
      • 半字长指令：指令长度 = 1/2机器字长。
      • 双字长指令：指令长度 = 2×机器字长（访存2次才能取出）。
    • 影响：指令字长直接决定取指令的时间开销（字长越长，访存次数可能越多）。

  • 指令字长结构

    • 定长指令字结构：
      • 定义：所有指令长度相同。
      • 性质/特点：执行速度快（控制逻辑简单），适用于字长32位及以上的计算机。
    • 变长指令字结构：
      • 定义：指令长度随功能而异（如简单指令短、复杂指令长）。
      • 性质/特点：可缩短平均指令长度，但增加指令译码复杂度；指令字长通常为字节的整数倍（因主存按字节编址）。

  • 按地址码数目分类的指令格式

    • 零地址指令
      • 结构：仅含操作码（OP），无地址码。
      • 指令含义：
        • 无需操作数：如空操作（NOP）、停机、关中断指令。
        • 堆栈专用：运算类指令从栈顶/次栈顶取操作数，结果压回堆栈（隐含堆栈寻址）。
    • 一地址指令
      • 结构：OP + 1个地址码（A₁）。
      • 指令含义：
        1. 单操作数指令：OP(A₁) → A₁（如INC、DEC，操作数与结果共用A₁地址）。
        2. 双操作数隐含指令：(ACC) OP(A₁) → ACC（累加器ACC隐含作为另一个操作数和结果地址）。
      • 寻址范围示例：若指令字长32位（OP占8位，A₁占24位），直接寻址范围为2242^{24}224=16MB。
      • 访存次数：取指令1次 + 取操作数1次 + 存结果1次（共3次，地址码为主存地址时）。
    • 二地址指令
      • 结构：OP + 2个地址码（A₁、A₂）。
      • 指令含义：(A₁) OP(A₂) → A₁（A₁为目的地址，A₂为源地址，结果存A₁）。
      • 寻址范围示例：指令字长32位（OP占8位，A₁/A₂各占12位），直接寻址范围为2122^{12}212=4KB。
      • 访存次数：取指令1次 + 取2个操作数2次 + 存结果1次（共4次）。
    • 三地址指令
      • 结构：OP + 3个地址码（A₁、A₂、A₃）。
      • 指令含义：(A₁) OP(A₂) → A₃（A₁/A₂为源地址，A₃为结果地址，不破坏源操作数）。
      • 寻址范围示例：指令字长32位（OP占8位，A₁/A₂/A₃各占8位），直接寻址范围为282^828=256。
    • 四地址指令
      • 结构：OP + 4个地址码（A₁、A₂、A₃、A₄）。
      • 指令含义：(A₁) OP(A₂) → A₃，A₄为下一条指令地址（替代PC的自增）。
      • 局限性：地址码字段占用空间大，指令利用率低，现已少见。

  • 操作码设计

    • 定长操作码
      • 定义：指令字中分配固定位数表示操作码（如最高n位）。
      • 指令条数计算：n位操作码最多可表示2n2^n2n条指令（如8位OP可表示256条）。
      • 优点：简化硬件设计，提高指令译码速度。
    • 扩展操作码
      • 定义：操作码长度随地址码减少而增加（变长操作码的主流形式），在指令字长有限时增加指令种类。
      • 设计原则：
        • 短码不能是长码的前缀（避免译码歧义）。
        • 各指令操作码不重复。
        • 高频指令分配短码，低频指令分配长码（减少译码时间）。
      • 示例：16位指令字（4位基本OP + 3个4位地址码）：
        • 三地址指令：用4位OP的前15种（0000~1110），1111留作扩展，共15条。
        • 二地址指令：用8位OP（1111 + 4位扩展）的前15种，11111111留作扩展，共15条。
        • 零地址指令：最终扩展为16位OP，共16条。

3. 寻址方式

  • 寻址方式核心定义

    • 定义：寻找指令或操作数有效地址的方法，分为指令寻址（找下一条指令地址）和数据寻址（找本条指令操作数地址）。
    • 关键概念：
      • 形式地址（A）：指令地址码字段给出的地址（非真实地址）。
      • 有效地址（EA）：操作数在主存/寄存器中的真实地址（由A结合寻址方式计算）。
      • 寻址特征字段：指令中用于指明寻址方式的字段（位数决定寻址方式种类）。

  • 1. 指令寻址

    • 顺序寻址
      • 定义：通过程序计数器（PC）自增，自动形成下一条指令地址。
      • PC自增规则：与编址方式、指令字长相关（按字节编址时）：
        • 指令字长16位（2字节）：PC += 2。
        • 指令字长32位（4字节）：PC += 4。
    • 跳跃寻址
      • 定义：通过转移类指令修改PC，实现程序执行顺序的跳转。
      • 类型：
        • 无条件转移：无需判断条件，直接修改PC。
        • 条件转移：仅当特定条件（如标志位为1）满足时转移。
      • 地址计算：
        • 绝对转移：A直接为目标地址 → EA = A。
        • 相对转移：EA = (PC) + A（A为相对于当前PC的偏移量，补码表示）。

  • 2. 常见数据寻址方式

    寻址方式 定义 有效地址（EA）计算 访存次数（不含取指令） 优点/适用场景 缺点/局限性
    隐含寻址 不显式给出操作数地址，隐含约定操作数位置（如累加器ACC）。 隐含地址（如ACC） 0（操作数在寄存器） 缩短指令字长，适用于单地址指令（如INC） 增加硬件复杂度（需专用寄存器）
    立即寻址 地址码字段给出的是操作数本身（立即数，补码表示）。 A即为操作数 0 执行速度最快（无需访存），适用于赋初值 操作数范围受A位数限制（如8位A仅±127）
    直接寻址 形式地址A即为有效地址。 EA = A 1 简单直观，仅1次访存 寻址范围受A位数限制，地址不易修改
    间接寻址 地址码A是有效地址的主存地址（操作数地址的地址）。 EA = (A) 2（一次间接） 扩大寻址范围（EA位数＞A位数） 访存次数多，执行速度慢
    寄存器寻址 地址码给出操作数所在寄存器的编号，操作数在寄存器内。 EA = Rᵢ（寄存器编号） 0 执行速度快（无访存），指令字长短 寄存器数量有限（通常32个以内）
    寄存器间接寻址 地址码给出的寄存器中存放的是操作数的主存地址。 EA = (Rᵢ) 1 扩大寻址范围，减少访存次数（对比间接寻址） 需访问寄存器+主存，速度慢于寄存器寻址
    相对寻址 有效地址 = PC当前值 + 形式地址A（A为偏移量，补码表示）。 EA = (PC) + A 1 程序可浮动（与指令地址偏移固定），适用于转移指令 寻址范围受A位数限制
    基址寻址 有效地址 = 基址寄存器（BR）内容 + A（BR由操作系统管理）。 EA = (BR) + A 1 多道程序设计（隔离程序地址空间），适用于浮动程序 偏移量A位数短，用户无法修改BR
    变址寻址 有效地址 = 变址寄存器（IX）内容 + A（IX由用户管理，A为数组首地址）。 EA = (IX) + A 1 数组/循环程序（修改IX遍历元素） 基址寄存器与变址寄存器功能易混淆
    堆栈寻址 操作数存于堆栈，通过堆栈指针（SP）指向栈顶，隐含栈顶/次栈顶操作数。 EA = (SP) 1（软堆栈，主存实现） 子程序调用、表达式求值（LIFO特性） 硬堆栈成本高，软堆栈访存次数多

  • 偏移寻址对比（相对、基址、变址）

    • 共性：均通过“寄存器内容 + 形式地址A”计算EA，属于偏移寻址。
    • 差异：
      维度 相对寻址 基址寻址 变址寻址
      核心寄存器 PC（程序计数器） BR（基址寄存器） IX（变址寄存器）
      寄存器管理方 硬件自动更新 操作系统 用户程序
      地址角色 A为偏移量，(PC)为基准 A为偏移量，(BR)为基准 A为基准（首地址），(IX)为偏移量
      适用场景 程序转移 多道程序地址隔离 数组遍历、循环

4. 数据的对齐和大/小端存放方式

  • 数据对齐 (Data Alignment)

    • 定义：数据在主存中的存储地址需是其长度的整数倍（如4字节int需存在地址0、4、8…）。
    • 作用：
      • 减少CPU访存次数（如未对齐4字节int可能需2次访存，对齐后仅1次）。
      • 简化硬件设计（避免地址跨边界处理）。

  • 大端存放 (Big-Endian)

    • 定义：操作数的最高有效字节（MSB）存于低地址，最低有效字节（LSB）存于高地址。
    • 示例：32位数据0x12345678（MSB=0x12，LSB=0x78），按字节编址的存储方式：
      地址 0x00 0x01 0x02 0x03
      数据 0x12 0x34 0x56 0x78
    • 适用场景：PowerPC、SPARC架构，网络协议（如TCP/IP）。

  • 小端存放 (Little-Endian)

    • 定义：操作数的最低有效字节（LSB）存于低地址，最高有效字节（MSB）存于高地址。
    • 示例：32位数据0x12345678，存储方式：
      地址 0x00 0x01 0x02 0x03
      数据 0x78 0x56 0x34 0x12
    • 适用场景：x86、ARM架构（默认），大多数PC和服务器。

  • 大/小端判断方法

    • 原理：通过存储16位数据（如0x1234），读取低地址字节值判断：
      • 若低地址字节为0x12 → 大端。
      • 若低地址字节为0x34 → 小端。

5. CISC与RISC的基本概念

  • 核心定义

    • CISC (复杂指令系统计算机)：增强指令功能，通过复杂指令实现软件功能硬化（如x86架构）。
    • RISC (精简指令系统计算机)：简化指令种类与功能，提高执行速度（如ARM、MIPS架构）。

  • 1. 复杂指令系统计算机 (CISC)

    • 性质/特点：
      • 指令系统复杂：指令数目≥200条，包含复杂指令（如字符串操作、多操作数运算）。
      • 指令格式与寻址：指令长度不固定，格式多（≥5种），寻址方式多（≥10种）。
      • 访存指令：无限制（如ADD指令可直接访存）。
      • 执行效率：指令使用频度差异大（20%简单指令占80%程序执行），多数指令需多个时钟周期。
      • 控制方式：以微程序控制为主（复杂指令无法用硬布线实现）。
      • 编译优化：难以生成高效目标代码（指令功能冗余）。

  • 2. 精简指令系统计算机 (RISC)

    • 性质/特点：
      • 指令系统精简：仅保留高频简单指令（≤100条），复杂功能由简单指令组合实现。
      • 指令格式与寻址：指令长度固定（通常32位），格式少（≤3种），寻址方式少（≤5种）。
      • 访存指令：仅LOAD/STORE指令可访存，其余指令操作均在寄存器间进行。
      • 寄存器数量：通用寄存器多（≥32个），减少访存次数。
      • 执行效率：采用指令流水线，多数指令1个时钟周期完成。
      • 控制方式：以硬布线控制为主（指令简单，逻辑简化）。
      • 编译优化：重视编译优化（指令类型少，易选择高效指令序列）。

  • CISC与RISC对比表

    对比维度 CISC RISC
    指令数目 ≥200条 ≤100条
    指令字长 不固定 固定（多为32位）
    访存指令 无限制 仅LOAD/STORE
    通用寄存器数量 较少（8~16个） 较多（≥32个）
    指令执行周期 多数需3~5个时钟周期 多数1个时钟周期
    控制方式 微程序控制为主 硬布线控制为主
    流水线支持 可选 必须支持
    软件兼容 向下兼容（高档机含低档机指令） 不兼容（指令系统简化）
    芯片面积利用率 控制存储器占50%以上 硬布线逻辑仅占10%左右

  • 发展趋势：现代CPU融合两者优势（如x86架构引入RISC内核、流水线技术；ARM引入复杂指令扩展），性能差距缩小。

6. 程序的机器级代码表示

  • 核心工具概念

    • 编译器 (Compiler)
      • 定义：将高级语言（如C/C++）代码转换为汇编语言代码的工具。
      • 作用：进行语法分析、优化（如循环展开、寄存器分配），生成高效中间代码。
    • 汇编器 (Assembler)
      • 定义：将汇编语言代码转换为机器码（二进制指令）的工具。
      • 作用：处理符号地址（如标签→物理地址）、指令编码（如OP+地址码组合）。
    • 链接器 (Linker)
      • 定义：将多个目标文件（.o）及库文件链接为可执行文件（.exe）的工具。
      • 作用：解决跨文件符号引用（如调用其他文件的函数），分配最终内存地址。

  • 1. 选择结构语句的机器级表示

    • 核心机制：通过条件码设置指令（如CMP、TEST）和条件转移指令（如JE、JG）实现分支。
    • 条件码：CPU状态字中的标志位，记录算术/逻辑运算结果属性：
      • ZF（零标志）：结果为0时置1。
      • SF（符号标志）：结果为负时置1。
      • CF（进位/借位标志）：无符号数运算溢出时置1。
      • OF（溢出标志）：有符号数运算溢出时置1。
    • 示例（if-then-else）：
      • C语言：if (a > b) { x = 1; } else { x = 0; }
      • 机器级转换步骤：
        1. CMP a, b（比较a与b，设置CF/SF/OF）。
        2. JLE else_label（若a≤b，跳转到else分支）。
        3. MOV x, 1（then分支：x=1）。
        4. JMP done_label（跳过else分支）。
        5. else_label: MOV x, 0（else分支：x=0）。
        6. done_label: （分支结束）。

  • 2. 循环结构语句的机器级表示

    • 核心机制：通过“条件测试 + 转移指令”实现，编译器通常将循环转换为do-while格式（确保循环体至少执行一次）。
    • 常见循环类型转换：
      • while循环：先测试条件，不满足则退出。
        • 转换：if (!cond) goto done; do { body } while (cond); done:
      • for循环：初始化→测试→循环体→更新。
        • 转换：init; if (!cond) goto done; do { body; update; } while (cond); done:

  • 3. 过程（函数）调用的机器级表示

    • 核心指令：
      • call 目标地址：将返回地址（下一条指令地址）压栈，然后跳转到目标函数。
      • ret：从栈中弹出返回地址，跳回调用者程序。
    • 执行步骤（调用者P → 被调用者Q）：
      1. P准备：将实参从右向左压栈（Q可通过栈访问）。
      2. P调用：执行call Q，返回地址压栈，PC指向Q入口。
      3. Q初始化：
        • 保存P的现场：将被调用者保存寄存器（EBX、ESI、EDI）压栈。
        • 建立栈帧：push ebp; mov ebp, esp（EBP为栈帧基指针，固定指向栈帧底部）。
        • 分配局部变量空间：sub esp, 局部变量大小（ESP指向栈顶）。
      4. 执行Q函数体：操作寄存器或栈中数据，结果存入EAX（返回值寄存器）。
      5. Q收尾：
        • 恢复现场：弹出栈中保存的寄存器值。
        • 释放栈帧：leave（等效于mov esp, ebp; pop ebp）。
        • 返回P：执行ret，弹出返回地址，PC指向P的下一条指令。
    • 栈帧结构（x86 32位）：
      地址（高→低） 内容 说明
      … 调用者P的栈帧 包含P的局部变量、实参等
      EBP+8 被调用者Q的第一个实参 Q通过EBP+偏移访问实参
      EBP+4 返回地址（P的下一条指令） call指令压栈
      EBP 旧EBP值（P的EBP） Q初始化时压栈
      EBP-4 Q的局部变量1 sub esp分配的空间
      EBP-8 Q的局部变量2 …
      ESP 栈顶（未使用空间） 动态变化
    • 寄存器保存规则：
      • 调用者保存寄存器（EAX、ECX、EDX）：P调用Q前需自行保存，Q可直接覆盖。
      • 被调用者保存寄存器（EBX、ESI、EDI）：Q使用前需保存，返回前恢复，避免破坏P的数据。
