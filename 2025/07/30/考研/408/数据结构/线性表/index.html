数据结构：线性表

概念要点

线性表基本概念

  • 线性表 (Linear List)
    • 定义：具有相同数据类型的 n(n≥0)n(n ≥0)n(n≥0) 个数据元素的有限序列，其中 nnn 为表长，n=0n=0n=0 时为一空表；若用 LLL 命名线性表，可表示为 L=(a1,a2,⋯ ,ai,ai+1,⋯ ,an)L=(a_{1}, a_{2}, \cdots, a_{i}, a_{i+1}, \cdots, a_{n})L=(a1​,a2​,⋯,ai​,ai+1​,⋯,an​)，其中 a1a_1a1​ 为表头元素，ana_nan​ 为表尾元素，除首尾元素外，每个元素有且仅有一个直接前驱和一个直接后继。
    • 性质/特点：
      • 表中元素的个数有限。
      • 表中元素具有逻辑上的顺序性，存在先后次序。
      • 表中元素均为单个数据元素（非组合元素）。
      • 表中元素的数据类型相同，占用相同大小的存储空间。
      • 表中元素具有抽象性，仅讨论元素间逻辑关系，不关注元素具体内容。
    • 核心操作：
      • InitList(&L)：初始化表，构造一个空的线性表。
      • Length(L)：求表长，返回线性表中数据元素的个数。
      • LocateElem(L, e)：按值查找，在表中查找具有给定关键字值的元素。
      • GetElem(L, i)：按位查找，获取表中第 iii 个位置的元素值。
      • ListInsert(&L, i, e)：插入操作，在表中第 iii 个位置插入指定元素 eee。
      • ListDelete(&L, i, e)：删除操作，删除表中第 iii 个位置的元素，并用 eee 返回删除元素值。
      • PrintList(L)：输出操作，按前后顺序输出线性表的所有元素值。
      • Empty(L)：判空操作，若表为空则返回true，否则返回false。
      • DestroyList(&L)：销毁操作，销毁线性表并释放占用的内存空间。
    • 关键区分：
      • 线性表是逻辑结构，表示元素间一对一的相邻关系；顺序表和链表是存储结构，属于不同层面的概念，不可混淆。
      • 基本操作的实现依赖存储结构，存储结构不同，算法实现不同；符号“&”表示C++的引用调用，C语言中用指针可达到同等效果。

线性表的顺序表示（顺序表）

  • 顺序表 (Sequential List)
    • 定义：线性表的顺序存储结构，用一组地址连续的存储单元依次存储线性表中的数据元素，使逻辑上相邻的元素在物理位置上也相邻；第 iii 个元素的存储位置可通过起始地址计算，支持随机存取。
    • 存储分配方式：
      • 静态分配：
        • 结构定义：通过#define定义最大长度MaxSize，用结构体包含数组（存储元素）和length（当前长度），数组大小固定。
        • 特点：空间事先固定，一旦占满再插入新元素会溢出，导致程序崩溃。
      • 动态分配：
        • 结构定义：结构体包含指针data（指向动态分配数组）、MaxSize（数组最大容量）和length（当前长度），初始容量通过Initsize定义。
        • 动态分配语句：
          • C语言：L.data=(ElemType*)malloc(sizeof(ElemType)*Initsize);
          • C++：L.data=new ElemType[Initsize];（文档中表述略有笔误，修正后规范形式）
        • 特点：空间可在运行时扩充，满时开辟更大空间并拷贝原元素，仍属顺序存储，物理结构不变。
    • 核心操作实现：
      • 初始化：
        • 静态分配：仅需将顺序表的length设为0，如void InitList(SqList &L){ L.length=0; }。
        • 动态分配：分配预定义大小的数组空间，length设为0，MaxSize设为Initsize，如void InitList(SeqList &L){ L.data=(ElemType *)malloc(Initsize*sizeof(ElemType)); L.length=0; L.MaxSize=Initsize; }。
      • 插入操作（ListInsert）：
        • 算法步骤：
          1. 判断插入位置 iii 是否合法（1≤i≤L.length+11≤i≤L.length+11≤i≤L.length+1），不合法则返回false。
          2. 判断当前存储空间是否已满（L.length≥MaxSizeL.length≥MaxSizeL.length≥MaxSize），满则返回false。
          3. 从表尾到第 iii 个元素，依次后移一个位置（for循环：j=L.length;j≥i;j−−j=L.length;j≥i;j--j=L.length;j≥i;j−−）。
          4. 在第 iii 个位置（数组下标 i−1i-1i−1）插入元素 eee，length加1，返回true。
      • 删除操作（ListDelete）：
        • 算法步骤：
          1. 判断删除位置 iii 是否合法（1≤i≤L.length1≤i≤L.length1≤i≤L.length），不合法则返回false。
          2. 用 eee 保存第 iii 个位置（数组下标 i−1i-1i−1）的元素值。
          3. 从第 i+1i+1i+1 个元素到表尾，依次前移一个位置（for循环：j=i;j<L.length;j−−j=i;j<L.length;j--j=i;j<L.length;j−−）。
          4. length减1，返回true。
      • 按值查找（LocateElem）：
        • 算法步骤：遍历数组，比较每个元素与 eee，找到则返回其位序（下标+1），未找到返回0，如int LocateElem(SqList L,ElemType e){ for(i=0;i<L.length;i++) if(L.data[i]==e) return i+1; return 0; }。
    • 复杂度分析：
      • 插入操作：
        • 时间复杂度：最好情况（表尾插入，i=n+1i=n+1i=n+1）O(1)O(1)O(1)；最坏情况（表头插入，i=1i=1i=1）O(n)O(n)O(n)；平均情况 O(n)O(n)O(n)（平均移动 n/2n/2n/2 个元素）。
        • 空间复杂度：O(1)O(1)O(1)（仅用少量临时变量）。
      • 删除操作：
        • 时间复杂度：最好情况（表尾删除，i=ni=ni=n）O(1)O(1)O(1)；最坏情况（表头删除，i=1i=1i=1）O(n)O(n)O(n)；平均情况 O(n)O(n)O(n)（平均移动 (n−1)/2(n-1)/2(n−1)/2 个元素）。
        • 空间复杂度：O(1)O(1)O(1)。
      • 按值查找：
        • 时间复杂度：最好情况（表头找到）O(1)O(1)O(1)；最坏情况（表尾或未找到）O(n)O(n)O(n)；平均情况 O(n)O(n)O(n)（平均比较 (n+1)/2(n+1)/2(n+1)/2 次）。
      • 按位查找：时间复杂度 O(1)O(1)O(1)（直接通过数组下标访问）。
    • 优点/适用场景：
      • 优点：支持随机存取，可通过首地址和元素序号在 O(1)O(1)O(1) 时间找到指定元素；存储密度高，每个结点仅存储数据元素。
      • 适用场景：频繁进行按位存取操作，且元素个数相对稳定的场景。
    • 缺点/局限性：
      • 缺点：插入和删除操作需移动大量元素，效率低；需连续的存储空间，分配不够灵活，静态分配易溢出，动态分配扩容时需拷贝元素。
      • 局限性：难以估计线性表长度或存储规模时，不适合使用。

线性表的链式表示

  • 单链表 (Singly Linked List)

    • 定义：线性表的链式存储结构，用一组任意的存储单元存储数据元素，每个结点包含数据域（data）和指针域（next，指向后继结点）；通过头指针标识链表，头指针为NULL时表示空表。
    • 结构/组成：
      • 结点结构定义：typedef struct LNode{ ElemType data; struct LNode *next; }LNode,*LinkList;（LinkList为指向LNode的指针类型）。
      • 头结点：在第一个数据结点前附加的结点，数据域可存表长等信息或空；头指针指向头结点，空表时头结点next为NULL。
    • 核心操作实现：
      • 初始化：
        • 带头结点：创建头结点，头指针指向头结点，next设为NULL，如bool InitList(LinkList&L){ L=(LNode*)malloc(sizeof(LNode)); L->next=NULL; return true; }。
        • 不带头结点：仅将头指针设为NULL，如bool InitList(LinkList &L){ L=NULL; return true; }。
      • 插入操作：
        • 第 iii 个位置插入：先找第 i−1i-1i−1 个结点（前驱），新结点s的next指向前驱的next，前驱的next指向s，需注意指针修改顺序（先连后继再连前驱，避免断链）。
        • 前插操作：可转化为后插（将s插入p后，交换p和s的数据域），时间复杂度 O(1)O(1)O(1)。
      • 删除操作：
        • 第 iii 个位置删除：找第 i−1i-1i−1 个结点（前驱），用q指向被删结点，前驱的next指向q的next，释放q。
        • 删除指定结点p：可转化为删除p的后继（将p后继的数据域赋给p，删除后继），时间复杂度 O(1)O(1)O(1)。
      • 建立链表：
        • 头插法：新结点插入表头（头结点后），读入数据顺序与链表元素顺序相反，可实现逆置，时间复杂度 O(n)O(n)O(n)。
        • 尾插法：新结点插入表尾（需尾指针r），读入数据顺序与链表元素顺序一致，时间复杂度 O(n)O(n)O(n)。
    • 复杂度分析：
      • 按位查找、按值查找、插入（非指定结点后）、删除（非指定结点）：时间复杂度均为 O(n)O(n)O(n)（需遍历链表）。
      • 指定结点后插入/删除：时间复杂度 O(1)O(1)O(1)。
      • 空间复杂度：O(1)O(1)O(1)（仅用指针变量）。
    • 优点/适用场景：
      • 优点：插入和删除无需移动元素，仅修改指针；无需连续存储空间，空间分配灵活。
      • 适用场景：频繁进行插入、删除操作，且线性表长度不确定的场景。
    • 缺点/局限性：
      • 缺点：非随机存取，查找需遍历；每个结点含指针域，存储密度低。

  • 双链表 (Doubly Linked List)

    • 定义：每个结点除data和next（指向后继）外，增加prior（指向前驱）的链表；表头结点prior和尾结点next为NULL，支持双向遍历。
    • 核心操作：
      • 插入（p后插入s）：算法步骤为 s->next=p->next; p->next->prior=s; s->prior=p; p->next=s;（①需在④前，避免断链）。
      • 删除（删除p的后继q）：算法步骤为 p->next=q->next; q->next->prior=p; free(q);。
    • 复杂度分析：插入、删除操作时间复杂度 O(1)O(1)O(1)（可直接访问前驱），查找仍为 O(n)O(n)O(n)。

  • 循环链表 (Circular Linked List)

    • 循环单链表：
      • 定义：表尾结点next指向头结点，无NULL指针；判空条件为L->next==L（L为头指针）。
      • 特点：设尾指针时，表头/表尾插入效率为 O(1)O(1)O(1)（尾指针next即头指针）。
    • 循环双链表：
      • 定义：头结点prior指向表尾，尾结点next指向头结点；空表时头结点prior和next均指向自身。

  • 静态链表 (Static Linked List)

    • 定义：用数组模拟链式存储，结点含data和next（游标，数组下标）；预先分配连续空间，next=-1为结束标志。
    • 特点：插入、删除无需移动元素，仅修改游标；但最大长度固定，不支持动态扩容，适用于不支持指针的语言。

  • 顺序表与链表的对比

    • 存取方式：
      • 顺序表：可顺序存取、随机存取；链表：仅顺序存取（从表头遍历）。
    • 逻辑与物理结构：
      • 顺序表：逻辑相邻→物理相邻；链表：逻辑相邻→物理不一定相邻（靠指针链接）。
    • 操作效率：
      • 查找：顺序表无序时 O(n)O(n)O(n)、有序时折半查找 O(log⁡2n)O(\log_2 n)O(log2​n)；链表均 O(n)O(n)O(n)。
      • 插入/删除：顺序表平均移动 n/2n/2n/2 个元素（O(n)O(n)O(n)）；链表仅修改指针（O(1)O(1)O(1)，需找到位置时 O(n)O(n)O(n)）。
    • 空间分配：
      • 顺序表：静态分配易溢出，动态分配需拷贝元素；链表：结点按需分配，灵活但存储密度低。

线性表的应用与试题要点

  • 常见算法设计场景

    • 顺序表算法：
      • 删除最小值元素（用最后一个元素填补空位置）。
      • 元素逆置（扫描前半部分，与后半部分对应元素交换）。
      • 删除所有值为 xxx 的元素（用k记录非 xxx 元素个数，前移元素）。
      • 合并两个有序顺序表（两两比较，小者存入结果表）。
    • 链表算法：
      • 单链表删除所有值为 xxx 的结点（前驱指针pre同步遍历，删除时断链并释放）。
      • 单链表就地逆置（头插法或指针反转）。
      • 有序单链表去重（扫描链表，删除相邻重复结点）。

  • 算法设计原则

    • 优先保证时间/空间复杂度最优，如顺序表逆置空间复杂度 O(1)O(1)O(1)、链表去重时间复杂度 O(n)O(n)O(n)。
    • 算法思想优先于代码细节，考场中可忽略边界判断、内存分配不足等细节，重点表达步骤。
    • 时间紧迫时可先用暴力法（如顺序查找），仍能获得大部分分数。

  • 试题考查重点

    • 单项选择题：考查线性表定义（如“线性表是n个数据元素的有限序列”）、存储结构特点（如顺序表随机存取、链表插入无需移元素）、操作复杂度（如顺序表按位查找 O(1)O(1)O(1)）
    • 综合应用题：
      • 要求设计算法并分析复杂度，如“循环左移数组”“找两个有序序列的中位数”“判断单链表是否有环”。
      • 需注意指针操作（如双链表插入/删除的指针修改顺序）、边界条件（如空表、表尾操作）。
