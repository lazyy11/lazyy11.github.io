计算机网络：物理层

一、通信基础

  • 1. 基本概念

    • 数据、信号与码元

      • 定义：
        • 数据：传送信息的实体（如文字、图像），分模拟数据（取值连续）和数字数据（取值离散）。
        • 信号：数据的电气/电磁表现，分模拟信号（连续变化）和数字信号（离散电平）。
        • 码元（Symbol）：固定时长的信号波形，代表1个k进制数，时长称为码元宽度（信号周期）；1码元可携带多比特信息（如2进制码元带1bit，4进制码元带2bit）。
      • 核心关系：码元携带的比特数 = log2klog_2klog2​k（k为码元的离散电平数量）。
      • 数字信号 vs 模拟信号（考纲内）：
        • 取值特性：数字信号取值离散（多为矩形/多电平波形），模拟信号取值连续（多为正弦/连续波形）。
        • 抗噪与再生：数字信号可经中继器再生（判决后重构波形），抗噪性强；模拟信号经放大器会连同噪声一起放大，远距更易失真。
        • 传输形态：数字信号可基带直接传输；模拟信号常为带通信号（经载波调制），适合无线与远距离。
        • 示例：
          • 数字信号：以太网双绞线上的NRZ/曼彻斯特/MLT-3等基带方波/多电平波形。
          • 模拟信号：广播AM/FM的射频正弦波；Wi‑Fi/蜂窝物理层在空口上传输的是模拟带通信号（用QAM/OFDM等数字调制承载比特）；传统话音在电话环路上为模拟。

    • 信源、信道与信宿

      • 定义：
        • 信源：产生/发送数据的源头（如计算机）。
        • 信宿：接收数据的终点（如终端）。
        • 信道：信号传输介质，是通信系统的核心通路。
      • 结构/组成：单向通信系统模型 = 信源 → 变换器 → 信道（+噪声源）→ 反变换器 → 信宿；双向通信含发送/接收2个信道。
      • 信道分类：
        • 按传输信号：模拟信道（传模拟信号）、数字信道（传数字信号）。
        • 按传输介质：有线信道（双绞线、光纤等）、无线信道（无线电波等）。
      • 传输相关概念：
        • 基带信号：信源发出的未调制原始电信号；直接传输基带信号称为基带传输（适用于局域网）。
        • 宽带信号：基带信号调制后的频分复用模拟信号；传输称为宽带传输（适用于远距离）。
        • 传输方式：
          • 串行传输：逐比特按序传输（适用于长距离，如计算机网络）。
          • 并行传输：多比特同时通过多信道传输（适用于近距离，如CPU与主存）。
        • 交互方式：
          • 单向通信（单工）：仅一个方向传输（如电视广播），需1个信道。
          • 半双工通信：双方可收发，但不能同时（如对讲机），需2个信道。
          • 全双工通信：双方可同时收发（如电话），需2个信道。

    • 速率、波特与带宽

      • 定义：
        • 速率：单位时间传输的数据量，分两类：
          • 码元传输速率（波特率）：每秒传输的码元数，单位Baud；与进制无关。
          • 信息传输速率（比特率）：每秒传输的比特数，单位b/s。
        • 带宽：
          • 模拟信号系统：信道能传输的频率范围（最高频-最低频），单位Hz。
          • 计算机网络：信道能传输数据的最高速率，单位b/s。
      • 核心关系：比特率 = 波特率 × log2Vlog_2Vlog2​V（V为码元的离散电平数量）；波特率 = 比特率 ÷ 每码元携带的比特数。

  • 2. 信道的极限容量

    • 奈奎斯特定理（Nyquist’s Theorem，奈氏准则）

      • 适用场景：理想低通信道（无噪声、带宽有限），解决码间串扰问题（高频分量衰减导致码元界限模糊）。
      • 核心公式：
        • 极限码元传输速率 = 2W2W2W Baud（W为信道频率带宽，单位Hz）。
        • 极限数据传输速率 = 2Wlog2V2W log_2V2Wlog2​V b/s（V为码元离散电平数量）。
      • 关键结论：
        • 码元传输速率有上限，超上限会出现严重码间串扰。
        • 信道带宽越大，传输码元的能力越强。
        • 仅限制码元速率，不限制信息速率（可通过多元制调制提高每码元携带的比特数）。

    • 香农定理（Shannon’s Theorem）

      • 适用场景：带宽受限且有高斯噪声干扰的实际信道。
      • 核心公式：信道极限数据传输速率 = Wlog2(1+S/N)W log_2(1+S/N)Wlog2​(1+S/N) b/s；其中：
        • W：信道频率带宽（Hz）。
        • S：信号平均功率，N：噪声平均功率，S/NS/NS/N为信噪比（无单位）。
      • 信噪比换算：分贝（dB）记法 = 10log10(S/N)10 log_{10}(S/N)10log10​(S/N)；计算时需将dB转换为无单位S/NS/NS/N（如30dB对应S/N=1000S/N=1000S/N=1000）。
      • 关键结论：
        • 带宽或信噪比越大，信息极限传输速率越高。
        • 固定带宽和信噪比时，信息速率上限确定。
        • 速率低于上限时，存在无差错传输方法；实际速率远低于理论值。

    • 奈氏准则与香农定理的区别

      • 奈氏准则：仅考虑带宽对码元速率的限制，不考虑噪声。
      • 香农定理：同时考虑带宽和信噪比，限制信息传输速率；表明每码元携带的比特数有限。

  • 3. 编码与调制

    • 定义：

      • 编码：将数据转换为数字信号的过程（用于数字传输）。
      • 调制：将数据转换为模拟信号的过程（用于模拟传输）。
      • 编码 vs 调制的区别（考纲内）：
        • 目标域：编码发生在基带/数字域（输出数字信号）；调制发生在带通/模拟域（输出模拟载波）。
        • 适用信道：编码用于数字基带链路（如以太网双绞线/光纤基带）；调制用于带通信道（如电话网铜线的高频段、无线空口、有线电视同轴）。
        • 输入/输出：编码输入数据位流→输出数字波形；调制输入数据（数字或模拟）→输出模拟带通信号。
        • 示例：
          • 编码：10BASE-T的曼彻斯特，100BASE-TX的MLT-3+4B/5B（输出仍为基带电信号）。
          • 调制：ADSL/VDSL、4G/5G、Wi‑Fi采用QAM/OFDM等（输出为空口/铜线上传输的模拟载波信号）。

    • 分类及细节：

      • 数字数据编码为数字信号（基带传输）

        • 核心要求：有效区分0和1，部分需提供同步机制。
        • 常见编码方式：
          • 归零编码（Return-to-Zero，RZ）：
            • 规则：高电平表1、低电平表0（或相反），每个码元中间跳变到零电平。
            • 特点：提供自同步（跳变调整时钟）；但归零占用带宽，传输效率低。
          • 非归零编码（Non-Return-to-Zero，NRZ）：
            • 规则：无归零，一个时钟全传数据（高电平1、低电平0或相反）。
            • 特点：编码效率较高；但不自同步，需额外同步机制（如帧定界/码间填充/时钟恢复），否则长零/长一会导致时钟恢复困难。
          • 反向非归零编码（Non-Return-to-Zero Inverted，NRZI）：
            • 规则：常见约定为“0致跳变、1不变”（亦有实现相反约定）。
            • 特点：改善时钟恢复（长1序列仍可产生边沿）；通常与位填充/帧定界配合使用；USB2.0采用“0致跳变、1不变”。
          • 曼彻斯特编码（Manchester Encoding）：
            • 规则：每个码元中间必跳变，跳变既作时钟（同步），又作数据（如下跳表1、上跳表0，或相反）。
            • 特点：自带同步；经典10 Mb/s以太网采用，带宽需求约为比特率的两倍。
          • 差分曼彻斯特编码（Differential Manchester Encoding）：
            • 规则：每个码元中间必跳变（仅作时钟）；数据由码元开始处是否跳变决定（无跳变表1，有跳变表0）。
            • 特点：抗干扰能力强且自同步，但带宽效率低于NRZ/NRZI；历史上用于Token Ring，并非主流高速以太网物理层方案。

      • 模拟数据编码为数字信号

        • 核心技术：脉冲编码调制（Pulse Code Modulation, PCM），常用于音频信号。
        • 算法步骤：
          1. 采样：周期性扫描模拟信号，将连续信号转为离散信号；采样频率 ≥ 模拟信号最大频率的2倍（奈奎斯特采样定理）。
          2. 量化：将采样电平按分级标度转换为离散数值并取整（连续→离散）。
          3. 编码：将量化后的整数转换为二进制编码。
        • 名词澄清：PCM 名含“调制”，但并非把信号搬移到载波频段；其本质是模拟→数字的 A/D 与编码过程，输出比特流。所谓“解调”是针对已调载波（如 ASK/FSK/PSK/QAM）恢复基带；若需得到数字比特，还需随后判决/解码。
        • 示例（编码）：
          • 录音：麦克风模拟话音 → 声卡A/D+PCM（如8 kHz采样、8/16 bit量化；话音常用A律/μ律压扩）→ WAV/VoIP比特流。
          • 电话网：PSTN/E1 话路：8 kHz × 8 bit = 64 kb/s/路；30/31路复用成 2.048 Mb/s 数字支路。
          • 图像采集：摄像头模拟电平 → 采样量化 → 数字像素流（随后JPEG/H.264为源编码/压缩，属于更高层编码）。
        • 对比例子（解调）：
          • 收音机：接收AM/FM载波 → 解调得到模拟音频（是基带模拟，不是比特）。
          • Wi‑Fi/4G：空口QAM/OFDM载波 → 解调/同步/均衡 → 符号判决 → 比特 → MAC帧（先解调出基带，再判决得到比特）。

      • 数字数据调制为模拟信号（需调制解调器）

        • 核心思路：改变载波的振幅、频率、相位来表示0和1。
        • 常见调制方式：
          • 幅移键控（Amplitude Shift Keying，ASK）：
            • 规则：改变载波振幅（如有载波表1、无载波表0）。
            • 特点：易实现；抗干扰能力差。
          • 频移键控（Frequency Shift Keying，FSK）：
            • 规则：改变载波频率（如频率f1f_1f1​表1、f2f_2f2​表0）。
            • 特点：易实现；抗干扰能力强，应用广。
          • 相移键控（Phase Shift Keying，PSK）：
            • 规则：改变载波相位（如相位0表1、π表0，绝对调相）。
            • 扩展：差分相移键控（Differential PSK，DPSK），通过当前与前一码元的相位差表示数据（如相位不变表1、变表0）。
          • 正交幅度调制（Quadrature Amplitude Modulation，QAM）：
            • 规则：结合ASK（振幅）和PSK（相位），频率相同，叠加信号。
            • 核心公式：比特率Rb=Rs log⁡2(mn)R_b = R_s\,\log_2(mn)Rb​=Rs​log2​(mn)（RsR_sRs​为符号率/波特率，mmm为相位数，nnn为每相位的振幅数）；与物理带宽WWW(Hz)区分。
            • 特点：高频带利用率；适用于高速数据传输。

      • 模拟数据调制为模拟信号

        • 目的：提高传输频率以实现有效性，可结合频分复用（Frequency Division Multiplexing，FDM）充分利用带宽。
        • 应用：电话机与本地局交换机的模拟信号传输。

  • 4. 交换技术

    • 电路交换

      • 定义：通信前建立专用物理通路，通信中独占通路，通信后释放通路（如电话网络）。
      • 算法步骤：
      1. 建立连接：主叫向交换机发连接请求，交换机逐级转接至被叫，建立专用通路。
      2. 数据传输：数据沿专用通路连续传输，无存储转发时延，但仍有发送时延与传播时延（处理时延通常可忽略）。
      3. 释放连接：通信结束后，双方释放通路资源。
      • 优点：通信延迟小、无冲突、适用于实时传输（如语音）。
      • 缺点：资源利用率低（通路独占）、不适用于突发数据传输、建立连接耗时。

    • 报文交换

      • 定义：将数据封装为“报文”（含完整目的地址），无需建立专用通路，报文在交换机中存储后转发（如早期电子邮件）。
      • 特点：
        • 交换机存储报文，按目的地址转发至下一跳。
        • 报文大小无固定限制，需交换机有较大缓存。
      • 优点：资源利用率高（共享通路）、无需建立连接。
      • 缺点：存储转发延迟大（报文需完整接收后转发）、不适用于实时传输、大报文易导致缓存溢出。

    • 分组交换

      • 定义：将报文分割为固定大小的“分组”（含目的地址和分组序号），分组独立在交换机中存储转发（如互联网）。
      • 分类：
        • 数据报：分组独立选路，可能失序、丢失，需接收方重组（如IP网络）。
        • 虚电路：通信前建立逻辑通路（虚电路号），分组按虚电路号传输，有序、无失序（如X.25、ATM）。
      • 优点：
        • 延迟小（分组小，转发快）、资源利用率高、适用于突发数据传输。
        • 数据报：灵活、故障时可重选路；虚电路：有序、传输效率高。
      • 缺点：
        • 数据报：分组可能失序、丢失，需接收方处理；虚电路：需建立连接，故障时虚电路失效。
        • 存在分组头部开销。

二、传输介质

  • 1. 导向传输介质（电磁波沿固体介质传播）

    • 双绞线（Twisted Pair）

      • 结构/组成：两根绝缘铜导线按规则绞合；外层可加金属屏蔽层（屏蔽双绞线，Shielded Twisted Pair，STP）或无（非屏蔽双绞线，Unshielded Twisted Pair，UTP）。
      • 性质/特点：
        • 绞合目的：减少相邻导线的电磁干扰。
        • 价格便宜，模拟/数字传输均可。
        • 通信距离：以太网UTP典型单段≤100 m（Cat5e/6/6A）；更长距离需中继/交换或改用光纤。传统电话环路可达数公里，但速率有限且需放大/再生。
      • 应用场景：局域网（如以太网）、传统电话网。

    • 同轴电缆（Coaxial Cable）

      • 结构/组成：内导体（硬铜线）→ 绝缘层 → 外导体屏蔽层（密织网状）→ 绝缘保护套。
      • 分类及应用：
        • 50Ω同轴电缆：传基带数字信号，早期局域网应用。
        • 75Ω同轴电缆：传宽带信号，有线电视系统应用。
      • 性质/特点：抗干扰性强（屏蔽层作用）、带宽高；但成本高于双绞线，目前局域网已少用。

    • 光纤（Optical Fiber）

      • 结构/组成：纤芯（高折射率，8~100μm）→ 包层（低折射率）→ 保护套。
      • 传输原理：光的全反射（光线从纤芯射向包层，入射角＞临界角时反射回纤芯，沿纤芯传输）。
      • 分类及区别：
        类型 纤芯直径 光源 传输距离 成本 特点
        多模光纤 粗（数十μm） 发光二极管（LED） 短（数百米） 低 多束光线传输，损耗高
        单模光纤 细（几μm） 半导体激光器 长（数千米至数十千米） 高 单束光线直线传输，损耗低
      • 性质/特点：
        • 优点：带宽潜力极大（可见光频率约10^14 Hz量级）、传输损耗小、抗电磁干扰/雷电、无串音、保密性好、体积小重量轻。
        • 缺点：成本高、安装维护复杂。
      • 应用场景：远距离高速传输（如骨干网、城域网）。

  • 2. 非导向传输介质（电磁波在自由空间传播，无线传输）

    • 无线电波（Radio Waves）

      • 性质/特点：穿透能力强、传输距离远、信号全向散播。
      • 优点：接收设备无需对准发射方向，简化连接。
      • 应用场景：无线手机通信、无线局域网（WLAN）。

    • 微波（Microwaves）

      • 性质/特点：频率高（2~40GHz）、带宽大、信号沿直线传播（需视线通路）。
      • 传输方式：地面微波（需中继站接力，传输距离有限）、卫星微波（地球同步卫星中继，覆盖广）。
      • 应用场景：卫星通信（全球通信）、地面微波链路（骨干网补充）。
      • 优缺点：
        • 优点：通信容量大、距离远、覆盖广。
        • 缺点：保密性差、卫星通信单程≈250 ms（RTT≈500–600+ ms）、受气候影响。

    • 红外线（Infrared）

      • 性质/特点：视线传播、定向性强、不能穿透障碍物。
      • 应用场景：短距离通信（如红外遥控、近距离设备互联）。

    • 激光（Laser）

      • 性质/特点：视线传播、定向性极强、带宽高、抗干扰性强。
      • 应用场景：短距离高速通信（如建筑物间链路）。
      • 缺点：易受天气影响（如雨、雾）、需精确对准。

  • 3. 物理层接口特性

    • 定义：物理层通过接口特性屏蔽传输介质差异，使数据链路层无需关注硬件细节。
    • 四大特性：
      • 机械特性：指明接口接线器的形状、尺寸、引脚数量及排列、固定/锁定装置（如RJ45接头的引脚排列）。
      • 电气特性：指明接口电缆各线的电压范围、判决阈值、传输速率与距离限制（如RS-232：+3 ~ +15V判0，-3 ~ -15V判1，阈值约±3V；典型线长≤15 m）。
      • 功能特性：指明某线电平的意义（如+11V表1）、每条线的功能（数据线、控制线、时钟线）。
      • 过程特性（规程特性）：指明不同功能的事件发生顺序（如数据发送前先发控制信号）。

三、物理层设备

  • 1. 中继器（Repeater）

    • 定义：工作在物理层，放大、整形并转发数字信号，消除信号衰减和失真，扩大网络传输距离的设备。
    • 核心功能：
      • 信号再生（非简单放大，含放大+整形）；
      • 仅作用于信号电气部分，不处理数据内容（如错误数据）。
    • 性质/特点：
      • 连接的网段属于同一局域网，不分割冲突域和广播域。
      • 有数量限制：受网络标准延迟限制（如以太网“5-4-3规则”：4个中继器串联5段介质，仅3段可挂计算机）。
      • 不能连接速率/协议不同的网段（无存储转发功能）。
    • 与放大器的区别：放大器放大模拟信号（同时放大噪声），中继器再生数字信号（减少失真）。

  • 2. 集线器（Hub）

    • 定义：工作在物理层，本质是多端口中继器；接收某端口信号后，整形放大并转发至所有其他工作端口。
    • 核心功能：
      • 信号再生与广播转发：无寻址功能，数据从输入端口以外的所有端口广播出去。
      • 扩展网络覆盖范围：集中连接多台设备（如计算机）。
    • 性质/特点：
      • 共享带宽：所有端口属于同一冲突域，任意时刻仅一个端口可输入有效数据（如10Mb/s Hub连接8台计算机，每台平均带宽≤1.25Mb/s）。
      • 半双工工作：不能同时收发数据。
      • 逻辑拓扑：物理上是星形结构，逻辑上是总线结构（广播方式）。
      • 不分割冲突域和广播域：多设备同时发送会导致冲突，降低效率。
    • 缺点：冲突频繁、带宽利用率低；目前已被交换机（数据链路层设备）取代。

四、本章关键总结与疑难点

  • 1. 传输介质与物理层的区别

    • 传输介质：属于“0层”，仅传输信号，不识别信号含义（不知道0/1）。
    • 物理层：OSI第一层，规定电气特性（如电平表0/1），能识别比特流；屏蔽传输介质差异，为数据链路层提供统一接口。

  • 2. 基带传输、频带传输、宽带传输的区别

    类型 定义 信号类型 传输介质 适用场景
    基带传输 直接传输未调制的基带信号 数字信号 双绞线、光纤 局域网（短距离）
    频带传输 传输基带信号调制后的模拟信号 模拟信号 同轴电缆、无线 广域网（长距离）
    宽带传输 频分复用多个子信道，同时传输多信号 模拟信号 同轴电缆、无线 多信号并发传输
    • 补充要点（考纲内）：
      • 基带传输：直接发送数字基带，无载波调制；实现简单，抗噪声能力依赖线路编码与再生；常用于以太网双绞线/局域光纤。
      • 频带传输（带通）：需调制到载波频段，便于远距与无线传输；可与FDM叠加；适用于公共信道承载。
      • 宽带传输：强调“多个并行子信道的频分复用”，每个子信道可承载独立信号（如CATV、有线调制解调系统）。
      • 易错点：
        • “带宽”在通信里既指频率范围(Hz)也指数据速率(b/s)；需结合上下文区分。
        • 基带不等于“低速”，高速以太网同样是基带传输（通过多电平PAM、编码/均衡实现）。

  • 3. 奈氏准则与香农定理的核心差异

    • 关注重点：奈氏管“码元速率上限”（无噪声），香农管“信息速率上限”（有噪声）。
    • 提升速率方式：奈氏可通过增加码元电平数量（多元制调制）提升信息速率；香农需增加带宽或信噪比。
    • 实际应用：需同时满足两者限制（取速率最小值）；如二进制信号在有噪声信道，需同时计算奈氏速率（2W2W2W）和香农速率（Wlog2(1+S/N)W log_2(1+S/N)Wlog2​(1+S/N)），取较小值。
    • 补充要点（考纲内）：
      • 上限类型区别：
        • 奈氏：Rsmax⁡=2WR_s^{\max}=2WRsmax​=2W（Baud），不含噪声；信息速率Rb=Rs log⁡2VR_b=R_s\,\log_2 VRb​=Rs​log2​V，对二进制V=2V=2V=2时Rbmax⁡=2WR_b^{\max}=2WRbmax​=2W。
        • 香农：Rbmax⁡=W log⁡2(1+S/N)R_b^{\max}=W\,\log_2(1+S/N)Rbmax​=Wlog2​(1+S/N)（b/s），含噪声上限；不直接给符号率，但限制可达的每码元比特数与总比特率。
      • 是否存在“min{奈氏, 香农}”考题：有，且常见于408计算题。
        • 做法：给定WWW与SNRdB\mathrm{SNR}_{\mathrm{dB}}SNRdB​（先换算S/NS/NS/N），若指明进制VVV，先算Rb(N)=2W log⁡2VR_b^{(N)}=2W\,\log_2 VRb(N)​=2Wlog2​V与Rb(S)=W log⁡2(1+S/N)R_b^{(S)}=W\,\log_2(1+S/N)Rb(S)​=Wlog2​(1+S/N)，取min⁡{Rb(N),Rb(S)}\min\{R_b^{(N)},R_b^{(S)}\}min{Rb(N)​,Rb(S)​}。
        • 若未指明VVV，题意多默认二进制或给出现代调制阶数（如MMM-QAM/MMM-PAM）。
      • 快速步骤：
        1. dB→线性：S/N=10dB/10S/N=10^{\mathrm{dB}/10}S/N=10dB/10；
        2. 奈氏：Rb(N)=2W log⁡2VR_b^{(N)}=2W\,\log_2 VRb(N)​=2Wlog2​V（二进制取2W2W2W）；
        3. 香农：Rb(S)=W log⁡2(1+S/N)R_b^{(S)}=W\,\log_2(1+S/N)Rb(S)​=Wlog2​(1+S/N)；
        4. 取较小者；必要时再反推Rs=Rb/log⁡2VR_s=R_b/\log_2 VRs​=Rb​/log2​V。
      • 易错点：
        • 把WWW(Hz)误作RsR_sRs​(Baud)；
        • 忘记将dB换算为无单位S/NS/NS/N；
        • 奈氏用于“无噪声低通”上限，不代表实际可无限提升VVV（受SNR与实现限制）。

  • 4. 信噪比采用分贝（dB）表示的原因

    • 数值简化：信噪比范围极大（如S/N=109S/N=10^9S/N=109），dB表示可将大数转为小数（10910^9109对应90dB），避免计数错误。
    • 计算方便：dB为对数尺度，多个设备的噪声叠加可通过加法计算（而非乘法）。
    • 公式：分贝信噪比 = 10log10(S/N)10 log_{10}(S/N)10log10​(S/N)；计算香农速率时需转换为无单位S/NS/NS/N（如30dB → S/N=1030/10=1000S/N=10^{30/10}=1000S/N=1030/10=1000）。

五、统一符号与记号（考纲内）

  • 符号：
    • WWW：物理带宽（Hz，频率范围）。
    • RsR_sRs​：符号率/波特率（Baud，码元/秒）。
    • RbR_bRb​：比特率（b/s）。
    • MMM：调制阶数（如M-QAM、M-PAM）。
    • S/NS/NS/N：无单位信噪比；SNRdB=10log⁡10(S/N)\text{SNR}_{\text{dB}}=10\log_{10}(S/N)SNRdB​=10log10​(S/N)。
    • 关系：Rb=Rs log⁡2MR_b = R_s\,\log_2 MRb​=Rs​log2​M；奈氏：Rs≤2WR_s\le 2WRs​≤2W；香农：Rb≤Wlog⁡2(1+S/N)R_b\le W\log_2(1+S/N)Rb​≤Wlog2​(1+S/N)。

六、性能指标（考纲内）

  • 时延组成：
    • 发送时延：tsend=L/Rbt_{\text{send}}=L/R_btsend​=L/Rb​（L比特，链路速率RbR_bRb​）。
    • 传播时延：tprop=d/vt_{\text{prop}}=d/vtprop​=d/v（距离d，介质传播速率v）。
    • 排队/处理时延：受负载与设备影响，常为期望值。
  • 带宽-时延积：Rb×RTTR_b\times\text{RTT}Rb​×RTT，表示信道“管内”比特量。
  • 吞吐量与利用率：
    • 吞吐量：单位时间成功传输数据量（随负载/拥塞变化）。
    • 利用率：有效占用/可用带宽之比。
    • 注：本节为考纲内常规计算与概念。

七、复用技术（考纲内）

  • FDM（频分复用）：各用户占不同频带，需防护带；模拟与带通信号常用。
  • TDM（时分复用）：等时隙轮转；统计时分（STDM）按需动态分配时隙。
  • WDM（波分复用）：光纤场景的“频分”，以波长划分通道，密集波分DWDM承载大容量干线。
  • CDM/CDMA（码分复用/多址，扩展）：各用户用正交伪随机码同时占用同频同时时隙；提升抗干扰与保密性（此点为扩展）。

八、常见线路编码对比（考纲内+扩展）

  • 要点维度：规则；是否自同步；是否有直流分量；频谱/带宽占用；典型应用。
  • NRZ-L：
    • 规则：电平值代表比特；
    • 自同步：否（长0/长1风险）；
    • 直流分量：有；
    • 带宽：较低；
    • 应用：早期链路，常配合上层同步机制（考纲内）。
  • NRZI：
    • 规则：0致跳变/1不变（USB2.0），亦有相反约定；
    • 自同步：部分改善（遇0产生边沿）；
    • 直流分量：较小于NRZ-L（视数据）；
    • 带宽：中等；
    • 应用：USB、部分串行链路（考纲内）。
  • 曼彻斯特：
    • 规则：每码元中间必跳变，边沿携带时钟；
    • 自同步：是；
    • 直流分量：低；
    • 带宽：约为2Rb2R_b2Rb​；
    • 应用：10BASE-T（考纲内）。
  • 差分曼彻斯特：
    • 规则：中间必跳变作时钟，起始是否跳变表示数据；
    • 自同步：是；
    • 直流分量：低；
    • 带宽：约为2Rb2R_b2Rb​；
    • 应用：Token Ring（扩展）。
  • AMI/HDB3（扩展）：
    • 规则：1用正负交替脉冲表示，0为零电平；
    • 特点：抑制直流分量、便于定时恢复；
    • 应用：E1/T载波等电信系统。
  • 4B/5B、8B/10B、MLT-3（扩展）：
    • 目的：引入足够边沿与控制直流分量；
    • 应用：Fast/Gigabit Ethernet、PCIe等。

九、易混概念对比（考纲内）

  • 波特率 vs 比特率：RsR_sRs​为每秒码元，Rb=Rslog⁡2MR_b=R_s\log_2 MRb​=Rs​log2​M为每秒比特。
  • 带宽(Hz) vs 容量(b/s)：前者是频率范围W，后者是RbR_bRb​上限（受奈氏/香农约束）。
  • 基带传输 vs 带通信号：是否直接传未调制基带；带通需调制、可用FDM/WDM。
  • 电路交换 vs 报文/分组交换：是否预建专用通路、是否存储转发、是否有序。
  • 数据报 vs 虚电路：前者逐分组独立选路，后者按VC号转发，有序。

十、典型小例题（考纲内）

  • 例1（奈氏+香农联合判定）：
    • 已知：W=3W=3W=3 MHz，SNRdB=30\text{SNR}_{\text{dB}}=30SNRdB​=30 dB，二进制（M=2M=2M=2）。
    • 奈氏上限：Rs≤2W=6R_s\le 2W=6Rs​≤2W=6 Mbaud，Rb≤6R_b\le 6Rb​≤6 Mbps。
    • 香农上限：S/N=1030/10=1000S/N=10^{30/10}=1000S/N=1030/10=1000，Rb≤Wlog⁡2(1+S/N)=3×106⋅log⁡2(1001)≈3×106⋅9.97≈29.9R_b\le W\log_2(1+S/N)=3\times10^6\cdot\log_2(1001)\approx 3\times10^6\cdot9.97\approx 29.9Rb​≤Wlog2​(1+S/N)=3×106⋅log2​(1001)≈3×106⋅9.97≈29.9 Mbps。
    • 结论：取较小者，Rbmax⁡≈6R_b^{\max}\approx 6Rbmax​≈6 Mbps。
  • 例2（时延与带宽-时延积）：
    • 已知：Rb=100R_b=100Rb​=100 Mb/s，单程距离 d=2000\,d=2000d=2000 km，光纤v≈2×108v\approx2\times10^8v≈2×108 m/s，分组L=1L=1L=1 MB。
    • 发送时延：tsend=L/Rb=8×106/108=0.08t_{send}=L/R_b=8\times10^6/10^8=0.08tsend​=L/Rb​=8×106/108=0.08 s。
    • 传播时延：tprop=d/v=2×106/2×108=0.01t_{prop}=d/v=2\times10^6/2\times10^8=0.01tprop​=d/v=2×106/2×108=0.01 s。
    • 管内数据量（单向RTT≈0.02 s）：BDP=Rb⋅RTT=108⋅0.02=2×106\text{BDP}=R_b\cdot \text{RTT}=10^8\cdot0.02=2\times10^6BDP=Rb​⋅RTT=108⋅0.02=2×106 bit≈0.25 MB。
    • 结论：发送时延为主，路径中同时“在途”的数据约0.25 MB。
