数据结构：查找

一、查找的基本概念

  • 查找 (Search)

    • 定义：在数据集合中寻找满足某种条件的数据元素的过程，结果分为“查找成功”（找到满足条件的元素）和“查找失败”（未找到）。
    • 核心目标：以尽可能少的关键字比较次数完成查找，衡量指标为平均查找长度（ASL）。

  • 查找表 (Search Table)

    • 定义：用于查找的数据集合，由同一类型的数据元素（或记录）组成。
    • 核心操作：
      • 查询操作：查找符合条件的数据元素。
      • 动态操作：插入、删除数据元素（仅动态查找表支持）。

  • 静态查找表与动态查找表

    • 静态查找表：
      • 定义：仅支持查找操作，无需动态修改的查找表。
      • 适用查找方法：顺序查找、折半查找、散列查找（部分场景）。
    • 动态查找表：
      • 定义：需支持动态插入、删除操作的查找表。
      • 适用查找方法：二叉排序树查找、散列查找、平衡二叉树查找。

  • 关键字 (Key)

    • 定义：数据元素中唯一标识该元素的某个数据项的值，基于关键字的查找结果唯一（如学生“学号”）。

  • 平均查找长度 (Average Search Length, ASL)

    • 定义：所有查找过程中关键字比较次数的平均值，是衡量查找算法效率的核心指标。
    • 数学公式：ASL=∑i=1nPiCiASL = \sum_{i=1}^{n} P_i C_iASL=∑i=1n​Pi​Ci​
      • nnn：查找表长度；
      • PiP_iPi​：查找第iii个元素的概率（默认等概率时Pi=1/nP_i = 1/nPi​=1/n）；
      • CiC_iCi​：找到第iii个元素所需的关键字比较次数。
    • 分类计算：
      • 查找成功ASL：仅统计查找成功场景的比较次数平均值。
      • 查找失败ASL：统计查找目标不存在时的比较次数平均值。

二、线性结构查找

  • 1. 顺序查找 (Linear Search)

    • 定义：从线性表一端开始，逐个检查关键字是否满足条件，直至找到目标或遍历结束。

    • 适用场景：顺序存储结构、链式存储结构（链表仅支持顺序查找）；无序表或有序表均适用。

    • 核心优化：哨兵机制

      • 原理：将待查找关键字存入查找表的“哨兵位”（如顺序表下标0），避免循环中判断数组越界，提升效率。

      • 查找失败：ASL失败=n+1ASL_{失败} = n+1ASL失败​=n+1，时间复杂度O(n)O(n)O(n)。

    • 缺点：nnn较大时ASL大，效率低。

  • 2. 折半查找 (Binary Search)

    • 定义：仅适用于有序顺序表，通过反复将查找范围减半，定位目标元素的查找方法。

    • 适用条件：

      • 存储结构：顺序存储（需随机存取，排除链表）。
      • 元素特性：关键字有序排列（升序或降序）。

    • 算法步骤（升序表为例）：

      1. 初始化查找范围：low = 0（表首），high = TableLen - 1（表尾）。
      2. 计算中间位置：mid = (low + high) // 2（向下取整，需保持统一取整规则）。
      3. 比较判断：
        • 若key == ST.elem[mid]：查找成功，返回mid。
        • 若key < ST.elem[mid]：目标在左半区，更新high = mid - 1。
        • 若key > ST.elem[mid]：目标在右半区，更新low = mid + 1。
      4. 循环直至low > high：查找失败，返回0。

    • 缺点：结构依赖插入顺序，易退化为单支树，性能下降。

  • 2. 平衡二叉树 (AVL树)

    • 定义：空树或满足以下性质的二叉排序树：
      1. 左右子树均为平衡二叉树；
      2. 任意节点的平衡因子（左子树高度 - 右子树高度）绝对值 ≤ 1（平衡因子取值：-1, 0, 1）。
    • 核心目标：避免BST退化为单支树，保证查找效率稳定在O(log⁡n)O(\log n)O(logn)。
    • 最小不平衡子树：
      • 定义：插入/删除后，第一个平衡因子绝对值 > 1的节点为根的子树（调整对象仅需此子树）。
    • 插入调整：4种旋转策略（基于插入位置与最小不平衡子树的关系）：
      1. LL旋转（右单旋）：插入节点在最小不平衡子树根的左孩子的左子树。
        • 步骤：将左孩子提升为新根，原根降为新根右孩子，原左孩子右子树变为原根左子树。
      2. RR旋转（左单旋）：插入节点在最小不平衡子树根的右孩子的右子树。
        • 步骤：将右孩子提升为新根，原根降为新根左孩子，原右孩子左子树变为原根右子树。
      3. LR旋转（先左后右双旋）：插入节点在最小不平衡子树根的左孩子的右子树。
        • 步骤：先对左孩子做RR旋转，再对原根做LL旋转。
      4. RL旋转（先右后左双旋）：插入节点在最小不平衡子树根的右孩子的左子树。
        • 步骤：先对右孩子做LL旋转，再对原根做RR旋转。
    • 删除调整：
      • 步骤：1. 按BST删除节点；2. 向上回溯找最小不平衡子树；3. 按插入调整的4种策略修复平衡；4. 若调整后子树高度减1，需递归检查祖先节点。
    • 复杂度分析：
      • 含nnn个节点的AVL树最大深度O(log⁡n)O(\log n)O(logn)，查找/插入/删除时间复杂度均为O(log⁡n)O(\log n)O(logn)。
      • 最小节点数公式：nh=nh−1+nh−2+1n_h = n_{h-1} + n_{h-2} + 1nh​=nh−1​+nh−2​+1（n0=0,n1=1,n2=2n_0=0, n_1=1, n_2=2n0​=0,n1​=1,n2​=2）。
    • 优点：查找效率稳定，无最坏情况性能骤降。
    • 缺点：插入/删除需频繁调整旋转，维护成本高。

  • 3. 红黑树 (Red-Black Tree)

    • 定义：满足以下5条性质的二叉排序树（通过“颜色”维护平衡，非严格高度平衡）：
      1. 每个节点为红色或黑色；
      2. 根节点为黑色；
      3. 叶节点（虚构NULL节点）为黑色；
      4. 无连续红节点（红节点的父/子均为黑色）；
      5. 任意节点到其所有叶节点的路径中，黑节点数相同（黑高bhbhbh）。
    • 核心性质：
      • 最长路径长度 ≤ 2×最短路径长度（红黑相间路径 vs 全黑路径）。
      • 含nnn个内部节点的红黑树高度h≤2log⁡2(n+1)h ≤ 2\log_2(n+1)h≤2log2​(n+1)。
    • 核心操作：
      • 插入：
        • 新节点初始为红色（避免破坏性质5），插入后通过“着色”或“旋转”修复性质4。
        • 调整场景（基于父节点与叔节点颜色）：
          1. 叔节点为红色：父/叔节点改黑色，爷节点改红色，向上回溯。
          2. 叔节点为黑色（分LR/LL/RR/RL子场景）：通过旋转+着色修复。
      • 删除：
        • 先按BST删除节点，若删除黑节点，需通过“双黑节点”（替代节点额外加一层黑色）修复性质5，调整逻辑类似插入（更复杂）。
    • 复杂度分析：
      • 查找/插入/删除时间复杂度均为O(log⁡n)O(\log n)O(logn)，调整操作（旋转/着色）为常数次。
    • 优点：维护成本低于AVL树，动态操作更高效，实际应用广泛（如C++ map、Java TreeMap）。
    • 缺点：理论理解复杂，实现难度高。

四、B树与B+树（多路平衡查找树）

  • 1. m阶B树

    • 定义：空树或满足以下性质的m叉树（用于外存查找，减少磁盘I/O）：
      1. 每个节点至多mmm棵子树（至多m−1m-1m−1个关键字）；
      2. 根节点非叶时，至少2棵子树（至少1个关键字）；
      3. 非根非叶节点至少⌈m/2⌉\lceil m/2 \rceil⌈m/2⌉棵子树（至少⌈m/2⌉−1\lceil m/2 \rceil - 1⌈m/2⌉−1个关键字）；
      4. 节点内关键字有序（K1<K2<...<KnK_1 < K_2 < ... < K_nK1​<K2​<...<Kn​），子树PiP_iPi​关键字范围：P0P_0P0​（<K1）、PiP_iPi​（Ki<x<Ki+1K_i < x < K_{i+1}Ki​<x<Ki+1​）、PnP_nPn​（>K_n）；
      5. 所有叶节点在同一层次（无信息，为失败节点）。
    • 核心操作：
      • 查找：先在节点内顺序/折半查找关键字，再按子树指针向下查找，直至叶节点（失败）或找到目标（成功）。
      • 插入：
        • 定位：找到终端节点（最底层非叶节点）插入关键字。
        • 分裂：若节点关键字数 > m−1m-1m−1，从中间⌈m/2⌉\lceil m/2 \rceil⌈m/2⌉处分裂为左右两节点，中间关键字提升至父节点；若父节点也溢出，递归分裂，直至根节点（树高+1）。
      • 删除：
        • 定位：若关键字在非终端节点，用中序后继/前驱替代，转化为删除终端节点关键字。
        • 调整：
          1. 直接删除：节点关键字数 > ⌈m/2⌉−1\lceil m/2 \rceil - 1⌈m/2⌉−1，无需后续操作。
          2. 兄弟借位：兄弟节点关键字数 ≥ ⌈m/2⌉\lceil m/2 \rceil⌈m/2⌉，父节点关键字下移，兄弟节点关键字上移。
          3. 节点合并：兄弟节点关键字数均为⌈m/2⌉−1\lceil m/2 \rceil - 1⌈m/2⌉−1，删除关键字后与兄弟节点、父节点中间关键字合并；若父节点关键字数为0，递归合并（树高-1）。
    • 复杂度分析：
      • 树高hhh：最小h=log⁡m(n+1)h = \log_m(n+1)h=logm​(n+1)（节点关键字数最大），最大h=log⁡⌈m/2⌉((n+1)/2)+1h = \log_{\lceil m/2 \rceil}((n+1)/2) + 1h=log⌈m/2⌉​((n+1)/2)+1（节点关键字数最小）。
      • 查找时间取决于磁盘I/O次数（即树高hhh），效率远高于二叉树（减少I/O）。
    • 适用场景：数据库索引、文件系统索引（外存存储）。

  • 2. m阶B+树

    • 定义：B树的变形，专为数据库设计，满足以下性质：

      1. 分支节点至多mmm棵子树，关键字数 = 子树数（B树为关键字数 = 子树数 - 1）；
      2. 非根分支节点至少⌈m/2⌉\lceil m/2 \rceil⌈m/2⌉棵子树，关键字数 ≥ ⌈m/2⌉\lceil m/2 \rceil⌈m/2⌉；
      3. 叶节点含全部关键字及记录指针，按关键字有序，且相邻叶节点通过指针链接（支持顺序查找）；
      4. 非叶节点仅存子树最大关键字（索引作用，无记录指针）。

    • 与m阶B树的核心差异：

      对比维度 B树 B+树
      关键字与子树数 关键字数 = 子树数 - 1 关键字数 = 子树数
      关键字存储 非叶节点关键字不重复 非叶节点关键字为叶节点副本
      查找终点 非叶节点找到即可返回 必须找到叶节点（统一路径长度）
      顺序查找 不支持 支持（叶节点链表）
      记录指针 非叶节点含记录指针 仅叶节点含记录指针

    • 优点：

      • 支持顺序查找（叶节点链表）和随机查找（多路索引）。
      • 非叶节点仅存关键字，单个节点可存储更多索引项，减少磁盘I/O。

    • 适用场景：关系数据库主键索引、聚簇索引（主流数据库索引方案）。

五、散列（Hash）表

  • 1. 散列基本概念

    • 散列函数 (Hash Function)：将关键字映射到存储地址的函数，记为Hash(key)=AddrHash(key) = AddrHash(key)=Addr（如数组下标）。
    • 冲突 (Collision)：不同关键字映射到同一地址（这些关键字称为“同义词”）。
    • 散列表 (Hash Table)：基于散列函数构建的直接访问数据结构，理想查找时间复杂度O(1)O(1)O(1)。

  • 2. 散列函数构造方法

    • 直接定址法：
      • 公式：H(key)=keyH(key) = keyH(key)=key 或 H(key)=a×key+bH(key) = a \times key + bH(key)=a×key+b（a,ba,ba,b为常数）。
      • 优点：无冲突，计算简单；缺点：关键字分布不连续时空间浪费大。
    • 除留余数法（最常用）：
      • 公式：H(key)=key%pH(key) = key \% pH(key)=key%p（ppp为不大于表长mmm的质数）。
      • 核心：选好ppp，使关键字均匀分布，减少冲突。
    • 数字分析法：
      • 原理：选取关键字中数码分布均匀的若干位作为地址（适用于已知关键字集合）。
    • 平方取中法：
      • 原理：取关键字平方值的中间几位作为地址，利用关键字各位信息，分布较均匀。

  • 3. 冲突处理方法

    • 1. 开放定址法：空闲地址向同义词和非同义词开放，递推公式：Hi=(H(key)+di)%mH_i = (H(key) + d_i) \% mHi​=(H(key)+di​)%m（i=1,2,...,m−1i=1,2,...,m-1i=1,2,...,m−1，did_idi​为增量序列）。

      • 线性探测法：di=1,2,...,m−1d_i = 1,2,...,m-1di​=1,2,...,m−1。
        • 优点：实现简单；缺点：易产生“堆积”（非同义词争夺地址），ASL增大。
      • 平方探测法：di=±12,±22,...,±k2d_i = \pm1^2, \pm2^2, ..., \pm k^2di​=±12,±22,...,±k2（k≤m/2k ≤ m/2k≤m/2，mmm需为4k+34k+34k+3型质数）。
        • 优点：避免堆积；缺点：无法探测所有地址（仅能探测一半）。
      • 双散列法：di=i×Hash2(key)d_i = i \times Hash_2(key)di​=i×Hash2​(key)（Hash2(key)Hash_2(key)Hash2​(key)为第二个散列函数）。
        • 优点：冲突概率低；缺点：需设计两个散列函数，实现复杂。
      • 伪随机序列法：通过伪随机数生成器生成增量序列，避免线性探测的堆积问题。
        • 优点：分布均匀，冲突概率低；缺点：需存储随机数表，增加空间开销。
      • 注意：开放定址法不能物理删除节点（会截断查找路径），需做“逻辑删除”（标记删除位）。

    • 2. 拉链法（链接法）：

      • 原理：将同义词存储在同一链表中，散列表存储各链表头指针。
      • 步骤：1. 计算关键字散列地址iii；2. 将关键字节点插入地址iii对应的链表（头插或尾插）。
      • 示例（关键字序列{19,14,23,01,68}，H(key)=key%13H(key)=key\%13H(key)=key%13）：
        • 地址1链表：14 → 01 → 27 → 79；
        • 地址5链表：55 → 68。
      • 优点：无堆积，插入/删除方便（仅操作链表），空间利用率高。
      • 缺点：需额外存储指针，缓存利用率低（链表节点分散）。

  • 4. 散列表性能分析

    • 核心影响因素：
      1. 散列函数：决定初始地址分布，好的函数可减少冲突。
      2. 冲突处理方法：拉链法性能优于开放定址法（无堆积）。

      • 拉链法：
        • 查找成功：ASL成功=1+α2ASL_{成功} = 1 + \frac{\alpha}{2}ASL成功​=1+2α​（α\alphaα为链表平均长度）。
        • 查找失败：ASL失败=αASL_{失败} = \alphaASL失败​=α（平均链表长度）。

    • 缺点：冲突不可完全避免，α\alphaα过大时性能下降明显。

数据结构：查找 核心考点与解题策略

一、核心考点速览 (Key Concepts)

  • 查找基本概念：
    • 查找结果：查找成功（找到满足条件元素）、查找失败（未找到）。
    • 查找表分类：静态查找表（仅查，如顺序/折半查找）、动态查找表（需插/删，如BST/AVL/B树）。
    • 关键字：唯一标识元素的数据项（如学号）。
    • 平均查找长度（ASL）：衡量效率的核心指标，公式为 ASL=∑i=1nPiCiASL = \sum_{i=1}^{n} P_i C_iASL=∑i=1n​Pi​Ci​（PiP_iPi​为查找第iii个元素的概率，CiC_iCi​为比较次数）。
  • 顺序查找：
    • 适用存储：顺序存储、链式存储（链表仅支持顺序查找）。
    • 哨兵机制：将待查关键字存入表首/尾，避免循环越界判断，提升效率。
    • ASL（等概率）：成功ASL成功=n+12ASL_{成功} = \frac{n+1}{2}ASL成功​=2n+1​，失败ASL失败=n+1ASL_{失败} = n+1ASL失败​=n+1。
  • 折半查找：
    • 适用条件：有序顺序表（需随机存取，排除链表）。
    • 判定树：平衡二叉树，成功比较次数=节点层数，失败比较次数=父节点层数。
    • 核心公式：树高h=⌈log⁡2(n+1)⌉h = \lceil \log_2(n+1) \rceilh=⌈log2​(n+1)⌉，成功ASL≈log⁡2(n+1)−1ASL \approx \log_2(n+1) - 1ASL≈log2​(n+1)−1，时间复杂度O(log⁡n)O(\log n)O(logn)。
  • 分块查找：
    • 结构：索引表（块内最大关键字+块首地址，有序）+ 查找表（块内无序、块间有序）。
    • ASL：ASL=LI+LSASL = L_I + L_SASL=LI​+LS​（LIL_ILI​为索引查找ASL，LSL_SLS​为块内查找ASL），最优块长s=ns = \sqrt{n}s=n​时ASL最优=n+1ASL_{最优} = \sqrt{n} + 1ASL最优​=n​+1。
  • 树形查找：
    • 二叉排序树（BST）：中序遍历有序，最坏ASLO(n)O(n)O(n)（单支树），理想O(log⁡n)O(\log n)O(logn)。
    • 平衡二叉树（AVL）：平衡因子∈{-1,0,1}，4种旋转（LL/RR/LR/RL），最小节点数nh=nh−1+nh−2+1n_h = n_{h-1} + n_{h-2} + 1nh​=nh−1​+nh−2​+1。
    • 红黑树：5条性质（根黑、无连续红节点、黑高一致），高度h≤2log⁡2(n+1)h ≤ 2\log_2(n+1)h≤2log2​(n+1)，维护成本低于AVL。
  • B树与B+树（m阶）：
    • B树：节点关键字数∈[⌈m/2⌉-1, m-1]，叶节点同层，插入超界分裂（中间关键字上移），删除不足合并/借位。
    • B+树：叶节点含全部关键字（链表链接，支持顺序查找），非叶节点仅存索引，磁盘I/O效率更高。
  • 散列表：
    • 散列函数：常用除留余数法（H(key)=key%pH(key) = key \% pH(key)=key%p，ppp为不大于表长的质数）。
    • 冲突处理：开放定址法（线性/平方/双散列，忌物理删除）、拉链法（无堆积，支持动态插删）。
    • 装填因子：α=nm\alpha = \frac{n}{m}α=mn​（nnn为关键字数，mmm为表长），α\alphaα越大冲突概率越高。

二、常见题型分析 (Question Types)

  1. 题型一：平均查找长度（ASL）计算

    • 识别特征：给定查找方法（顺序/折半/分块/散列）和查找表（或关键字序列），要求计算查找成功/失败的ASL。
    • 考察能力：对查找算法逻辑的理解，细致的比较次数统计与数学计算能力。

  2. 题型二：查找判定树构造

    • 识别特征：① 给定有序表，构造折半查找判定树；② 给定关键字序列，构造二叉排序树（BST）。
    • 考察能力：对查找算法执行流程的可视化转化能力，理解“中间位置”“左小右大”的核心逻辑。

  3. 题型三：查找算法正确性判断

    • 识别特征：① 判断某关键字序列是否为折半/BST查找序列；② 判断某结构是否为AVL树/B树/红黑树。
    • 考察能力：对查找算法规则、数据结构性质的精准理解，排除不符合规则的选项。

  4. 题型四：B树插入/删除后的结构确定

    • 识别特征：给定m阶B树和关键字，要求画出插入/删除后的B树结构（需处理分裂/合并）。
    • 考察能力：对B树节点关键字数范围、分裂/合并逻辑的应用能力，步骤化操作思维。

  5. 题型五：散列表构造与冲突处理

    • 识别特征：给定关键字序列、散列函数（如除留余数法）和冲突处理方法（开放定址/拉链法），要求构造散列表并计算ASL。
    • 考察能力：散列函数应用、冲突处理逻辑执行、ASL计算的综合能力。

针对 [平均查找长度（ASL）计算——折半查找为例] 的解题步骤：

  1. 第一步：准备工作（确定基础信息）
    • 提取题干中的有序表，确定表长nnn（如有序表{7,10,13,16,19,29,32,33,37,41,43}，n=11n=11n=11）。
    • 明确计算目标：查找成功ASL或查找失败ASL（题干未说明时需同时计算）。
  2. 第二步：构造折半查找判定树

    • 查找失败节点：在判定树的非叶节点间隙添加方形节点，其层数=父节点层数+1（失败比较次数=父节点层数）。

  1. 第一步：确定核心参数
    • 提取关键字序列（如{19,14,23,01,68,20,84,27,55,11,10,79}）。
    • 明确散列函数（如H(key)=key%13H(key) = key \% 13H(key)=key%13）和表长mmm（通常mmm与散列函数参数匹配，此处m=13m=13m=13）。
  2. 第二步：逐关键字构造散列表
    • 对每个关键字keykeykey，计算初始地址H0=H(key)H0 = H(key)H0=H(key)。
    • 若地址H0H0H0对应的链表为空，直接将关键字作为链表头节点；若不为空，将关键字节点插入链表尾部（或头部，题干未说明时选尾部，保证顺序）。

  3. 第三步：计算ASL

    • 查找失败ASL：统计每个地址对应的链表长度（空地址比较1次，非空地址比较链表长度+1次），总次数除以表长mmm。

针对 [B树插入后的结构确定——5阶B树为例] 的解题步骤：

  1. 第一步：明确B树规则
    • 5阶B树：节点关键字数范围∈[⌈5/2⌉-1, 5-1]即[2,4]，子树数=关键字数+1。
    • 插入规则：插入到终端节点，若关键字数>4则分裂，中间关键字（第3个）上移至父节点。
  2. 第二步：定位插入的终端节点
    • 从根节点开始，根据关键字大小比较，确定待插入关键字所在的终端节点（最底层非叶节点）。
      示例：插入关键字42到5阶B树，根节点关键字为22，42>22，进入右子树；右子树关键字为36,45，36<42<45，插入到该终端节点。
  3. 第三步：处理节点分裂（若需）
    • 插入后检查节点关键字数：若≤4，插入完成；若>4（如插入后节点关键字为36,42,45,50,56，共5个），执行分裂：
      1. 取中间关键字（第3个，45），上移至父节点。
      2. 原节点分裂为左节点（36,42）和右节点（50,56），成为父节点45的左、右子树。
    • 若父节点插入后关键字数>4，递归执行分裂，直至根节点（根节点分裂则树高+1）。
  4. 第四步：画出最终B树结构
    • 按层级标注各节点的关键字和子树指针，确保所有节点关键字数符合范围，叶节点在同一层次。

四、易错点与避坑指南 (Common Pitfalls)

  • 折半查找的存储结构混淆：误认为支持链式存储，实际仅适用于顺序存储（需通过下标随机访问中间元素，链表无法实现）。
  • ASL计算漏算失败场景：仅关注查找成功ASL，忽略题干要求的“查找失败ASL”（如折半查找需计算n+1n+1n+1个失败节点的比较次数）。
  • B树阶数与关键字数关系错误：混淆“m阶B树”的关键字数上限，误记为“m个”，实际为“m-1个”（子树数=m，关键字数=子树数-1）。
  • 散列开放定址法物理删除：直接删除节点导致后续同义词查找路径中断，正确做法是“逻辑删除”（标记删除位，不释放空间）。
  • 二叉排序树ASL依赖插入顺序：相同关键字序列，不同插入顺序会生成不同结构的BST，ASL可能差异极大（如有序插入生成单支树，ASL=O(n)）。
  • AVL树旋转场景判断错误：混淆LR/RL双旋转的顺序（LR需先对左孩子RR旋转，再对根LL旋转；RL相反），导致调整后树失衡。
  • B+树与B树的关键字存储差异：误记“B+树非叶节点关键字不重复”，实际非叶节点关键字是叶节点的副本，叶节点才存储全部不重复关键字。
