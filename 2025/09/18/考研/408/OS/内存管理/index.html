操作系统：内存管理

内存管理基础

  • 内存管理的基本原理和要求

    • 定义：操作系统对内存空间进行合理划分和有效的动态分配，是操作系统设计中最重要、最复杂的内容之一，用于支持多道程序并发执行，解决内存容量有限与进程内存需求的矛盾。
    • 主要功能：
      • 内存空间的分配与回收：记录内存空闲空间和分配情况，回收已结束进程的内存。
    • 地址转换：将进程的逻辑地址转换为物理地址，需硬件（如内存管理单元(Memory Management Unit, MMU)）支持。
      • 内存空间的扩充：利用虚拟存储技术从逻辑上扩充内存容量。
      • 内存共享：允许多个进程访问内存同一部分（如可重入代码），需受控访问机制。
      • 存储保护：保证进程在各自存储空间运行，互不干扰，需硬件寄存器（如上下限寄存器、重定位寄存器）配合。

  • 逻辑地址与物理地址

    • 逻辑地址（虚拟地址）：
      • 定义：编译后目标模块从0号单元编址的相对地址，链接后形成统一的逻辑地址空间（32位系统范围为0 232−10~2^{32}-10 232−1），进程运行时使用的地址，不同进程可存在相同逻辑地址。
      • 特点：用户程序和程序员仅需关注逻辑地址，内存管理机制对用户透明。
    • 物理地址：
      • 定义：内存中物理单元的集合，是地址转换的最终地址，进程执行指令和访问数据需通过物理地址存取。
    • 地址重定位：
      • 定义：装入程序将可执行代码装入内存时，将逻辑地址转换为物理地址的过程，分为静态重定位和动态重定位。
      • 实现依赖：操作系统通过内存管理部件（MMU）和页表（由操作系统维护、处理器引用）完成映射。

  • 程序的链接与装入

    • 程序执行的三步流程：
      1. 编译：编译程序将用户源代码编译为若干目标模块。
      2. 链接：链接程序将目标模块及所需库函数链接为完整装入模块，解决相对地址修改和外部调用符号变换问题。
      3. 装入：装入程序将装入模块装入内存运行，分为三种装入方式。
    • 装入方式：
      • 绝对装入：
        • 定义：仅适用于单道程序环境，编译时已知程序内存位置，生成绝对地址目标代码，装入时无需修改地址。
        • 特点：逻辑地址与物理地址完全相同，依赖编译时确定内存位置。
      • 可重定位装入（静态重定位）：
        • 定义：装入模块始址从0开始，装入时根据内存情况分配位置，一次性完成相对地址到物理地址的修改（重定位）。
        • 特点：需分配连续内存，装入后进程不可移动、不可再申请内存，无动态调整能力。
      • 动态运行时装入（动态重定位）：
        • 定义：装入时不转换地址，推迟至程序执行时进行，需重定位寄存器（存放装入模块始址）支持。
        • 澄清：在仅靠“基址/限长”模型下，进程装入区仍需连续；实现非连续分配需与分页/分段（或段页式）结合。
        • 优点：运行前可仅装入部分代码；执行期间支持紧凑/移动；与分页/分段结合时便于共享。
    • 链接方式：
      • 静态链接：运行前将目标模块及库函数链接为完整装入模块，后续不拆开。
      • 装入时动态链接：装入内存时边装入边链接，便于修改更新和目标模块共享。
      • 运行时动态链接：执行中需要目标模块时才链接，未用到的模块不调入内存，节省空间、加快装入。

  • 进程的内存映像

    • 定义：程序调入内存运行时形成的进程内存结构，区别于硬盘上的可执行文件。

    • 结构/组成：

      • 代码段：只读的二进制代码，可被多个进程共享（如共享库函数），包含.init（初始化函数）、.text（机器代码）、.rodata（只读数据）。
      • 数据段：存放全局变量和静态变量，包含.data（已初始化）、.bss（未初始化或初始化为0）。
        进程控制块(process control block, PCB)：存于内核的系统区（非进程用户地址空间内），操作系统通过PCB控制管理进程。
      • 堆：动态分配变量区域，通过malloc/free函数向高地址扩展/收缩。
      • 栈：实现函数调用，从用户空间最大地址向低地址增长，调用函数时增长、返回时收缩。

    • 定义：确保每个进程拥有独立内存空间，保护操作系统和用户进程不受非法访问。

    • 实现方法：

    • 上下限寄存器：存放用户进程内存的下限和上限地址，中央处理器(Central Processing Unit, CPU)访问地址时与寄存器值比较判断越界。

      • 重定位寄存器（基地址寄存器）+ 界地址寄存器（限长寄存器）：
        • 重定位寄存器：存放进程起始物理地址，用于“加”运算（逻辑地址+基地址=物理地址）。
        • 界地址寄存器：存放进程最大逻辑地址，用于“比”运算（逻辑地址≤限长则合法）。

    • 硬件依赖：加载重定位寄存器和界地址寄存器需特权指令，仅操作系统内核可操作。

  • 内存共享

    • 定义：允许多个进程访问内存同一部分。可重入代码必须以只读方式共享；此外，操作系统还提供读写共享（如共享内存、mmap的MAP_SHARED），由权限与同步机制保障一致性。
    • 可重入代码（纯代码）：允许多个进程同时访问但不允许修改的代码，进程执行时需配套局部数据区（修改私有数据，不改变共享代码）。
    • 实现示例：40个用户同时执行160KB代码区+40KB数据区的文本编辑程序，共享代码区后仅需40KB×40+160KB=1760KB40KB×40+160KB=1760KB40KB×40+160KB=1760KB内存（原需8000KB）。
    • 分页 vs 分段共享：
      • 分页共享：每个进程页表需建立共享代码区所有页面的页表项（如160KB代码区占40个4KB页面，每个进程页表需40个项指向共享物理页）。
      • 分段共享：仅需为共享段设置一个段表项（指向共享段始址和段长），实现更简单。

  • 内存分配与回收

    • 发展历程：单一连续分配→固定分区分配→动态分区分配→离散分配（分页、分段），后一种方式解决前一种的不足。
    • 核心目标：提高内存利用率，适应多道程序环境下不同进程的内存需求。

连续分配管理方式

  • 定义：为用户程序分配连续的内存空间，如需100MB则分配一块连续100MB区域。

  • 单一连续分配

    • 结构/组成：内存分为系统区（低地址，供操作系统使用）和用户区（仅一道用户程序独占）。
    • 优点：实现简单；无外部碎片。
    • 缺点：仅适用于单用户、单任务操作系统；存在内部碎片；内存利用率极低。

  • 固定分区分配

    • 定义：将用户内存空间划分为若干固定大小分区，每个分区仅装入一道作业，通过分区使用表管理（记录始址、大小、状态）。
    • 分区划分方式：
      • 分区大小相等：灵活性差，程序过小浪费空间、过大无法装入。
      • 分区大小不等：包含多个小分区、适量中等分区、少量大分区，适应不同程序大小。
    • 分配/回收流程：
      • 分配：检索分区使用表，找到满足大小且未分配的分区，置状态为“已分配”。
      • 回收：将对应分区状态置为“未分配”。
    • 优点：实现简单，支持多道程序。
    • 缺点：存在内部碎片（程序小于分区大小时浪费空间）；无法共享主存区；内存利用率低。

  • 动态分区分配

    • 定义：根据进程实际需求动态分配内存，分区大小和数量可变，通过空闲分区链（表）管理。
    • 核心问题：外部碎片（内存中分散的小空闲块，无法满足大进程需求），可通过紧凑技术（移动进程整理空闲块）解决（需动态重定位支持，耗时）。
    • 分配算法（基于顺序搜索）：
      • 首次适应（First Fit）：
        • 定义：空闲分区按地址递增排序，分配时顺序查找第一个满足大小的分区。
        • 优点：保留高地址大空闲分区，利于后续大作业；开销小，回收无需重新排序。
        • 缺点：低地址产生大量小碎片，增加查找开销。
      • 邻近适应（Next Fit）：
        • 定义：从上次查找结束位置继续查找，循环搜索。
        • 缺点：高低地址空闲分区被同等分配为小分区，无高地址大空闲分区，性能通常比首次适应差。
      • 最佳适应（Best Fit）：
        • 定义：空闲分区按容量递增排序，分配时选择最小满足需求的分区。
        • 优点：更多留下大空闲分区。
        • 缺点：产生最多外部碎片（大量难以利用的小空闲块），性能差。
      • 最坏适应（Worst Fit）：
        • 定义：空闲分区按容量递减排序，分配时选择最大满足需求的分区，分割后剩余部分仍为空闲。
        • 缺点：快速耗尽大空闲分区，后续无大分区可用，性能差。
    • 分配算法（基于索引搜索）：
      • 快速适应算法：按进程常用空间大小分类，为每类空闲分区设独立链表，通过索引表管理，查找效率高；但仍会产生外部碎片，且回收时合并复杂、开销大。
      • 伙伴系统：
        • 定义：分区大小为2k2^k2k（k为正整数），分配时先找对应大小空闲分区，无则拆分更大分区（拆分出的两个分区为“伙伴”），回收时合并相邻伙伴分区。
        • 优点：分配和回收效率高，碎片少。
      • 哈希算法：建立以空闲分区大小为关键字的哈希表，每个表项记录对应链表头指针，分配时通过哈希函数快速定位链表。
    • 内存回收流程：根据回收分区始址查找空闲分区链，分四种情况处理：
      1. 无相邻空闲分区：新建表项插入链中，空闲区数+1。
      2. 仅与前一空闲分区相邻：合并为一个分区，修改前一分区大小，空闲区数不变。
      3. 仅与后一空闲分区相邻：合并为一个分区，修改后一分区始址和大小，空闲区数不变。
      4. 与前后空闲分区都相邻：合并为一个分区，修改前一分区大小，删除后一分区表项，空闲区数-1。

基本分页存储管理

  • 定义：将内存划分为固定大小的页框（物理块），进程逻辑地址空间划分为与页框大小相等的页面，以页框为单位分配内存，解决连续分配的碎片问题。

  • 分页的基本概念

    • 页面和页面大小：
      • 页面（逻辑）：进程逻辑地址空间的分区，编号为页号（从0开始）。
      • 页框（物理）：内存空间的分区，编号为页框号（从0开始）。
      • 页面大小：需为222的整数次幂，过大会增加页内碎片（每个进程平均产生半个页框大小的内部碎片），过小会导致页表过长、地址转换开销大（通常为4KB、8KB等）。
    • 地址结构：
      • 组成：逻辑地址=页号P + 页内偏移量W，如32位地址中，0~11位为页内偏移（页面大小212B=4KB2^{12}B=4KB212B=4KB），12~31位为页号（最多2202^{20}220个页面）。
      • 计算：页号P=A/LP=A/LP=A/L（A为逻辑地址，L为页面大小，整数除法），页内偏移量W=A%LW=A\%LW=A%L（取余）。
    • 页表：
      • 定义：为每个进程建立的页面映射表，记录页面与页框的对应关系。
    • 结构/组成：每个页面对应一个页表项(page table entry, PTE)，包含页号和物理块号（部分实现中页号隐含，不占存储空间）。
      • 作用：实现从页号到物理块号的地址映射，进程执行时通过页表查找物理块号。

  • 基本地址变换机构

    • 核心组件：页表寄存器(Page Table Register, PTR)，存放页表在内存的始址F和页表长度M（单CPU系统仅设一个，进程未执行时，页表始址和长度存于PCB，调度时装入PTR）。
    • 地址变换步骤：
      1. 计算页号P=A/LP=A/LP=A/L、页内偏移量W=A%LW=A\%LW=A%L。
      2. 越界检查：若P≥MP≥MP≥M，产生越界中断；否则继续。
      3. 查找页表项：页表项地址=F + P×页表项长度，取出物理块号b。
      4. 计算物理地址：E=b×L+WE=b×L + WE=b×L+W，用E访存。
    • 特点：地址变换由硬件自动完成，页式管理地址空间为一维（仅需一个整数表示地址）。
    • 页表项大小约束：需满足容纳物理块号，如32位地址、4KB页面，需20位表示物理块号，理论下限为3B（按字节编址，⌈20/8⌉=3\lceil20/8\rceil=3⌈20/8⌉=3）。实际主流实现通常为4B或8B，以容纳有效/脏/访问/权限等标志并满足对齐。

  • 具有快表的地址变换机构

    • 快表(Translation Lookaside Buffer, TLB)：高速缓冲存储器（相联存储器），存放当前访问的若干页表项，加速地址变换（主存页表称为慢表）。
    • 地址变换步骤：
      1. CPU给出逻辑地址，硬件将页号与快表中所有页号比较。
      2. 命中：直接取物理块号，与W拼接成物理地址，仅1次访存。
      3. 未命中：访问内存页表获取物理块号，拼接成物理地址（2次访存），同时将该页表项存入快表（快表满时按算法淘汰旧项）。
    • 性能分析：快表命中率通常≥90%，分页速度损失≤10%，依赖局部性原理。

  • 两级页表

    • 背景：32位系统中，4KB页面、4B页表项，进程页表需2202^{20}220个页表项（占4MB），且需连续内存，不切实际。
    • 解决思路：对页表离散分配，建立外层页表（页目录）索引内层页表，仅调入当前需用的页表项（虚拟内存思想）。
    • 地址结构：如32位地址分为“一级页号（10位）+ 二级页号（10位）+ 页内偏移（12位）”，外层页表占4KB（1K个页表项，每个4B）。
    • 地址变换步骤：
      1. 外层页表寄存器存放页目录始址，用一级页号索引页目录，获取二级页表始址。
      2. 用二级页号索引二级页表，获取物理块号。
      3. 物理块号+页内偏移=物理地址，共3次访存。
    • 适用场景：大逻辑地址空间（如64位系统需多级页表），避免页表占用连续大内存。

基本分段存储管理

  • 定义：从用户视角出发，将进程逻辑地址空间划分为大小不等的段（如主程序段、子程序段、栈段、数据段），段内连续、段间不连续，地址空间为二维（需段号和段内偏移）。

  • 分段的基本概念

    • 段：
      • 定义：进程逻辑地址空间的逻辑单位，包含一组意义完整的信息，段号从0开始，段长由程序逻辑决定。
      • 地址结构：逻辑地址=段号S + 段内偏移量W，如32位地址中，16位段号（最多216=655362^{16}=65536216=65536段）、16位段内偏移（最大段长216B=64KB2^{16}B=64KB216B=64KB）。
      • 用户可见性：段号和段内偏移需用户显式提供（编译程序完成），区别于分页（对用户透明）。
    • 段表：
      • 定义：为每个进程建立的段映射表，记录段与内存区的对应关系。
    • 结构/组成：每个段对应一个段表项，包含段号（隐含）、段长C（该段可访问的字节数/最大偏移+1）、段基址b（该段在内存的起始物理地址）。
      • 作用：实现从逻辑段到物理内存区的映射，进程执行时通过段表查找段的物理位置。

  • 地址变换机构

    • 核心组件：段表寄存器，存放段表始址F和段表长度M（进程未执行时，段表信息存于PCB）。
    • 地址变换步骤：
      1. 从逻辑地址A中提取段号S和段内偏移量W。
      2. 一级越界检查：若S≥MS≥MS≥M，产生越界中断；否则继续。
      3. 查找段表项：段表项地址=F + S×段表项长度，取出段长C和段基址b。
      4. 二级越界检查：若W≥CW≥CW≥C，产生越界中断；否则继续。
      5. 计算物理地址：E=b+WE=b + WE=b+W，用E访存。
    • 段表项大小约束：需容纳段长和段基址，如32位物理地址、16位段长，段表项最少占16+32=4816+32=4816+32=48位（6B）。

  • 分页与分段的对比

    对比维度 分页管理 分段管理
    单位性质 物理单位（提高内存利用率） 逻辑单位（满足用户需求）
    大小 固定（系统决定） 不固定（用户程序决定）
    地址空间 一维（单个整数表示） 二维（段号+段内偏移）
    对用户透明性 透明（硬件实现） 可见（需显式提供段信息）
    碎片类型 内部碎片（页内） 外部碎片（段间）

  • 段的共享与保护

    • 段的共享：
      • 实现：配置共享段表，记录共享段的段号、段长、内存始址、共享进程计数count（仅当count=0时回收），不同进程可对共享段使用不同段号。
      • 优势：比分页共享简单，仅需为共享段设置一个段表项（分页需为每个共享页面设置页表项）。
    • 段的保护：
      • 存取控制保护：限制进程对段的访问权限（如读、写、执行）。
      • 地址越界保护：一级检查段号（S≥段表长度），二级检查段内偏移（W≥段长），分页仅需检查页号（页内偏移不会越界）。

段页式存储管理

  • 定义：结合分页（提高内存利用率）和分段（满足用户逻辑需求）的优点，先将进程划分为段，再将每段划分为固定大小的页面，内存按页框分配。

  • 核心概念与结构

    • 逻辑地址结构：逻辑地址=段号S + 页号P + 页内偏移量W，如32位地址中，段号（8位）、页号（10位）、页内偏移（12位）。
    • 数据结构：
      • 段表：每个进程1张，每个段对应一个段表项，包含段号（隐含）、页表长度、页表始址。
      • 页表：每个段1张，每个页面对应一个页表项，包含页号（隐含）、物理块号。
      • 段表寄存器：存放段表始址和段表长度，用于段表寻址和越界检查。

  • 地址变换机构

    • 地址变换步骤：
      1. 从逻辑地址A中提取段号S、页号P、页内偏移W。
      2. 段号越界检查：若S≥S≥S≥段表长度M，产生越界中断；否则继续。
      3. 查找段表项：段表项地址=段表始址F + S×段表项长度，获取该段的页表始址和页表长度。
      4. 页号越界检查：若P≥P≥P≥页表长度，产生越界中断；否则继续。
      5. 查找页表项：页表项地址=页表始址 + P×页表项长度，获取物理块号b。
      6. 计算物理地址：E=b×L+WE=b×L + WE=b×L+W（L为页面大小），用E访存。
    • 访存次数：无快表时需3次访存（段表→页表→物理内存），有快表时（关键字为段号+页号）可减少至1次访存。
    • 地址空间维度：二维（需段号和页号，分页对用户透明，仅需显式提供段信息）。

虚拟内存管理

  • 虚拟内存的基本概念

    • 传统存储管理的缺陷：一次性（作业需全部装入内存才能运行）、驻留性（作业装入后一直驻留内存），导致大作业无法运行、多道程序并发度低。
    • 局部性原理：
      • 时间局部性：程序指令或数据被访问后，短期内可能再次访问（如循环操作）。
      • 空间局部性：程序访问某存储单元后，短期内可能访问其附近单元（如指令顺序执行、数组存储）。
      • 作用：虚拟内存技术的核心依据，实现“内存-外存”两级存储的高速缓存效果。
    • 虚拟存储器的定义：通过请求调页（段）和页面（段）置换，仅将当前需用的页面（段）装入内存，从逻辑上扩充内存容量的“存储器”，实际不存在，对用户透明。
    • 核心特征：
      • 多次性：作业可分多次调入内存，无需一次性装入。
      • 对换性：作业运行中，暂不使用的页面（段）可换出到外存，需用时再换入。
    • 虚拟性：逻辑内存容量远大于物理内存，取决于架构的虚拟地址位数（如32位系统虚拟地址空间232B=4GB2^{32}B=4GB232B=4GB）；物理地址总线宽度决定可寻址物理内存上限。
    • 实现方式：基于离散分配，包括请求分页、请求分段、请求段页式管理，需硬件支持（内存/外存、页表/段表机制、中断机构、地址变换机构）。

  • 请求分页管理方式

    • 定义：在基本分页基础上，增加请求调页和页面置换功能，仅装入当前需用页面，缺页时调入，内存满时置换。
    • 页表机制：请求页表项在基本页表项基础上增加4个字段：
      • 状态位P：标记页面是否在内存（1=在内存，0=在外存）。
      • 访问字段A：记录页面访问次数或最近未访问时间，供置换算法使用。
      • 修改位M：标记页面是否被修改（1=修改，0=未修改），换出时需判断是否写回外存。
      • 外存地址：记录页面在外存的存放地址，供调入时使用。
    • 缺页中断机构：
      • 触发条件：访问页面不在内存（P=0）时产生。
      • 处理流程：
    1. 保护中央处理器(Central Processing Unit, CPU)环境，进程阻塞（放入阻塞队列）。
      2. 检查内存是否有空闲页框：
      - 有空闲页框：分配页框，从外存调入缺页，更新页表（P=1、物理块号、A=1）。
      - 无空闲页框：用页面置换算法选择淘汰页，若淘汰页M=1则写回外存，再调入缺页。
    2. 唤醒进程（放回就绪队列），恢复中央处理器(Central Processing Unit, CPU)环境，重新执行被中断指令。
    • 特点：指令执行期间产生（内部异常）；一条指令可能产生多次缺页中断。
    • 地址变换机构：在基本分页地址变换基础上，增加缺页中断处理和页面置换，步骤如下：
      1. 检索快表：命中则修改A（写指令还需改M），拼接物理地址；未命中则查页表。
      2. 查页表：页面在内存（P=1）则将页表项写入快表，拼接物理地址；不在内存（P=0）则触发缺页中断。
      3. 缺页处理：调入页面后更新页表和快表，获取物理块号，拼接物理地址。

  • 页框分配与回收

    • 驻留集：分配给进程的页框集合，大小需平衡多道程序并发度和缺页率（驻留集过小→缺页率高；过大→并发度低）。
    • 内存分配策略：
      • 固定分配局部置换：
        • 定义：为每个进程分配固定数量页框，缺页时仅从该进程页框中置换。
        • 缺点：难确定页框数量，过少缺页频繁，过多浪费内存。
      • 可变分配全局置换：
        • 定义：为进程分配初始页框，缺页时从空闲页框队列分配新页框（全局置换，不限制进程页框数）。
        • 优点：动态调整页框数；缺点：盲目增加页框，降低并发度。
      • 可变分配局部置换：
        • 定义：为进程分配初始页框，缺页时仅从该进程页框中置换；缺页频繁则增加页框，缺页率低则减少页框。
        • 优点：平衡进程调页和系统并发度，性能最优。
    • 物理块调入算法：
      • 平均分配：空闲物理块平均分配给进程。
      • 按比例分配：根据进程大小（页面数）按比例分配。
      • 优先权分配：为高优先级进程分配更多物理块（分两部分：按比例+按优先级）。
    • 调入页面的时机：
      • 预调页策略：根据局部性原理，提前调入相邻页面（首次调入时用，预测成功率约50%）。
      • 请求调页策略：运行中需访问页面不在内存时调入（必被访问，实现简单，主流方式）。
    • 从何处调入页面：
      • 外存分为文件区（离散分配，慢）和对换区（连续分配，快），调入优先级：对换区（已换出页面）→文件区（未运行页面、共享页面）。
    • 页框回收：
      • 可回收页框：多数用户页框（代码段、数据段、堆、栈）可回收；内核也存在可回收页（如页缓存等）与不可回收页（被固定的pinned页、关键常驻页）。
      • 回收算法：页面缓冲算法（设置空闲页面链表和修改页面链表，批量写回外存，减少I/O开销）；Linux中用kswapd守护进程定期回收，用伙伴算法合并连续页框。

  • 页面置换算法

    • 核心目标：选择“最合适”的页面淘汰，降低缺页率，减少磁盘I/O开销。
    • 最佳(Optimal, OPT)置换算法：
      • 定义：淘汰“未来永不使用”或“最长时间内不再使用”的页面，缺页率最低。
      • 优点：理论最优，可用于评价其他算法。
      • 缺点：无法实现（无法预测未来访问）。
      • 示例：页面引用串7,0,1,2,0,3,0,4,2,3,0,3,2,1,2,0,1,7,0,1，3个物理块，缺页次数9次，置换次数6次。
    • 先进先出(First-In, First-Out, FIFO)置换算法：
      • 定义：淘汰最早进入内存的页面，按调入顺序形成队列，换出队头页面。
      • 优点：实现简单。
      • 缺点：未利用局部性原理，性能差；可能产生Belady异常（物理块数增加，缺页次数反而增加）。
      • 示例：同上引用串，3个物理块，缺页次数12次，置换次数9次；Belady异常示例：引用串3,2,1,0,3,2,4,3,2,1,0,4，3个物理块缺页9次，4个物理块缺页10次。
    • 最近最久未使用(Least Recently Used, LRU)置换算法：
      • 定义：淘汰“最近最长时间未使用”的页面，认为过去未访问的页面未来也难访问。
      • 优点：符合局部性原理，性能接近OPT。
      • 缺点：需硬件支持（记录页面访问时间），实现开销大（如用寄存器、栈或计数器）。
      • 示例：同上引用串，3个物理块，缺页次数10次，置换次数7次。
    • 时钟(Clock, CLOCK)置换算法：
      • 简单CLOCK算法（Second-Chance）：
        • 定义：为页面设访问位A（1=访问过，0=未访问），页面连成循环队列，置换指针指向上次换出页面的下一页；选择A=0的页面淘汰，A=1则置0并继续查找。
        • 优点：实现简单，开销低。
        • 缺点：仅用访问位，精度低。
      • 改进型CLOCK算法：
        • 定义：结合访问位A和修改位M，优先淘汰置换代价低的页面，分为4类（优先级：(0,0)>(0,1)>(1,0)>(1,1)）。
        • 算法步骤：
          1. 第一轮扫描：找A=0且M=0的页面，找到则淘汰。
          2. 第二轮扫描：找A=0且M=1的页面，找到则淘汰（扫描中置所有A=0）。
          3. 若前两轮失败，重置指针和A，重复1-2轮。
        • 优点：减少磁盘I/O（优先淘汰未修改页面），性能优于简单CLOCK。

  • 抖动和工作集

    • 抖动（颠簸）：
      • 定义：频繁的页面换入换出，进程大部分时间用于页面调度，CPU利用率趋近于0。
      • 根本原因：进程驻留集过小，无法满足基本内存需求，缺页频繁。
      • 解决措施：撤销部分进程（减少内存竞争）、增大驻留集（分配更多页框）。
    • 工作集：
      • 定义：某段时间间隔（工作集窗口Δ）内，进程需访问的页面集合，反映进程短期内的访问局部性。
      • 作用：驻留集大小需≥工作集大小，否则缺页频繁；示例：窗口Δ=5，访问序列…2,3,5,1,2…，t1时刻工作集{2,3,5}，t2时刻{1,2,3,4}。

  • 内存映射文件（Memory-Mapped Files）

    • 定义：操作系统提供的系统调用，在磁盘文件与进程虚拟地址空间间建立映射，进程可按访问内存方式读/写文件（对用户透明）。
    • 核心机制：
      • 映射时不实际装入文件，访问页面时才按需调入（缺页时从文件读入）。
      • 进程退出或关闭映射时，修改后的页面批量写回磁盘。
    • 优点：
    • 编程简单：无需文件输入/输出(Input/Output, I/O)操作，直接访存。
      • 便于共享：多进程映射同一文件，通过页表映射到相同物理内存，实现进程通信。

  • 虚拟存储器性能影响因素

    • 缺页率：核心影响因素，由以下因素决定：
      • 页面大小：页面过大→页内碎片多；过小→缺页率高，需平衡（通常4KB~8KB）。
      • 驻留集大小：驻留集越大，缺页率越低（超过阈值后改善不明显）。
      • 页面置换算法：LRU、改进型CLOCK算法缺页率低，FIFO缺页率高。
      • 程序局部性：局部性越好，缺页率越低（如按行遍历数组比按列遍历好）。
    • 磁盘I/O效率：
      • 写回频率：批量写回修改页面（如用修改页面链表），减少磁盘I/O次数。
      • 预调页策略：合理预调相邻页面，减少请求调页次数。
    • 地址变换速度：
      • 快表命中率：命中率越高，地址变换越快（如增大快表容量、提高页面大小）。
      • 多级页表优化：减少页表访存次数（如用快表缓存多级页表项）。

408高频计算与概念速记（新增）

  • 有效访问时间（EAT）：
    • 有效访问时间(Effective Access Time, EAT)。
    • 单级页表近似：EAT=α (tTLB+tmem)+(1−α) (tTLB+2 tmem)EAT = \alpha\,(t_{TLB}+t_{mem}) + (1-\alpha)\,(t_{TLB}+2\,t_{mem})EAT=α(tTLB​+tmem​)+(1−α)(tTLB​+2tmem​)（未命中需访存页表+数据）。多级页表将“2”替换为“层数+1”。
    • 若考虑命中直接取物理地址：可简化为EAT≈α thit+(1−α) tmissEAT \approx \alpha\,t_{hit} + (1-\alpha)\,t_{miss}EAT≈αthit​+(1−α)tmiss​，按题设代入。
  • 页表大小估算：
    • 页面大小L=2bL=2^bL=2b，虚拟地址位宽VVV，页数N=2V−bN=2^{V-b}N=2V−b。页表项大小SSS（字节）。单级页表大小=N×S=N\times S=N×S。示例：32位、4KB页、4B页表项(page table entry, PTE)→220×4=42^{20}\times4=4220×4=4MB。
    • 多级页表：逐级按表项数×项大小累计，常见32位二级（10/10/12）顶层4KB。
  • 栈算法判定：
    • 定义：分配更多页框时，驻留集合具包含性（k帧子集于k+1帧），必不出现Belady异常。OPT、LRU为栈算法；FIFO非栈算法。
  • 工作集与PFF控制：
    • 工作集窗口Δ\DeltaΔ内唯一页集合即工作集，驻留集应≥工作集大小以避免抖动。
    • PFF（缺页频率）法：设上下阈值，缺页频率高于上阈→增帧，低于下阈→减帧，以自适应调节。

虚拟内存管理

  • 地址翻译与Cache/TLB综合示例（结合计组）
    • 背景：统考趋势下的学科综合考点，结合高速缓存(Cache)说明虚实地址转换过程，需配合硬件（TLB、数据缓存data cache）实现。
    • 系统条件（文档示例）：
      • 虚拟地址14位，物理地址12位，页面大小64B（页内偏移6位）。
      • TLB（快表）：四路组相联，16个条目（4组，每组4个条目），虚拟页号8位（高6位为TLB标记，低2位为TLB组索引）。
      • data Cache：物理寻址，直接映射，行大小4B（块偏移2位），16组（组索引4位），标记6位（物理地址高6位）。
    • 地址结构：
      • 虚拟地址：TLB标记（6位） + TLB组索引（2位） + 页内偏移（6位），共14位。
      • 物理地址：Cache标记（6位） + Cache组索引（4位） + 块偏移（2位），共12位（页内偏移6位=Cache组索引4位+块偏移2位）。
    • 地址转换步骤（文档示例：访问虚拟地址0x03d4、0x00f1、0x0229）：
      1. 虚拟地址解析：
        • 将十六进制虚拟地址转为二进制，拆分TLB标记、TLB组索引、页内偏移。
    • 示例：0x03d4（十六进制）=0000001111010100（二进制），TLB标记=0x03（高6位），TLB组索引=3（低2位），页内偏移=0x14（低6位）。
      2. TLB查找：
      • 用TLB组索引定位组，比较组内TLB标记与虚拟地址标记，有效位为1则命中，获取物理页号。
      • 命中示例：0x03d4的TLB组3中存在标记0x03，物理页号0x0d，拼接页内偏移得物理地址0x354。
      • 未命中示例：0x00f1的TLB组3无标记0x00，需访问页表。
      1. 页表查找：
        • 虚拟页号=TLB标记+TLB组索引（8位），查找页表获取物理页号（有效位为1则在内存，为0则缺页）。
        • 示例：0x00f1的虚拟页号0x03，页表有效位1，物理页号0x02，拼接页内偏移得物理地址0x0b1；0x0229的虚拟页号0x08，页表有效位0，产生缺页中断。
      2. Cache查找（物理地址解析后）：
        • 拆分物理地址为Cache标记、Cache组索引、块偏移，定位Cache组，比较标记和有效位，命中则获取数据，未命中则访问内存。
        • 示例：0x354（物理地址）=001101010100（二进制），Cache标记0x0d，组索引5，块偏移0，Cache组5中标记0x0d有效，命中，获取数据36H。
    • 核心逻辑：虚实地址转换顺序为“TLB→页表→物理内存/Cache”，TLB命中减少页表访存，Cache命中减少内存访存，共同提升地址转换效率。

    • 目的差异：
      • 分页：仅为系统管理需求，实现离散分配，提高内存利用率，与用户需求无关。
      • 分段：为满足用户需求（如模块化编程、信息共享/保护、动态链接），段是逻辑单位（如主程序段、数据段）。
    • 长度与地址空间：
      • 分页：页面大小固定（系统决定，如4KB），地址空间一维（单个整数表示，如0x03d4）。
      • 分段：段长不固定（用户程序决定，如数据段长度取决于全局变量多少），地址空间二维（需段号+段内偏移，如(2, 154)）。
    • 碎片与共享：
      • 分页：存在内部碎片（页内未用空间，每个进程平均半个页面），无外部碎片；共享需为每个共享页面建立页表项，实现复杂。
      • 分段：存在外部碎片（段间未用空间，可通过紧凑技术解决），无内部碎片；共享仅需为共享段建立一个段表项，实现简单。
    • 透明性与硬件依赖：
      • 分页：对用户透明（硬件自动完成地址转换），依赖页表寄存器、MMU、快表。
      • 分段：对用户可见（需显式指定段信息），依赖段表寄存器、界地址寄存器（越界检查）。

补充说明（文档核心概念强化）

  • 多级页表的本质：为解决“大页表需连续内存”问题，对页表进行“分页”，形成索引链（如二级页表=页目录+页表），仅需将当前需用的页表分页装入内存，减少内存占用；代价是增加地址转换的访存次数（如二级页表需3次访存，需快表优化）。
  • Belady异常与栈算法：发生Belady异常的是非栈算法（FIFO为典型）；具包含性的栈算法（如OPT、LRU）不出现Belady异常。CLOCK不保证一定不出现异常。
  • 动态重定位的核心优势：是实现内存紧凑、进程移动、离散分配的基础，依赖重定位寄存器（记录进程基地址），地址转换推迟至执行时，使进程可灵活调整内存位置。

进阶补充（考纲外，了解即可）

  • TLB高级机制：地址空间标识(Address Space Identifier, ASID)/进程上下文标识(Process-Context Identifier, PCID)减少上下文切换时TLB冲刷；多核下需TLB shootdown维持一致性。
  • 大页/超页（Huge Page）：用更大页（如2MB/1GB）提升TLB覆盖范围，但增加内部碎片。
  • 写时复制(Copy-On-Write, COW)与按需零页：fork后只读共享，写入时复制；首次触页由零页提供并在写时分配真实页。
  • Linux实现概览：伙伴系统+slab/slub；active/inactive LRU与kswapd回收；可移动/不可移动分区；透明大页(Transparent Huge Pages, THP)。
  • I/O与零拷贝：直接内存访问(Direct Memory Access, DMA)需要pin页；mmap、sendfile、splice等减少拷贝层次。
  • 安全与保护：W^X、不可执行(No-eXecute, NX)/数据执行保护(Data Execution Prevention, DEP)/地址空间布局随机化(Address Space Layout Randomization, ASLR)、内存保护键(Memory Protection Keys, MPK)与内存标记扩展(Memory Tagging Extension, MTE)。
