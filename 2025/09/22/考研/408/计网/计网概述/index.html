第1章 计算机网络体系结构

一、计算机网络概述

  • 计算机网络的概念

    • 定义：将众多分散的、自治的计算机系统，通过通信设备与线路连接起来，由功能完善的软件实现资源共享和信息传递的系统；由若干节点（Node，如计算机、集线器、交换机、路由器）和连接节点的链路（Link）组成，网络间通过路由器互连构成互连网。
    • 关键术语区分：
      • internet（互连网）：通用名词，泛指多个计算机网络互连而成的网络，可使用任意通信协议。
      • Internet（互联网/因特网）：专用名词，指当前全球最大的、开放的特定计算机网络，采用TCP/IP族作为通信规则。

  • 计算机网络的组成

    • 从组成部分看：
      • 硬件：主机（端系统）、通信链路（双绞线、光纤）、交换设备（路由器、交换机）、通信处理机（网卡）。
      • 软件：资源共享软件、用户工具软件（E-mail程序、FTP程序、聊天程序）。
      • 协议：网络核心，规定网络传输数据的遵循规范（类似交通规则）。
    • 从工作方式看（以Internet为例）：
      • 边缘部分：由连接到互联网的主机组成，用于通信（传输数据、音频、视频）和资源共享。
      • 核心部分：由大量网络和连接网络的路由器组成，为边缘部分提供连通性和交换服务。
    • 从功能组成看：
      • 通信子网：由传输介质、通信设备、网络协议组成，实现数据传输、交换、控制、存储，完成联网计算机间的数据通信。
      • 资源子网：实现资源共享的设备及软件集合，提供硬件、软件、数据资源的共享服务。

  • 计算机网络的功能

    • 分布式处理：将负荷过重的计算机的复杂任务分配给其他空闲计算机，提升系统整体利用率。
    • 其他功能：实现电子化办公与服务、远程教育、娱乐，满足社会需求。

  • 三种数据交换方式

    • 电路交换

      • 定义：传统电话网采用的交换方式，按动态分配传输线路资源的方式实现通信，需建立专用物理通信路径。
      • 核心步骤：
        1. 建立连接：占用通信资源，建立端到端专用物理路径。
        2. 传输数据：独占物理路径，比特流连续直达目的节点（类似“管道传输”）。
        3. 释放连接：通信结束后归还通信资源。
      • 优点：
        • 通信时延小（数据直达、速率高）；
        • 有序传输（无失序问题）；
        • 无冲突（各通信方独占信道）；
        • 实时性强（连接建立后可随时通信）。
      • 缺点：
        • 建立连接时间长（对计算机突发通信不友好）；
        • 线路利用率低（空闲时也无法供其他用户使用）；
        • 灵活性差（链路故障需重新建立连接）；
        • 难以实现差错控制（中间节点无存储检验能力）。
      • 适用场景：连续传送大量数据，且数据传输时间远大于连接建立时间的场景。

    • 报文交换

      • 定义：以“报文（Message）”为数据交换单位（用户数据+源地址、目的地址等控制信息），采用“存储-转发”技术，每个报文可独立选择路径。
      • 核心机制：报文先传至相邻节点，全部存储后查找转发表，转发至下一个节点，直至目的节点。
      • 优点：
        • 无建立连接时延（可随时发送报文）。
        • 灵活分配线路（故障时可重新选路）。
        • 线路利用率高（仅在传输时占用链路）。
        • 支持差错控制（交换节点可检验报文差错）。
      • 缺点：
        • 转发时延高（需接收完整报文后才转发）。
        • 缓存开销大（报文大小无限制，需大缓存）。
        • 错误处理低效（报文长，出错概率高，重传代价大）。

    • 分组交换

      • 定义：基于报文交换改进，将长报文分割为若干“分组（Packet）”（通常为变长；少数技术如 ATM（Asynchronous Transfer Mode）采用固定长度信元），每个分组含数据段与首部控制信息（源/目的地址、编号等），采用“存储-转发”技术。
      • 核心机制：源主机发送分组至交换网，交换机接收分组后缓存，提取目的地址查转发表，转发至下一个交换机，最终到达目的主机。
      • 优点：
        • 方便存储管理（相较报文更短，缓冲粒度更细；实现中常需支持变长队列与分片/重组）。
        • 传输效率高（分组逐个传输，后一个分组存储与前一个分组转发并行，流水线减少时延）。
        • 减少出错概率与重传代价（分组短，出错概率低，重传数据量小）。
        • 继承报文交换的无连接、线路利用率高、灵活选路等优点。
      • 缺点：
        • 存储转发时延较大（比电路交换大，需更强节点处理能力）。
        • 传输额外信息量（每个分组需加控制信息，开销增 5%~10%，控制复杂）。
      • 服务模式：
        • 无连接的数据报服务：可能失序/丢失/重复，需目的主机重排/重传。
        • 面向连接的虚电路服务：需建立/数据传输/释放三阶段。
      • 适用场景：突发式数据传送（计算机通信多为此类），整体信道利用率优于电路交换，时延优于报文交换。

  • 计算机网络的分类

    • 按分布范围分类：
      • 广域网（Wide Area Network, WAN）：覆盖范围直径几十至几千千米，为互联网核心部分，连接节点交换机的链路为高速链路，通信容量大。
      • 城域网（Metropolitan Area Network, MAN）：覆盖范围跨越街区至整个城市，直径5~50km，多采用以太网技术，常并入局域网讨论。
      • 局域网（Local Area Network, LAN）：覆盖范围直径几十至几千米，主机通过高速线路连接。现代 LAN 以交换式以太网为主（主机与交换机端口点到点连接，交换机基于媒体访问控制（Media Access Control, MAC）学习与转发表转发），仅在特定帧类型时在广播域内进行广播/泛洪。
      • 个人区域网（Personal Area Network, PAN）：个人工作区域内，用无线技术连接消费电子设备（平板、手机）的网络，也称无线个人区域网（Wireless Personal Area Network, WPAN）。
    • 按传输技术分类：
      • 广播式网络：所有计算机共享公共通信信道，发送分组时所有计算机“收听”，接收方通过目的地址判断是否接收（如局域网、广域网无线/卫星通信）。
      • 点对点网络：每条物理线路连接一对计算机，无直接线路时需中间节点存储转发（如广域网核心部分）。
    • 按拓扑结构分类（指通信子网拓扑）：
      • 总线形网络：
        • 特点：单根传输线连接计算机。
        • 优点：
          • 建网易。
          • 增减节点方便。
          • 节省线路。
        • 缺点：
          • 重负载时通信效率低。
          • 总线故障敏感。
      • 星形网络：
        • 特点：终端/计算机通过单独线路连中央设备（交换机/路由器）。
        • 优点：
          • 便于集中控制管理。
        • 缺点：
          • 成本高。
          • 中央设备故障敏感。
      • 环形网络：计算机接口设备连成环（单环/双环），信号单向传输（如令牌环局域网，Token Ring）。
      • 网状网络：
        • 特点：每个节点至少两条路径连其他节点（多用于广域网），分规则型与非规则型。
        • 优点：
          • 可靠性高。
        • 缺点：
          • 控制复杂。
          • 线路成本高。
    • 按使用者分类：
      • 公用网（Public Network）：电信公司建造，缴费即可使用的大型网络。
      • 专用网（Private Network）：单位为特殊业务建造，不向外部提供服务（如铁路、电力、军队专用网）。
    • 按传输介质分类：
      • 有线网络：采用双绞线、同轴电缆等有线介质（如双绞线网络、同轴电缆网络）。
      • 无线网络：采用无线介质（如蓝牙、微波、无线电网络）。

  • 计算机网络的性能指标

    • 速率（Speed）

      • 定义：节点在数字信道上传送数据的速率，也称数据传输速率、数据率、比特率。
      • 单位：b/s（比特/秒）、kb/s（k=103k=10^3k=103）、Mb/s（M=106M=10^6M=106）、Gb/s（G=109G=10^9G=109）。

    • 带宽（Bandwidth）

      • 定义：原指通信线路允许通过的信号频率范围（单位Hz）；计算机网络中，指信道能传送的“最高数据传输速率”，与速率单位一致（b/s）。

    • 吞吐量（Throughput）

      • 定义：单位时间内通过某个网络（或信道、接口）的实际数据量，用于测量实际网络数据通行能力。

    • 时延（Delay）

      • 定义：数据（报文/分组）从网络一端传至另一端的总时间，由四部分组成。
      • 组成与计算公式：
        • 发送时延（传输时延）：节点将分组所有比特推向链路的时间；公式：发送时延=分组长度/发送速率发送时延 = 分组长度 / 发送速率发送时延=分组长度/发送速率。
        • 传播时延：电磁波在信道中传播一定距离的时间；公式：传播时延=信道长度/电磁波在信道上的传播速率传播时延 = 信道长度 / 电磁波在信道上的传播速率传播时延=信道长度/电磁波在信道上的传播速率。
        • 处理时延：交换节点为存储转发进行必要处理的时间（如分析首部、差错检验、查找路由）。
        • 排队时延：分组在路由器输入/输出队列中等待的时间。
      • 总时延公式：总时延=发送时延+传播时延+处理时延+排队时延总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延总时延=发送时延+传播时延+处理时延+排队时延（考试中通常忽略处理时延和排队时延）。
      • 关键区分：发送时延取决于分组长度和发送速率；传播时延取决于信道长度和传输介质，与分组长度/发送速率无关。

    • 时延带宽积

      • 定义：发送端第一个比特到达终点时，发送端已发出的比特数，也称“以比特为单位的链路长度”。
      • 公式：时延带宽积=传播时延×信道带宽时延带宽积 = 传播时延 × 信道带宽时延带宽积=传播时延×信道带宽（类比“圆柱形管道容量”，长度=传播时延，横截面积=带宽）。考试常结合端到端流水线估算使用 BDP≈RTT×带宽BDP \approx RTT × 带宽BDP≈RTT×带宽。

    • 往返时延（RTT）

      • 定义：发送端发送完短分组到收到接收端确认（接收端收数据后立即发送确认）的总时间，含中间节点处理时延、排队时延、转发发送时延。

    • 信道利用率

      • 定义：信道有数据通过的时间占总时间的比例。
      • 公式：信道利用率=有数据通过的时间/(有数据通过的时间+无数据通过的时间)信道利用率 = 有数据通过的时间 / (有数据通过的时间 + 无数据通过的时间)信道利用率=有数据通过的时间/(有数据通过的时间+无数据通过的时间)。
      • 特点：并非越高越好；利用率过低浪费资源，过高易导致时延增大、网络拥塞（类比公路车流量过大拥堵）。

    • QoS（服务质量，Quality of Service）与业务质量（408常识）

      • 常见指标：时延、抖动（时延变化）、丢包率、带宽/吞吐、误码率（Bit Error Rate, BER）、可用性。
      • 要点：实时业务（语音/视频）对抖动和丢包更敏感；数据业务对带宽与可靠更敏感。

    • 有效吞吐量（Goodput）

      • 定义：单位时间内成功交付给应用的有效负载速率（不计首部、重传）。
      • 提示：Goodput ≤ Throughput，受协议首部、握手、ACK（Acknowledgment，确认）、重传影响。

    • 利用率与排队时延趋势（直觉）

      • 业务强度：ρ≈到达速率/服务速率；当 ρ→1 时，排队时延急剧上升。
      • 解题提醒：识别瓶颈链路；单位统一（b/B, s/ms/μs）；分清发送时延 vs 传播时延 vs RTT；用 BDP 估窗口/流水线。

误区与单位换算速记

  • b 与 B：1B=8b；速率常用 b/s，数据量常用 B。
  • 时延单位：注意 ms 与 μs 换算；链路长度需转成传播时延。
  • BDP 应用：链路一程容量用 传播时延×带宽；端到端窗口估算常用 RTT×带宽。

二、计算机网络体系结构与参考模型

  • 计算机网络分层结构

    • 网络体系结构定义：计算机网络的各层及其协议的集合，是网络功能的精确定义（抽象），与具体硬件/软件实现（具体）无关。
    • 分层基本原则：
      1. 每层实现相对独立功能，降低系统复杂度。
      2. 层间接口清晰，相互交流少。
      3. 层功能定义独立于实现方法，可采用最优技术。
      4. 下层对上层独立，上层单向使用下层服务。
      5. 分层结构促进标准化工作。
    • 关键术语：
      • 实体：第n层活动元素（硬件/软件进程，如特定软件模块），不同机器同一层为“对等层”，对等层实体为“对等实体”。
      • 协议数据单元（Protocol Data Unit, PDU）：对等层间传送的数据单位，第n层PDU记为n-PDU，由“服务数据单元（Service Data Unit, SDU）+协议控制信息（Protocol Control Information, PCI）”组成。
      • 服务数据单元（Service Data Unit, SDU）：层间交换的数据单位，第n层SDU记为n-SDU。
      • 协议控制信息（Protocol Control Information, PCI）：控制协议操作的信息，第n层PCI记为n-PCI。
    • 各层PDU通俗名称：物理层（比特流）、数据链路层（帧）、网络层（分组）、传输层（报文段）。
    • 数据封装关系：n-SDU + n-PCI = n-PDU = (n-1)-SDU（发送方逐层封装，接收方逆向拆封）。
    • 分层结构含义：
      • 第n层实体需使用第n-1层服务实现自身功能，并向第n+1层提供“本层+下层”的综合服务。
      • 最低层仅提供服务（基础），最高层面向用户提供服务。
      • 上层仅通过相邻层接口使用下层服务，不可跨层调用。
      • 对等层逻辑上有直接信道，可“直接”传信息。

  • 协议、接口、服务的概念

    • 协议（Network Protocol）

      • 定义：为对等实体间进行通信而建立的规则、标准或约定（水平方向，仅对等实体间存在协议）。
      • 核心三要素：
        • 语法：数据与控制信息的格式（如TCP报文段格式）。
        • 语义：需发出的控制信息、完成的动作及应答（如TCP三次握手的操作定义）。
        • 同步（时序）：执行操作的条件、时序关系（如TCP四次挥手的报文发送顺序）。

    • 接口（服务访问点，Service Access Point, SAP）

      • 定义：同一节点内相邻两层实体交换信息的逻辑接口，第n层SAP是第n+1层访问第n层服务的“入口”。
      • 特点：仅允许相邻层定义接口，不可跨层定义。

    • 服务（Service）

      • 定义：下层为紧邻上层提供的功能调用（垂直方向），对等实体在协议控制下实现本层服务，需依赖下层服务。
      • 与协议的区别：
        • 服务是“垂直”的（下层向上层提供），协议是“水平”的（对等实体间控制规则）。
        • 上层仅可见服务，不可见下层协议（协议对上层透明）。
        • 仅本层协议实现后，才能向上层提供服务，并非层内所有功能都是服务（仅上层“看得见”的功能是服务）。
      • 服务分类：
        • 按连接方式：
          • 面向连接服务：需建立连接→数据传输→释放连接三阶段（如TCP），分配资源，通信可靠。
          • 无连接服务：无需建立连接，直接发送带目的地址的分组，“尽最大努力交付”，不可靠（如IP、UDP）。
        • 按可靠性：
          • 可靠服务：含纠错、检错、应答机制，保证数据正确交付。
          • 不可靠服务：仅尽力交付，正确性需应用/用户保障。
        • 按应答方式：
          • 有应答服务：接收方收数据后自动回应用答（肯定/否定），由传输系统实现（如文件传输）。
          • 无应答服务：接收方不收数据不自动应答，需应答则由高层实现（如基于 UDP 的简单查询、以太网无确认帧传输）。

  • ISO/OSI参考模型

    • 模型概述：国际标准化组织（International Organization for Standardization, ISO）提出的开放系统互连参考模型（Open Systems Interconnection Reference Model, OSI/RM），共7层（自下而上：物理层→数据链路层→网络层→传输层→会话层→表示层→应用层），是抽象框架，定义层功能与服务，不规定具体实现。

    • 各层核心功能与特点

      • 记忆口诀：物联网，叔会使用（物理层、数据链路层、网络层、传输层、会话层、表示层、应用层）。

      • 物理层（第1层）

        • 传输单位：比特（bit）。
        • 核心功能：在物理介质上透明传输原始比特流（“透明”指接收方收到的比特流与发送方一致）。
        • 主要研究内容：
          • 接口参数：机械形状、尺寸、引脚数量排列（如网线接口）。
          • 信号定义：比特0/1对应的电信号特征（如信号X代表0）。
        • 注意：物理介质（双绞线、光纤）不属于物理层，常视为“第0层”。

      • 数据链路层（第2层）

        • 传输单位：帧（Frame）。
        • 核心功能：将物理层的“可能出错物理连接”改造为“逻辑无差错数据链路”，实现相邻节点间可靠传输。
        • 主要任务：
          • 封装分组为帧（添加首部、尾部）。
          • 差错控制：检测并丢弃错误帧（如因噪声导致的比特翻转）。
          • 流量控制：协调相邻节点发送/接收速率，避免接收方被“淹没”。
          • 广播式网络额外功能：控制共享信道访问（如载波侦听多路访问/碰撞检测，Carrier Sense Multiple Access with Collision Detection, CSMA/CD）。

      • 网络层（第3层）

        • 传输单位：分组（数据报）。
        • 核心功能：负责分组从源主机到目的主机的无连接转发与路由（host-to-host，跨网络），管理通信子网运行。
        • 主要任务：
          • 路由选择：为分组选择合适路径（如基于路由算法选路）。
          • 拥塞控制：缓解网络节点因分组过多导致的拥塞（如丢弃部分分组）。
          • 差错处理：有限差错检测与报告（如 IPv4（Internet Protocol version 4）首部校验、ICMP（Internet Control Message Protocol）差错报告），不保证可靠/无错交付。
          • 网际互联：实现不同网络间的分组传输。
          • 服务类型：提供有连接可靠的虚电路服务、无连接不可靠的数据报服务。

      • 传输层（第4层）

        • 传输单位：报文段（TCP）/用户数据报（UDP）。
        • 核心功能：实现主机中“进程间”的端到端通信，屏蔽通信子网差异。
        • 主要任务：
          • 端到端流量控制、差错控制、连接建立与释放。
          • 复用与分用：多个应用进程共享传输层服务（复用），将接收数据交付对应进程（分用，通过端口号实现）。
        • 与数据链路层区别：数据链路层是“点到点”（相邻节点）通信，传输层是“端到端”（进程间）通信。

      • 会话层（第5层）

        • 核心功能：管理不同主机上进程间的“会话”（建立、维护、终止）。
        • 主要机制：
          • 同步点/会话同步：为表示层/用户进程建立会话并进行同步，保证会话内数据的有序与一致性。
          • 检查点机制：通信中断或失败时，可从最近检查点恢复（如断点续传）。

      • 表示层（第6层）

        • 核心功能：处理不同主机间交换信息的“表示方式”，实现数据格式统一。
        • 主要任务：
          • 数据格式转换：定义抽象标准数据结构，解决不同机器编码差异。
          • 数据压缩/加密：减少数据量，保障数据安全（如文件压缩、SSL（Secure Sockets Layer）加密）。

      • 应用层（第7层）

        • 核心功能：用户与网络的接口，为特定网络应用提供访问OSI环境的手段。
        • 特点：最复杂，协议最多（如HTTP、SMTP、FTP、DNS），直接面向用户需求。

  • TCP/IP模型

    • 模型概述：实际应用中的“事实标准”，TCP/IP（Transmission Control Protocol/Internet Protocol），先有协议栈后建立模型，共4层（自下而上：网络接口层→网际层→传输层→应用层），支持异构网络互联。

    • 各层核心功能与协议

      • 网络接口层（最下层）

        • 对应OSI层：物理层+数据链路层。
        • 核心功能：接收IP分组，通过物理网络发送（如局域网、电话网、ATM网）。
        • 特点：TCP/IP未定义具体协议，仅要求主机使用适配物理网络的协议。
        • 主要任务：成帧与比特传输、链路接入与冲突避免、差错检测（如CRC）、MTU/分片适配。

      • 网际层（核心层）

        • 对应OSI层：网络层。
        • 传输单位：IP数据报。
        • 核心协议：IP（网际协议，Internet Protocol，当前以IPv4为主，IPv6为下一代）。
        • 核心功能：为IP数据报独立选择路由，实现主机到主机的无连接传输（不保证分组有序，可靠性由高层负责）。
        • 主要任务：选路与转发、分片与重组（IPv4）、差错报告（ICMP）、网际互联与地址规划、ARP/邻居解析。

      • 传输层

        • 对应OSI层：传输层。
        • 核心协议：
          • TCP（传输控制协议，Transmission Control Protocol）：面向连接，可靠传输，数据单位为报文段，含三次握手、四次挥手、滑动窗口、超时重传等机制。
          • UDP（用户数据报协议，User Datagram Protocol）：无连接，尽最大努力交付，数据单位为用户数据报，协议开销小，实时性强。
        • 核心功能：实现进程间端到端通信，提供可靠（TCP）/不可靠（UDP）服务。
        • 主要任务（TCP）：连接管理、可靠传输（确认/重传）、流量控制（滑窗）、拥塞控制（慢启动/拥塞避免）。
        • 主要任务（UDP）：无连接复用/分用、保留报文边界、最小协议开销。

      • 应用层（最上层）

        • 对应OSI层：会话层+表示层+应用层。
        • 核心协议：Telnet（虚拟终端）、FTP（文件传输，File Transfer Protocol）、DNS（域名解析，Domain Name System）、SMTP（电子邮件，Simple Mail Transfer Protocol）、HTTP（超文本传输，HyperText Transfer Protocol）。
        • 核心功能：提供各类用户应用服务，直接满足业务需求。
        • 关键特点：“everything over IP”（所有应用基于IP）、“IP over everything”（IP支持所有物理网络）。
        • 主要任务：应用语义与交互流程定义、数据表示/编码（部分由库承担）、会话管理与应用级重传/鉴权。

  • OSI参考模型与TCP/IP模型的比较

    • 相似点：

      1. 均采用分层体系结构，层功能大体相似（如传输层均负责端到端通信）。
      2. 均基于“协议栈”概念，层间通过接口交互。
      3. 均支持异构网络互联，实现不同厂家计算机通信。

    • 差异点：

      1. 概念定义：OSI明确区分“服务、协议、接口”；TCP/IP未明确区分。
      2. 层数与对应关系：OSI 7层，TCP/IP 4层（TCP/IP将OSI表示层+会话层合并到应用层，物理层+数据链路层合并到网络接口层）。
      3. 模型与协议关系：OSI先有模型后有协议（通用性强，适合各类网络）；TCP/IP先有协议后有模型（仅适用于TCP/IP网络）。
      4. 通信模式：
        • OSI：网络层支持无连接+面向连接，传输层仅支持面向连接。
        • TCP/IP：网际层仅支持无连接，传输层支持无连接（UDP）+面向连接（TCP）。

    • 折中体系结构：实际学习中常采用“5层协议体系”（物理层→数据链路层→网络层→传输层→应用层），综合OSI与TCP/IP优点，平衡理论与实践。

对比与速记（高频考点）

  • 虚电路 vs 数据报（网络层服务类型，对应分组交换的两种模式）

    • 连接：虚电路需建立/维护/释放连接；数据报无连接。
    • 路由：虚电路在建链时确定路径，后续分组沿同一路径；数据报每个分组独立选路。
    • 中间状态：虚电路中间节点需保存连接状态；数据报无需保存端到端状态。
    • 顺序与可靠：虚电路天然有序、易于保证 QoS；数据报可能失序/丢失/重复。
    • 故障影响：虚电路路径故障需重建连接；数据报可动态绕行但顺序性更弱。
    • 开销：虚电路有建链开销；数据报首部开销分布在每个分组。
    • 适用：虚电路适合长时会话、稳定带宽需求；数据报适合突发业务和弹性路由。

  • TCP vs UDP（传输层服务）

    • 连接性：TCP 面向连接（三次握手/四次挥手）；UDP 无连接。
    • 可靠性：TCP 可靠（确认、重传、滑窗、拥塞控制）；UDP 尽力而为。
    • 流量/拥塞控制：TCP 具备慢启动、拥塞避免等；UDP 无（应用自控）。
    • 首部开销：TCP ≥20B；UDP 8B。
    • 时延与抖动：TCP 额外时延与抖动更大；UDP 更小，利于实时。
    • 分用：均靠端口号；TCP 有字节流语义，UDP 报文边界保留。
    • 应用：TCP→HTTP/FTP/SMTP/SSH（Secure Shell）；UDP→DNS/视频/语音/在线游戏等。

  • 数据传输过程（协议栈通信）

    • 发送方：应用层数据→传输层（加TCP/UDP首部，形成报文段/用户数据报）→网络层（加IP首部，形成IP数据报）→数据链路层（加帧首部/尾部，形成帧）→物理层（转为比特流）→传输介质。
    • 接收方：物理层（比特流→帧）→数据链路层（拆帧，提取IP数据报）→网络层（拆IP首部，提取报文段/用户数据报）→传输层（拆TCP/UDP首部，提取应用层数据）→应用层（交付用户）。
    • 核心特点：逐层封装（发送方）、逐层拆封（接收方），每层仅处理本层首部信息，对上层透明。

考前速记清单（1页）

  • 交换方式

    • 电路：连接→传输→释放；独占、低时延；低利用率；适合连续长数据。
    • 报文：整报文存储转发；无建链；缓存大，时延高。
    • 分组：分组存储转发；流水线；支持数据报/虚电路。

  • 模型与分层

    • OSI 7 层；TCP/IP 4 层；常用 5 层折中。
    • 传输单位：比特/帧/分组(数据报)/段；仅传输层是端到端进程通信。

  • 网络层服务（高频对比）

    • 虚电路：建链、路径固定、有序、需维护状态、故障需重建。
    • 数据报：无连接、逐包选路、可能失序/丢失/重复、容错弹性好。

  • TCP vs UDP（高频对比）

    • TCP：面向连接、可靠、滑窗+拥塞控制、首部≥20B、字节流。
    • UDP：无连接、尽力而为、首部8B、保留报文边界、低时延。

  • 性能速算

    • 发送时延 = L(bits)/R(bps)；传播时延 = d(m)/v(m/s)。
    • 总时延 ≈ 发送 + 传播（考试常忽略处理/排队）。
    • BDP = 传播时延×带宽；窗口/流水线估算用 RTT×带宽。
    • 利用率↑ → 排队时延急增（ρ→1）。

  • QoS 与 Goodput

    • QoS：时延、抖动、丢包率、BER、可用性；实时对抖动/丢包敏感。
    • Goodput：应用有效负载速率 ≤ Throughput（受首部/重传/ACK影响）。

  • 设备与域

    • Hub：物理层；1个碰撞域；不分广播域。
    • Switch：二层；分割碰撞域；VLAN 分割广播域。
    • Router：三层；隔离广播域；IP转发与选路。

  • 支撑协议

    • ARP：IP→MAC，广播请求/单播响应；同一链路内。
    • ICMP：差错/可达性报告；Ping=回显请求/应答。
    • DHCP/NAT：地址自动分配/转换（了解）。

  • 常见易错

    • 网络层非端到端可靠；TCP 才提供端到端可靠。
    • 分组长度通常变长，固定长是 ATM 信元。
    • WWW 基于 TCP 非“无应答”；无应答示例看 UDP/以太网无确认。
    • 现代 LAN 为交换式，广播仅在广播域内发生。

超纲提示：IPv6 关键点（了解）— 128 位地址、固定40B首部、扩展头、仅源分片、弱化 NAT 需求。

常见支撑协议与典型设备（408高频识别）

  • 支撑协议

    • ARP（地址解析协议，Address Resolution Protocol）：解析同一链路内 IP→MAC；广播请求，单播响应。
    • ICMP（网际控制报文协议，Internet Control Message Protocol）：用于报告差错与网络可达性（如目的不可达、超时）；Ping 使用回显请求/应答。
    • DHCP（动态主机配置协议，Dynamic Host Configuration Protocol）：动态分配 IP/网关/DNS 等参数（注：实现细节可略）。
    • NAT（网络地址转换，Network Address Translation）：私网地址与公网地址转换（注：原理了解，深度实现可略）。

  • 典型设备与层次

    • 集线器（Hub）：物理层，信号转发，1个碰撞域，广播域不分割。
    • 交换机（Switch）：数据链路层，基于 MAC 学习与转发表转发，分割碰撞域；VLAN（虚拟局域网，Virtual LAN）可分割广播域。
    • 路由器（Router/三层交换）：网络层，基于 IP 转发，隔离广播域，执行路由选择。
    • 网关（Gateway）：广义跨层/跨协议转换设备，通常工作在应用层或更高层抽象。

  • 广播域/碰撞域与 VLAN

    • 碰撞域：使用共享媒体时发生冲突的范围；交换机端口彼此分隔碰撞域。
    • 广播域：二层广播可达的范围；VLAN 可进一步划分广播域。

超纲提示：IPv4 vs IPv6 关键差异（了解）

  • 地址长度：32 位→128 位；无类寻址简化为前缀表示。
  • 首部：IPv6 首部固定 40B、字段更少、无首部校验，效率更高。
  • 扩展头：通过扩展头实现功能扩展（如分片、路由、安保），中间设备更易处理。
  • 分片：IPv6 仅源分片；中间路由器不分片。
  • NAT：IPv6 倾向端到端寻址（但实践仍可能存在 NAT66）。

数据链路层 数据链路层的功能 链路管理 定义：负责数据链路连接的建立、维持和释放，主要用于面向连接的服务。 核心任务： 建立连接：确认对方就绪状态，交换信息初始化帧序号。 维持连接：确保传输过程中链路稳定。 释放连接：传输完毕后终止链路。 适用场景：点对点链路接入与链路配置（如PPP）；PPP提供成帧与检错，可靠性通常由上层（如TCP）保证。 封装成帧与透明传输 封装成帧： 定义：在网络层递交的分组前后添加首部和尾部，构成数据链路层的协议数据单元（帧）。 结构：帧 = 首部 + 数据（网络层分组） +...
