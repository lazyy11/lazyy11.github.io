数据结构：树与二叉树

1. 树的基本概念

  • 树的定义

    • 定义：树是 n(n≥0)n(n \geq 0)n(n≥0) 个结点的有限集。当 n=0n=0n=0 时为空树；非空树满足：①有且仅有一个根结点（无前驱）；②其余结点可分为 m(m>0)m(m>0)m(m>0) 个互不相交的有限集（每个集合为根的子树，递归定义）。
    • 核心特性：①分层结构，根无前驱，其余结点仅1个前驱；②每个结点可有0或多个后继；③ nnn 个结点的树有 n−1n-1n−1 条边。

  • 基本术语

    • 亲属关系：
      • 祖先/子孙：从根到某结点路径上的所有结点为其祖先，该结点为祖先的子孙（如结点K的祖先含B、E）。
      • 双亲/孩子：路径中最接近某结点的结点为双亲，该结点为孩子（如E是K的双亲）。
      • 兄弟/堂兄弟：同一双亲的结点为兄弟（K与L）；双亲在同一层的结点为堂兄弟（G与E、F等）。
    • 结点层级相关：
      • 层次：根为第1层，孩子为第2层，以此类推。
      • 深度：结点所在的层次（与层次同义）。
      • 高度：树的高度=最大层数；结点的高度=以该结点为根的子树高度（图5.1树高为4）。
    • 结点度数相关：
      • 结点的度：该结点的孩子个数（如B的度为2，D的度为3）。
      • 树的度：树中结点的最大度数（图5.1树的度为3）。
      • 分支结点/叶结点：度>0的结点为分支结点（非终端结点），度=0的结点为叶结点（终端结点）。
    • 其他关键术语：
      • 有序树/无序树：子树从左到右有次序（不可互换）为有序树，否则为无序树。
      • 路径/路径长度：两结点间的结点序列为路径，路径长度=边的个数（树中路径仅从上向下）。
      • 森林：m(m≥0)m(m \geq 0)m(m≥0) 棵互不相交的树的集合（删树的根得森林，给森林加根得树）。

  • 树的性质（核心公式与结论）

    • 性质1（结点数与度数关系）：树的结点数 nnn = 所有结点的度数之和 + 1。
      推导：度数之和=边数（每个孩子对应一条边），而边数=n−1n-1n−1（除根外每个结点对应一条边），故 n=∑i度数(i)+1n = \sum_{i} \text{度数}(i) + 1n=∑i​度数(i)+1（2010、2016统考考点）。

    • 性质2（m叉树层结点数）：度为 mmm 的树中，第 iii 层至多有 mi−1m^{i-1}mi−1 个结点（i≥1i \geq 1i≥1）。
      示例：第1层（根）至多1个（m0m^0m0），第2层至多 mmm 个（m1m^1m1）。

    • 性质3（m叉树总结点数）：高度为 hhh 的 mmm 叉树至多有 mh−1m−1\frac{m^h - 1}{m - 1}m−1mh−1​ 个结点。
      推导：等比数列求和 1+m+m2+⋯+mh−1=mh−1m−11 + m + m^2 + \dots + m^{h-1} = \frac{m^h - 1}{m - 1}1+m+m2+⋯+mh−1=m−1mh−1​。

    • 性质4（m叉树最小高度）：度为 mmm、含 nnn 个结点的树，最小高度 hmin=⌈log⁡m(n(m−1)+1)⌉h_{\text{min}} = \lceil \log_m(n(m-1) + 1) \rceilhmin​=⌈logm​(n(m−1)+1)⌉。
      推导：前 h−1h-1h−1 层满（结点数 mh−1−1m−1\frac{m^{h-1} - 1}{m - 1}m−1mh−1−1​），满足 mh−1−1m−1<n≤mh−1m−1\frac{m^{h-1} - 1}{m - 1} < n \leq \frac{m^h - 1}{m - 1}m−1mh−1−1​<n≤m−1mh−1​（2022统考考点：三叉树最小高度分析）。

    • 性质5（m叉树最大高度）：度为 mmm、含 nnn 个结点的树，最大高度 hmax=n−m+1h_{\text{max}} = n - m + 1hmax​=n−m+1。
      推导：仅一个结点有 mmm 个孩子（同层），其余层仅1个结点，故高度=1+(n−m)1 + (n - m)1+(n−m)。

2. 二叉树

  • 二叉树的定义与主要特性

    • 定义：n(n≥0)n(n \geq 0)n(n≥0) 个结点的有限集，空树（n=0n=0n=0）或由根、左子树、右子树组成（左、右子树有序，不可颠倒）。
      关键区别：与“度为2的有序树”相比，二叉树可空，且单孩子需区分左/右（度为2的有序树单孩子无需区分）。
    • 特殊二叉树类型：
      • 满二叉树：高度为 hhh，含 2h−12^h - 12h−1 个结点的二叉树（每层结点数最大）。
        性质：叶结点集中在最下层，分支结点度均为2；层序编号（根为1）时，结点 iii 的双亲为 ⌊i/2⌋\lfloor i/2 \rfloor⌊i/2⌋，左孩子 2i2i2i，右孩子 2i+12i+12i+1。
      • 完全二叉树：高度为 hhh、含 nnn 个结点，且每个结点与高度为 hhh 的满二叉树编号 1∼n1 \sim n1∼n 一一对应（2009、2011、2018统考考点）。
        核心性质：①最后一个分支结点编号 ⌊n/2⌋\lfloor n/2 \rfloor⌊n/2⌋；②叶结点仅在最后两层；③度为1的结点至多1个（仅左孩子）；④ nnn 为奇数时所有分支结点有左/右孩子，偶数时最后分支结点（n/2n/2n/2）仅左孩子。
      • 二叉排序树：左子树所有结点关键字 < 根，右子树所有结点关键字 > 根，左右子树亦为二叉排序树（后续7.3节详细讲解）。
      • 平衡二叉树：任意结点左、右子树高度差绝对值 ≤ 1（后续7.3节详细讲解）。
      • 正则二叉树：仅含度为0或2的结点（分支结点均有2个孩子）。
    • 二叉树核心性质：
      • 性质1（叶结点与度2结点关系）：非空二叉树中，叶结点数 n0n_0n0​ = 度为2的结点数 n2+1n_2 + 1n2​+1（高频考点）。
        推导：n=n0+n1+n2n = n_0 + n_1 + n_2n=n0​+n1​+n2​，边数 B=n−1=n1+2n2B = n - 1 = n_1 + 2n_2B=n−1=n1​+2n2​，联立得 n0=n2+1n_0 = n_2 + 1n0​=n2​+1。
      • 性质2（层结点数）：第 kkk 层至多有 2k−12^{k-1}2k−1 个结点（k≥1k \geq 1k≥1）。
      • 性质3（总结点数）：高度为 hhh 的二叉树至多有 2h−12^h - 12h−1 个结点。
      • 性质4（完全二叉树高度）：含 nnn 个结点的完全二叉树高度为 ⌈log⁡2(n+1)⌉\lceil \log_2(n+1) \rceil⌈log2​(n+1)⌉ 或 ⌊log⁡2n⌋+1\lfloor \log_2 n \rfloor + 1⌊log2​n⌋+1。
        推导：2h−1≤n<2h2^{h-1} \leq n < 2^h2h−1≤n<2h → h−1≤log⁡2n<hh-1 \leq \log_2 n < hh−1≤log2​n<h → h=⌊log⁡2n⌋+1h = \lfloor \log_2 n \rfloor + 1h=⌊log2​n⌋+1。

  • 二叉树的存储结构

    • 顺序存储结构：
      • 结构：用连续存储单元按层序存储完全二叉树，结点 iii 存储在数组下标 i−1i-1i−1 处（建议下标从1开始，与编号一致）。
      • 适用场景：完全二叉树、满二叉树（空间利用率高）；一般二叉树需补空结点（最坏情况：高度 hhh 的单支树需 2h−12^h - 12h−1 个单元，利用率低）。
    • 链式存储结构：
      • 二叉链表（最常用）：
        • 结构：每个结点含3个域：数据域（data）、左孩子指针（lchild）、右孩子指针（rchild）。
        • 关键结论：含 nnn 个结点的二叉链表，空指针域数为 n+1n+1n+1（用于线索二叉树）。
      • 三叉链表：在二叉链表基础上增加“双亲指针域（parent）”，便于查找双亲，但增加空间开销。

4. 树、森林

  • 树的存储结构

    • 1. 双亲表示法：
      • 结构：用数组存储结点，每个结点含“数据域”和“双亲下标域”（根的双亲下标为-1）。
      • 优点：快速查找双亲；缺点：查找孩子需遍历整个数组。
      • 存储结构（C语言）：
    • 2. 孩子表示法：
      • 结构：每个结点的孩子构成单链表，nnn 个头指针组成数组（顺序存储）。
      • 优点：快速查找孩子；缺点：查找双亲需遍历所有孩子链表。
    • 3. 孩子兄弟表示法（二叉树表示法）：
      • 结构：每个结点含“数据域”、“第一个孩子指针”、“右兄弟指针”（最灵活，便于树转二叉树）。
      • 优点：便于树转二叉树、查找孩子；缺点：查找双亲需遍历（可加双亲域优化）。
      • 存储结构（C语言）：

  • 树、森林与二叉树的转换（2009、2011、2014、2020、2021统考考点）

    • 1. 树转二叉树：
      • 规则：左孩子右兄弟（每个结点左指针指向第一个孩子，右指针指向相邻右兄弟）。
      • 画法：①兄弟间加连线；②仅保留结点与第一个孩子的连线；③顺时针旋转45°。
      • 结论：树转二叉树后，二叉树无右子树（根无兄弟）。
    • 2. 森林转二叉树：
      • 规则：①每棵树转二叉树；②森林中树的根视为兄弟，第二棵树作为第一棵树的右子树，依次类推。
      • 画法：①每棵树转二叉树；②根间加连线；③以第一棵树根为轴心顺时针旋转45°。
    • 3. 二叉树转森林：
      • 规则：①二叉树根及其左子树为第一棵树；②根的右子树视为剩余森林转二叉树，递归拆分；③每棵二叉树转树。
      • 结论：转换唯一，森林树的棵数=二叉树中根及其右链上的结点数。

  • 树和森林的遍历（2019、2020统考考点）

    • 树的遍历：
      • 先根遍历：①访问根；②依次先根遍历各子树 → 对应二叉树的先序遍历。
      • 后根遍历：①依次后根遍历各子树；②访问根 → 对应二叉树的中序遍历。
      • 层次遍历：按层访问（同二叉树层次遍历）。
      • 示例：图5.22树先根序列ABEFCDG，后根序列EFBCGDA。
    • 森林的遍历：
      • 先序遍历：①访问第一棵树根；②先序遍历第一棵树子树森林；③先序遍历剩余森林 → 对应二叉树的先序遍历。
      • 中序遍历：①中序遍历第一棵树子树森林；②访问第一棵树根；③中序遍历剩余森林 → 对应二叉树的中序遍历。
      • 示例：图5.23森林先序序列ABCDEFGHI，中序序列BCDAFEHIG。

    • 核心考点1（森林转二叉树结点数）：3棵树（结点数M1、M2、M3）的森林转二叉树，根的右子树结点数=M2+M3（选择题03答案：D）。
    • 核心考点2（树与二叉树遍历对应）：树的后根遍历=对应二叉树的中序遍历（2019统考考点，选择题18答案：B）。
    • 核心考点3（森林树的棵数）：二叉树先序a b d c e g f、中序b d a e g c f，森林树的棵数=3（根a、c、f）（2021统考考点，选择题20答案：C）。
    • 综合题解法：孩子兄弟表示法求森林叶结点数 → 递归统计：无孩子（firstchild=NULL）则为叶结点，否则统计孩子子树和兄弟子树的叶结点数。

5. 树与二叉树的应用

  • 哈夫曼树与哈夫曼编码（2010、2012、2014、2017、2018、2020、2021、2022、2023统考考点）

    • 哈夫曼树的定义：
      • 关键概念：
        • 结点的权：结点的数值（如字符出现频次）。
        • 带权路径长度（WPL）：结点权值 × 根到该结点的路径长度；树的WPL=所有叶结点WPL之和。
        • 哈夫曼树（最优二叉树）：含 nnn 个带权叶结点的二叉树中，WPL最小的树。
    • 哈夫曼树的构造算法：
      • 步骤：
        1. 初始化：nnn 个叶结点各为一棵单结点树，构成森林F。
        2. 选最小：从F中选两棵根权值最小的树，作为新结点的左/右子树，新结点权值=两树根权值之和。
        3. 更新森林：将新树加入F，删除原两棵树。
        4. 重复：直至F中仅一棵树。
      • 核心性质：
        • 每个初始叶结点最终为叶结点，权值越小路径越长。
        • 共新建 n−1n-1n−1 个非叶结点（双分支），总结点数= 2n−12n-12n−1。
        • 无度为1的结点。
      • 示例：权值{7,5,2,4}的哈夫曼树WPL=7×1 +5×2 +2×3 +4×3=35。
    • 哈夫曼编码：
      • 前缀编码：无一个编码是另一个编码的前缀（如0、10、110是前缀编码，0、00不是），确保译码唯一。
      • 哈夫曼编码构造：
        1. 以字符频次为权，构造哈夫曼树。
        2. 左分支为0，右分支为1，根到叶结点的路径序列为该字符的编码。
      • 优点：平均编码长度最短，数据压缩效率高（如频次45、13、12、16、9、5的字符，哈夫曼编码WPL=224，比3位定长编码（300位）压缩25%）。
      • 注意：哈夫曼树不唯一（左/右子树可互换），但WPL唯一。

  • 并查集（Disjoint Set）

    • 基本概念：
      • 定义：一种集合表示，支持“初始化（Initial）”“合并（Union）”“查找（Find）”三种操作，用于处理不相交集合的合并与查询。
      • 适用场景： Kruskal算法求最小生成树、判断无向图连通性、处理等价关系。
    • 存储结构：
      • 双亲表示法：用数组存储，下标为元素名，根结点的双亲域为负数（绝对值为集合大小），其他结点的双亲域为父结点下标。
      • 示例：全集合{0~9}初始化时，每个元素双亲为-1；合并后集合{0,6,7,8}的根（0）双亲为-4（大小4）。

      • 3. 合并（Union）：将Root2集合并入Root1（未优化，简单合并）。
