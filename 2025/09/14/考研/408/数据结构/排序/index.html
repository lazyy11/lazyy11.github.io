数据结构：排序算法

排序的基本概念

  • 排序的定义

    • 定义：重新排列表中元素，使表中元素满足按关键字有序的过程。输入为 n 个记录 R1,R2,…,RnR_1,R_2,\dots,R_nR1​,R2​,…,Rn​（对应关键字 k1,k2,…,knk_1,k_2,\dots,k_nk1​,k2​,…,kn​），输出为记录的重排 R1′,R2′,…,Rn′R_1',R_2',\dots,R_n'R1′​,R2′​,…,Rn′​，满足 k1′≤k2′≤⋯≤kn′k_1' \leq k_2' \leq \dots \leq k_n'k1′​≤k2′​≤⋯≤kn′​（“≤”可替换为其他比较符号）。

  • 算法稳定性

    • 定义：若待排序表中存在关键字相同的元素 RiR_iRi​ 和 RjR_jRj​（keyi=keyjkey_i = key_jkeyi​=keyj​），且排序前 RiR_iRi​ 在 RjR_jRj​ 之前，排序后 RiR_iRi​ 仍在 RjR_jRj​ 之前，则称该算法稳定；否则不稳定。
    • 性质/特点：稳定性仅描述算法性质，不衡量算法优劣；若关键字无重复，稳定与否无关紧要。
    • 判断方法：对不稳定算法，只需举一组关键字实例证明其不稳定性即可。

  • 内部排序与外部排序

    • 内部排序：排序期间所有元素均存放在内存中的排序，核心操作是“比较”和“移动”（基数排序除外，不基于比较）。
    • 外部排序：排序期间元素无法全部存入内存，需在内外存间移动数据的排序（如大文件排序）。

  • 性能衡量标准

    • 核心指标：时间复杂度、空间复杂度，其中时间复杂度主要由“比较次数”和“移动次数”决定。
    • 适用存储：多数内部排序更适用于顺序存储的线性表；部分算法（如直接插入、冒泡）也支持链式存储。

插入排序

  • 直接插入排序

    • 定义：每次将待排序记录按关键字大小插入前面已排好序的子序列，直至全部插入完成。
    • 算法步骤：
      1. 初始时，将 L[1]L[1]L[1] 视为已排序子序列（假设表为 L[1..n]L[1..n]L[1..n]）。
      2. 对 i=2i=2i=2 到 nnn，将 L[i]L[i]L[i] 插入 L[1..i−1]L[1..i-1]L[1..i−1]：
        • 查找 L[i]L[i]L[i] 在 L[1..i−1]L[1..i-1]L[1..i−1] 中的插入位置 kkk。
        • 将 L[k..i−1]L[k..i-1]L[k..i−1] 中所有元素后移一位。
        • 将 L[i]L[i]L[i] 复制到 L[k]L[k]L[k]。
    • 复杂度分析：
      • 时间复杂度：最好情况（正序）O(n)O(n)O(n)，最坏情况（逆序）O(n2)O(n^2)O(n2)，平均情况 O(n2)O(n^2)O(n2)。
      • 空间复杂度：O(1)O(1)O(1)（原地排序，仅用常数辅助单元）。
    • 性质/特点：稳定排序；适用于顺序存储和链式存储（链式存储无需移动元素）。

  • 折半插入排序

    • 定义：对直接插入排序的优化——先通过折半查找确定待插入位置，再统一移动元素（分离“比较”和“移动”操作）。
    • 核心改进：利用顺序存储的随机访问特性，用折半查找替代顺序查找，减少比较次数。
    • 复杂度分析：
      • 时间复杂度：比较次数降至 O(nlog⁡2n)O(n\log_2 n)O(nlog2​n)，但移动次数仍为 O(n2)O(n^2)O(n2)，总复杂度仍为 O(n2)O(n^2)O(n2)。
      • 空间复杂度：O(1)O(1)O(1)（同直接插入排序）。
    • 性质/特点：稳定排序；仅适用于顺序存储的线性表。

  • 希尔排序（缩小增量排序）

    • 定义：基于直接插入排序改进，先将待排序表分割成若干“增量子表”（相隔某个增量 ddd 的元素组成），对各子表做直接插入排序；逐步缩小增量，直至 d=1d=1d=1，对全体元素做最后一次直接插入排序。
    • 算法步骤：
      1. 选取增量序列（如 d1=n/2,d2=d1/2,…,dt=1d_1 = n/2, d_2 = d_1/2, \dots, d_t=1d1​=n/2,d2​=d1​/2,…,dt​=1）。
      2. 对每个增量 dkd_kdk​：
        • 将表分为 dkd_kdk​ 个增量子表（L[i,i+dk,i+2dk,… ]L[i, i+d_k, i+2d_k, \dots]L[i,i+dk​,i+2dk​,…]）。
        • 对每个子表执行直接插入排序。
    • 复杂度分析：
      • 时间复杂度：依赖增量序列，平均约 O(n1.3)O(n^{1.3})O(n1.3)，最坏情况 O(n2)O(n^2)O(n2)（无最优增量序列）。
      • 空间复杂度：O(1)O(1)O(1)（原地排序）。
    • 性质/特点：不稳定排序（相同关键字可能被分到不同子表，改变相对次序）；仅适用于顺序存储的线性表。

交换排序

  • 冒泡排序

    • 定义：通过两两比较相邻元素，若为逆序则交换，使关键字最小的元素“漂浮”到前端（或最大元素“下沉”到后端）；每趟确定一个元素的最终位置，最多需 n−1n-1n−1 趟。
    • 算法优化：引入标志位 flag，若某趟无交换则表已有序，直接退出（减少无效趟数）。
    • 算法步骤：
      1. 初始 flag = false（标记是否交换）。
      2. 对 i=0i=0i=0 到 n−2n-2n−2（共 n−1n-1n−1 趟）：
        • 对 j=n−1j=n-1j=n−1 到 i+1i+1i+1，比较 A[j−1]A[j-1]A[j−1] 和 A[j]A[j]A[j]，逆序则交换，设 flag = true。
        • 若 flag = false，退出循环（表已有序）。
    • 复杂度分析：
      • 时间复杂度：最好情况（正序）O(n)O(n)O(n)，最坏情况（逆序）O(n2)O(n^2)O(n2)，平均情况 O(n2)O(n^2)O(n2)。
      • 空间复杂度：O(1)O(1)O(1)（原地排序）。
    • 性质/特点：稳定排序；适用于顺序存储和链式存储；每趟生成的有序子序列是“全局有序”的（有序子序列元素均小于无序子序列元素）。

  • 快速排序

    • 定义：基于分治思想，选一个元素作为枢轴（pivot，通常为首元素），通过一趟划分将表分为两部分：左部分元素均小于枢轴，右部分均大于等于枢轴；枢轴确定最终位置，再递归排序左右子表。
    • 核心操作：划分（Partition）：
      1. 设枢轴 pivot = A[low]，指针 i=low，j=high。
      2. j 从后往前找第一个小于枢轴的元素，赋值给 A[i]。
      3. i 从前往后找第一个大于枢轴的元素，赋值给 A[j]。
      4. 重复步骤 2-3，直至 i=j，将枢轴赋值给 A[i]（枢轴就位）。
    • 复杂度分析：
      • 时间复杂度：最好情况（枢轴均分表）O(nlog⁡2n)O(n\log_2 n)O(nlog2​n)，最坏情况（正序/逆序）O(n2)O(n^2)O(n2)，平均情况 O(nlog⁡2n)O(n\log_2 n)O(nlog2​n)（内部排序中平均性能最优）。
      • 空间复杂度：递归栈开销，最好情况 O(log⁡2n)O(\log_2 n)O(log2​n)，最坏情况 O(n)O(n)O(n)，平均情况 O(log⁡2n)O(\log_2 n)O(log2​n)。
    • 性质/特点：不稳定排序；仅适用于顺序存储的线性表；每趟确定枢轴的最终位置，但不生成有序子序列。

选择排序

  • 简单选择排序

    • 定义：每趟（第 iii 趟）从待排序子表 L[i..n]L[i..n]L[i..n] 中选关键字最小的元素，与 L[i]L[i]L[i] 交换；每趟确定一个元素的最终位置，需 n−1n-1n−1 趟。
    • 算法步骤：
      1. 对 i=0i=0i=0 到 n−2n-2n−2（共 n−1n-1n−1 趟）：
        • 初始化 min = i（记录最小元素位置）。
        • 对 j=i+1j=i+1j=i+1 到 n−1n-1n−1，若 A[j]<A[min]A[j] < A[min]A[j]<A[min]，更新 min = j。
        • 若 min != i，交换 A[i]A[i]A[i] 和 A[min]A[min]A[min]。
    • 复杂度分析：
      • 时间复杂度：比较次数固定为 O(n2)O(n^2)O(n2)（与初始状态无关），移动次数最多 O(n)O(n)O(n)，总复杂度 O(n2)O(n^2)O(n2)。
      • 空间复杂度：O(1)O(1)O(1)（原地排序）。
    • 性质/特点：不稳定排序（交换可能改变相同关键字的相对次序）；适用于顺序存储和链式存储，关键字较少的场景。

  • 堆排序

    • 堆的定义：nnn 个关键字序列 L[1..n]L[1..n]L[1..n] 满足以下条件之一：
      • 大根堆（大顶堆）：L[i]≥L[2i]L[i] \geq L[2i]L[i]≥L[2i] 且 L[i]≥L[2i+1]L[i] \geq L[2i+1]L[i]≥L[2i+1]（根为最大值，子树亦为大根堆）。
      • 小根堆（小顶堆）：L[i]≤L[2i]L[i] \leq L[2i]L[i]≤L[2i] 且 L[i]≤L[2i+1]L[i] \leq L[2i+1]L[i]≤L[2i+1]（根为最小值，子树亦为小根堆）。
      • （注：1≤i≤⌊n/2⌋1 \leq i \leq \lfloor n/2 \rfloor1≤i≤⌊n/2⌋，堆可视为完全二叉树）。
    • 核心操作：
      1. 建堆（BuildMaxHeap）：从最后一个非叶结点（⌊n/2⌋\lfloor n/2 \rfloor⌊n/2⌋）开始，向前对每个结点做“筛选”（调整子树为堆），直至根结点。
      2. 堆调整（HeadAdjust）：堆顶元素输出后，将堆底元素移至堆顶，向下筛选调整堆（保持堆性质）。
    • 堆排序步骤：
      1. 对初始序列建大根堆（或小根堆，取决于排序方向）。
      2. 对 i=ni=ni=n 到 222：
        • 交换堆顶 A[1]A[1]A[1] 和堆底 A[i]A[i]A[i]（堆顶元素就位）。
        • 对 A[1..i−1]A[1..i-1]A[1..i−1] 执行堆调整（保持堆性质）。
    • 复杂度分析：
      • 时间复杂度：建堆 O(n)O(n)O(n)，排序阶段 O(nlog⁡2n)O(n\log_2 n)O(nlog2​n)，总复杂度 O(nlog⁡2n)O(n\log_2 n)O(nlog2​n)（最好、最坏、平均一致）。
      • 空间复杂度：O(1)O(1)O(1)（原地排序）。
    • 性质/特点：不稳定排序；仅适用于顺序存储的线性表；适合关键字较多的场景（如海量数据选前 kkk 小元素）。

归并排序

  • 二路归并排序
    • 定义：基于分治思想，将待排序表视为 nnn 个长度为 1 的有序子表，两两归并为长度为 2 或 1 的有序子表；重复归并，直至合并为一个长度为 nnn 的有序表。
    • 核心操作：合并（Merge）：将两个相邻的有序子表 A[low..mid]A[low..mid]A[low..mid] 和 A[mid+1..high]A[mid+1..high]A[mid+1..high] 合并为一个有序表：
      1. 将 A[low..high]A[low..high]A[low..high] 复制到辅助数组 BBB。
      2. 设指针 i=lowi=lowi=low（左子表）、j=mid+1j=mid+1j=mid+1（右子表）、k=lowk=lowk=low（合并后表）。
      3. 比较 B[i]B[i]B[i] 和 B[j]B[j]B[j]，将较小值存入 A[k]A[k]A[k]，移动对应指针；直至某一子表遍历完。
      4. 将剩余子表的元素直接复制到 A[k..high]A[k..high]A[k..high]。
    • 算法步骤（递归版）：
      1. 分解：若 low<highlow < highlow<high，计算 mid=(low+high)/2mid = (low+high)/2mid=(low+high)/2，将表分为 A[low..mid]A[low..mid]A[low..mid] 和 A[mid+1..high]A[mid+1..high]A[mid+1..high]。
      2. 递归：分别对两个子表执行二路归并排序。
      3. 合并：调用 Merge 合并两个有序子表。
    • 复杂度分析：
      • 时间复杂度：每趟归并 O(n)O(n)O(n)，共需 ⌈log⁡2n⌉\lceil \log_2 n \rceil⌈log2​n⌉ 趟，总复杂度 O(nlog⁡2n)O(n\log_2 n)O(nlog2​n)（最好、最坏、平均一致）。
      • 空间复杂度：O(n)O(n)O(n)（需辅助数组 BBB 存储全部元素）。
    • 性质/特点：稳定排序；适用于顺序存储和链式存储；可并行执行（归并段独立）。

基数排序

  • 基数排序
    • 定义：非比较类排序，基于“多关键字排序”思想，将单关键字拆分为多位子关键字（如整数拆分为个位、十位、百位），按子关键字权重递增（LSD，最低位优先）或递减（MSD，最高位优先）排序。
    • 核心思想：以 rrr 为基数（如十进制 r=10r=10r=10），使用 rrr 个队列，通过“分配”和“收集”实现排序：
      1. 分配：对第 iii 位（从 0 到 d−1d-1d−1，ddd 为关键字位数），将每个元素按第 iii 位子关键字存入对应队列 QkQ_kQk​（kkk 为子关键字值）。
      2. 收集：将 Q0Q_0Q0​ 到 Qr−1Q_{r-1}Qr−1​ 中元素依次首尾相接，形成新的有序序列。
    • 算法步骤（LSD 法）：
      1. 确定关键字的位数 ddd（如最大元素为 3 位数，则 d=3d=3d=3）。
      2. 对 i=0i=0i=0 到 d−1d-1d−1（从最低位到最高位）：
        • 执行“分配”操作（按第 iii 位子关键字分入队列）。
        • 执行“收集”操作（合并队列，更新序列）。
    • 复杂度分析：
      • 时间复杂度：O(d(n+r))O(d(n+r))O(d(n+r))（ddd 为位数，rrr 为基数，与初始状态无关）。
      • 空间复杂度：O(r)O(r)O(r)（需 rrr 个队列存储元素）。
    • 性质/特点：稳定排序；适用于顺序存储和链式存储；关键字需可拆分为独立子关键字，且子关键字范围较小（避免队列过多）。

外部排序

  • 外部排序的基本概念

    • 定义：待排序文件无法全部存入内存，需在内外存间多次移动数据的排序（如 10TB 大文件），核心目标是减少 I/O 次数（磁盘 I/O 时间远大于内存运算时间）。
    • 核心阶段：
      1. 生成初始归并段：按内存缓冲区大小，将外存文件分为若干子文件，用内部排序排序后写回外存（称为“归并段”或“顺串”）。
      2. 多路归并：对初始归并段逐趟归并，直至合并为一个有序文件。
    • 总时间构成：外部排序总时间 = 内部排序时间 + 外存 I/O 时间 + 内部归并时间（I/O 时间占主导）。

  • 多路平衡归并与败者树

    • 多路平衡归并：增大归并路数 kkk 可减少归并趟数 S=⌈log⁡kr⌉S = \lceil \log_k r \rceilS=⌈logk​r⌉（rrr 为初始归并段数），从而减少 I/O 次数。
    • 败者树：优化多路归并的比较操作——一棵完全二叉树，叶结点为各归并段当前元素，内部结点记录“失败者”（大元素），根结点为“胜利者”（最小元素）。
      • 优势：从 kkk 个元素选最小元素仅需 O(log⁡2k)O(\log_2 k)O(log2​k) 次比较，与 kkk 无关，避免因 kkk 增大导致比较次数激增。

  • 置换-选择排序（生成初始归并段）

    • 定义：生成更长初始归并段的算法（减少归并段数 rrr），利用内存工作区 WA（容量 www），通过“置换”和“选择”生成长度大于 www 的归并段。
    • 算法步骤：
      1. 从外存读 www 个记录到 WA。
      2. 选 WA 中关键字最小的记录（MINIMAX 记录），输出到归并段文件。
      3. 若外存还有记录，读下一个记录到 WA 中 MINIMAX 记录的位置。
      4. 从 WA 中选关键字大于 MINIMAX 的最小记录，作为新的 MINIMAX 记录。
      5. 重复步骤 2-4，直至 WA 中无符合条件的 MINIMAX 记录，输出归并段结束标志；重复步骤 2-5 直至 WA 为空。

  • 最佳归并树

    • 定义：基于哈夫曼树思想，将初始归并段长度视为“权值”，构造带权路径长度（WPL）最小的 kkk 叉树，使归并过程的 I/O 次数最少（WPL 对应总读记录数，I/O 次数 = 2×WPL）。
    • 虚段补充：若初始归并段数 rrr 不满足 kkk 叉哈夫曼树条件（(r−1)%(k−1)=0(r-1) \%(k-1) = 0(r−1)%(k−1)=0），需补充长度为 0 的“虚段”，使总段数满足条件（虚段权值为 0，不影响 I/O 次数）。

  • 排序算法的选择原则

    1. 数据规模：小数据量（n<1000n<1000n<1000）选直接插入/简单选择排序；大数据量（n>1000n>1000n>1000）选快速/堆/归并排序。
    2. 初始状态：基本有序选直接插入/冒泡排序；随机分布选快速排序。
    3. 稳定性要求：需稳定排序选归并/基数/直接插入/冒泡排序；无要求选快速/堆排序。
    4. 存储结构：顺序存储可选所有算法；链式存储选直接插入/冒泡/归并/基数排序（避免希尔/堆/快速排序）。
    5. 关键字特性：关键字可拆分选基数排序；关键字信息量大选简单选择排序（减少移动次数）。
