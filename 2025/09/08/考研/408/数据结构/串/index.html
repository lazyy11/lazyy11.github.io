串的定义与基本操作

  • 串的定义

    • 定义：由零个或多个字符组成的有限序列，记为 S=′a1a2⋯an′S='a_1a_2\cdots a_n'S=′a1​a2​⋯an′​（n≥0n \geq 0n≥0），其中 SSS 为串名，单引号内字符序列为串值，aia_iai​ 可为字母、数字或其他字符，nnn 为串的长度。
    • 关键概念：
      • 空串：n=0n=0n=0 的串，用 ∅\emptyset∅ 表示，不含任何字符。
      • 空格串：由一个或多个空格（特殊字符）组成的串，长度为空格个数，非空串。
      • 子串：串中任意多个连续字符组成的子序列；包含子串的串称为主串。
      • 串的位置：字符在串中的序号；子串在主串的位置以其第一个字符在主串的位置表示。
      • 串相等：两个串长度相等且对应位置字符均相等。
    • 示例：串 A=′ChinaBeijing′A='China Beijing'A=′ChinaBeijing′（长度13）、B=′Beijing′B='Beijing'B=′Beijing′（长度7）、C=′China′C='China'C=′China′（长度5），BBB 和 CCC 是 AAA 的子串，BBB 在 AAA 中位置为7，CCC 为1。

  • 串的基本操作

    • 最小操作子集（不可用其他串操作实现）：
      • StrAssign(&T, chars)：赋值，将串 TTT 赋值为 charscharschars。
      • StrCompare(S, T)：比较，S>TS>TS>T 返回>0，S=TS=TS=T 返回0，S<TS<TS<T 返回<0。
      • StrLength(S)：求串长，返回串 SSS 的字符个数。
      • Concat(&T, S1, S2)：串联接，将 S1S1S1 和 S2S2S2 连接成串 TTT（文档提及未详细列，属最小子集）。
      • Substring(&Sub, S, pos, len)：求子串，用 SubSubSub 返回 SSS 中第 pospospos 个字符起长度为 lenlenlen 的子串。
    • 其他操作（可基于最小子集实现）：
      • StrCopy(&T, S)：复制，由串 SSS 复制得到 TTT。
      • StrEmpty(S)：判空，SSS 为空串返回TRUE，否则返回FALSE。
      • Index(S, T)：定位，主串 SSS 中存在与 TTT 相等的子串时，返回首次出现位置，否则返回0。
      • ClearString(&S)：清空，将 SSS 清为空串。
      • DestroyString(&S)：销毁，释放串 SSS 占用的存储空间。

  • 串与线性表的区别

    • 数据对象：串的数据对象限定为字符集；线性表的数据对象可为任意数据类型。
    • 操作对象：串的基本操作以“子串”为单位（如查找子串、插入子串）；线性表的基本操作以“单个元素”为单位（如查找单个元素、插入单个元素）。

串的存储结构

  • 定长顺序存储表示

    • 定义：类比线性表的顺序存储，用一组地址连续的固定长度存储单元存放串的字符序列，通常用“定长数组”实现。
    • 结构组成（C语言示例）：
    • 串长表示方法：
      1. 显式表示：用额外变量 length 存储实际长度（如上）。
      2. 隐式表示：在串值末尾加不计入长度的结束标记 '\0'（如C语言字符串）。
    • 性质/特点：
      • 优点：存储密度高，随机访问字符效率高。
      • 缺点：存在“截断问题”——若串值长度超过 MAXLEN，超出部分会被舍弃；无法动态调整存储空间。

  • 堆分配存储表示

    • 定义：仍用地址连续的存储单元存放串值，但存储空间在程序执行时动态分配（依赖“堆”内存区域）。
    • 结构组成（C语言示例）：
    • 存储管理：
      • 分配：通过 malloc() 函数申请与串长匹配的存储空间，成功则返回基地址给 ch，失败返回 NULL。
      • 释放：通过 free() 函数释放已分配的存储空间，避免内存泄漏。
    • 性质/特点：
      • 优点：无截断问题，存储空间利用率高（按需分配）。
      • 缺点：动态分配/释放需手动管理，易出现内存操作错误（如野指针、内存泄漏）。

  • 块链存储表示

    • 定义：类比线性表的链式存储，用链表存放串值；每个链表结点称为“块”，可存储1个或多个字符。
    • 结构组成：
      • 结点大小：
        1. 结点大小=1：每个结点仅存1个字符（存储密度低，指针开销大）。
        2. 结点大小>1（如4）：每个结点存多个字符，末尾不满时用特殊字符（如 '#'）填充（平衡存储密度与指针开销）。
    • 示例结构：
      • 结点大小=4的块链：head -> [A W D W] -> [B C # #]（存储串 'AWDWBC'）。
      • 结点大小=1的块链：head -> [A] -> [W] -> [D] -> [W] -> [B] -> [C]。
    • 性质/特点：
      • 优点：插入/删除子串时无需大规模移动字符，操作灵活。
      • 缺点：存储密度低于顺序存储，需额外存储指针域；随机访问效率低（需遍历链表）。

串的模式匹配算法

  • 暴力匹配法（简单模式匹配）

    • 定义：从主串起始位置开始，逐个字符与模式串比较，匹配失败则主串指针回溯、模式串指针重置，直至匹配成功或遍历结束。
    • 算法步骤（设主串 SSS 长度为 nnn，模式串 TTT 长度为 mmm，下标从1开始）：
      1. 初始化指针 i=1i=1i=1（主串当前比较位置），j=1j=1j=1（模式串当前比较位置）。
      2. 若 i≤ni \leq ni≤n 且 j≤mj \leq mj≤m：
        • 若 S.ch[i]==T.ch[j]S.ch[i] == T.ch[j]S.ch[i]==T.ch[j]，则 i++i++i++、j++j++j++（继续比较后继字符）。
        • 若 S.ch[i]!=T.ch[j]S.ch[i] != T.ch[j]S.ch[i]!=T.ch[j]，则 i=i−j+2i = i - j + 2i=i−j+2（主串回溯至上次匹配起始位置的下一位）、j=1j=1j=1（模式串重置）。
      3. 匹配结束判断：
        • 若 j>mj > mj>m，返回 i−mi - mi−m（匹配成功，返回主串中模式串首次出现的起始位置）。
        • 若 i>ni > ni>n，返回0（匹配失败）。
    • 复杂度分析：
      • 时间复杂度：最坏情况 O(nm)O(nm)O(nm)（如主串 '000...0'、模式串 '000...1'，每趟比较 mmm 次，共 n−m+1n-m+1n−m+1 趟）；平均情况近似 O(n+m)O(n+m)O(n+m)。
      • 空间复杂度：O(1)O(1)O(1)（仅用常数个指针变量）。
    • 缺点：主串频繁回溯，存在大量重复比较，效率低。

  • KMP算法

    • 核心原理：利用模式串自身的“部分匹配信息”（前缀与后缀的最长相等序列），避免主串指针回溯，仅调整模式串指针，降低时间复杂度。
      • 前缀：除最后一个字符外，串的所有头部子串（如 'ababa' 的前缀为 'a'、'ab'、'aba'、'abab'）。
      • 后缀：除第一个字符外，串的所有尾部子串（如 'ababa' 的后缀为 'a'、'ba'、'aba'、'baba'）。
      • 部分匹配值（PM）：前缀与后缀的最长相等序列长度（如 'ababa' 的PM表为 [0,0,1,2,3]）。
    • next数组
      • 定义：当模式串第 jjj 个字符失配时，模式串指针应跳转的位置（记为 next[j]），其值由PM表变形得到。

        1. 固定值：next[1] = 0（第1个字符失配，主串与模式串指针同步后移）、next[2] = 1（第2个字符失配，模式串指针重置为1）。
